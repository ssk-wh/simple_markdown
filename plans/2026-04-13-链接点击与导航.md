# 预览区链接点击与导航

## 背景

当前预览区能够正确渲染 Markdown 中的链接（`[text](url)`、裸 URL），但**无法被激活**：点击只影响选区/光标，链接字段 `InlineRun::linkUrl` 只用于样式染色。用户要把预览当「文档浏览器」用时无法跳转，必须复制 URL 到浏览器或打开文件管理器手动点目标文件，体验割裂。

设计目标是在不破坏现有选区/拖选/双击选词交互的前提下，用 **Ctrl+左键** 作为明确的跳转触发，并根据链接类型分发到合适的处理通道：本地 Markdown → 应用内新 Tab；可执行文件 → 拒绝；其他文档 → 系统默认应用；`http(s)`/`mailto` → 系统浏览器/邮件客户端。

## 目标

落地 `specs/模块-preview/09-链接点击与导航.md` 中的所有 INV-1 到 INV-14，使 T-1 到 T-15 全部通过。

具体可观测目标：

1. 用户 Ctrl+点击 `[doc](sub/a.md)` 时在新 Tab 打开 `sub/a.md`
2. 用户 Ctrl+点击 `https://...` 时系统浏览器弹出对应页面
3. 用户鼠标悬停时光标变手型；按 Ctrl 时链接视觉强调
4. 对 `.exe`/`.bat`/`.sh` 等的链接一律拒绝并在状态栏提示
5. 不存在的文件不崩溃，给出友好提示

## 风险与回滚

### 风险 1：改动 mousePressEvent 可能破坏现有选区行为

**影响**：用户反馈点击定位光标失效、拖选失效。

**缓解**：
- 只在「命中链接 + Ctrl 按下 + 左键」三条件全部满足时吃事件，其余情况原样派发
- 增加测试 T-3 专门覆盖非 Ctrl 左键点击链接的情况
- 代码审查时对 `mousePressEvent` 分支优先级做重点 review

### 风险 2：Ctrl 状态与 QApplication::keyboardModifiers() 不同步

**影响**：悬停时按 Ctrl 不高亮，或者释放后仍显示强调。

**缓解**：完全依赖 `keyPressEvent`/`keyReleaseEvent` 维护 `m_ctrlDown` 成员状态，不读 `QApplication::keyboardModifiers()` 作为实时真值源（见 Spec 陷阱 8.1）。

### 风险 3：安全绕过（用户被诱导点击隐藏 exe）

**影响**：恶意 Markdown 引导用户 Ctrl 点击 `[readme](./setup.exe)` 执行木马。

**缓解**：INV-6 黑名单强制拒绝，INV-7 路径规范化。单元层面 T-9 覆盖常见可执行扩展名。**安全审查**阶段用 `security-reviewer` agent 复审 `activateLink` 与 `classifyLink` 两个函数的分支完整性。

### 风险 4：高 DPI 下链接矩形与文本渲染位置漂移

**影响**：悬停光标显示在链接右侧 10px 的空白处，无法命中。

**缓解**：严格遵守 `横切关注点/40-高DPI适配.md` INV-2，所有 `QFontMetricsF` 调用带 `viewport()` 或 `p->device()` 参数。T-11 覆盖此回归。

### 风险 5：文件不存在时弹窗打断用户

**影响**：用户连续 Ctrl 点击多个链接，对每个不存在文件弹模态窗口。

**缓解**：INV-13 规定只走状态栏，不使用 `QMessageBox`。

### 回滚方案

所有改动集中在 5 个文件（PreviewWidget.h/cpp, PreviewPainter.cpp, MainWindow.cpp, Spec 文件）。若线上出问题，`git revert` 对应的 commit 即可。Spec 文件保留，status 标为 `aborted` 或降级为 `draft`，不必删除。
