# 预览区双击选词与复制粘贴异常

## 背景

用户在 2026-05-13 反馈，预览区域存在两类交互 bug：

1. **双击选词偶发性选不全整个单词**
   - 复现现象：在预览区双击某个单词，期望选中整个词，但实际有时只选到部分字符 / 邻近字符 / 漏掉首尾。
   - 出现频率：偶发，疑似与点击位置（字符边界 vs 字符中部）相关。

2. **复制后无法粘贴**
   - 复现现象：在预览区框选一段内容 → 复制（Ctrl+C 或右键 Copy）→ 切到其他程序（如记事本 / 浏览器地址栏 / 输入框）粘贴时无内容，或粘贴目标拒绝接收。
   - 出现频率：未知，需补充复现场景。

## 受影响范围

- 入口 1：`PreviewWidget::mouseDoubleClickEvent`（src/preview/PreviewWidget.cpp:575）
- 入口 2：`PreviewWidget::copySelection`（src/preview/PreviewWidget.cpp:722）
- 涉及剪贴板 MIME 投递（同文件 :737 / :758）

## 初步推测（先不动手修复，待复现确认）

**Bug 1 双击选词**：
- 怀疑路径：双击 → `textIndexAtPoint` snap 行为（红旗清单反模式 C）→ 单词边界判定基于 snap 后的 index 错位。
- 也可能是字符 segment 切分阶段就把"单词"拆开了（CJK / 英文混排分词不一致）。

**Bug 2 复制无法粘贴**：
- 怀疑路径 A：`setMimeData` 投的 payload 只含富文本（HTML / custom MIME），漏了 `text/plain` 兜底，导致只接收 plain text 的目标拿不到内容。
- 怀疑路径 B：MIME data 被设置后立即被覆盖 / 被 owner 释放（Qt 剪贴板的 ownership 陷阱）。
- 怀疑路径 C：单实例机制 + 进程间剪贴板交互的兼容问题。
