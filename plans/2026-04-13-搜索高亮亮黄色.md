# 搜索高亮改为亮黄色

## 背景

用户反馈：在编辑区按 Ctrl+F 搜索后，匹配文本的视觉提示不够明显，要在屏幕里慢慢找才能定位到匹配位置。

现状（`src/core/Theme.h` / `src/core/Theme.cpp`）：

- 浅色主题 `editorSearchMatch = QColor(255, 235, 59, 128)`（alpha 128 半透明黄），与白底叠加后明显发淡，对比度低
- 浅色主题 `editorSearchMatchCurrent = QColor(255, 152, 0, 180)`（半透明橙）
- 深色主题 `editorSearchMatch = #613214`（暗棕色，与 `#1E1E1E` 背景几乎同色）
- 深色主题 `editorSearchMatchCurrent = #A05A2C`（暗橙色）

需要把匹配高亮改为更显眼的**亮黄色**（类似浏览器/IDE 的 search highlight），与选区色形成明显区分，并保证深色主题下也足够醒目。

> 备注：`specs/模块-editor/06-查找替换.md` 当前不存在，本 Plan 仅在 related_specs 中留指针，待主线程后续补 Spec。

## 动作

- [ ] 在 `src/core/Theme.h` 调整两条字段的浅色默认值：
  - `editorSearchMatch`：浅色主题改为不透明亮黄 `#FFEB3B`（或 `#FFF176` 偏柔和）
  - `editorSearchMatchCurrent`：浅色主题改为更饱和的橙黄 `#FFA000`（或同色 + 边框样式）
- [ ] 在 `src/core/Theme.cpp` 的 `Theme::dark()` 中调整深色主题对应字段：
  - `editorSearchMatch`：暗金黄 `#B8860B` 或 `#A88B00`，保证 `editorFg = #D4D4D4` 在其上仍清晰可读
  - `editorSearchMatchCurrent`：更亮的金黄 `#DAA520`，与"全部命中"形成视觉差异
- [ ] 在 `src/editor/EditorPainter.cpp` 的搜索高亮绘制段（约第 47-85 行 `painter->fillRect` 处）保持现有逻辑，仅依赖 Theme 字段；如改用"边框 + 填充"区分当前命中，则在 `i == currentMatchIndex` 分支额外绘制 1-2px 边框（颜色用 `editorSearchMatchCurrent` 加深）
- [ ] 验证绘制顺序：当前是先 fillRect（背景高亮）→ 再 `tl->draw`（文字+选区）。亮黄色填充应使用**不透明**色，文字由 QTextLayout 在其上绘制，不会被遮挡；选区由 `FormatRange` 在文字层叠加，与高亮叠合时仍可识别（选区蓝色 + 高亮黄 ≈ 偏绿，可接受）
- [ ] 预览区无 search 功能（`src/preview/` 内未发现 search/match 相关代码），**不需要同步预览侧**
- [ ] CHANGELOG.md 在 `[0.2.3] - 2026-04-13` 的 `### Changed` 段追加：「编辑器搜索高亮改为亮黄色，搜索结果更显眼」

## 验收

- [ ] 浅色主题下匹配文本背景为亮黄色（`#FFEB3B` 级别），与白色正文形成明显对比，扫一眼即可定位
- [ ] 深色主题下匹配文本背景为暗金色（`#B8860B` 级别），与 `#1E1E1E` 背景对比明显，文字（`#D4D4D4`）仍可读
- [ ] 当前命中（按 F3 / Enter 循环到的那一个）与其他匹配在视觉上有明显区分（更饱和的橙黄 / 加边框任选其一）
- [ ] 选区色（`editorSelection`）与匹配高亮色叠加时，仍可识别"这是被选中的匹配项"
- [ ] 切换主题（菜单 → 主题）后颜色立即生效，无需重新搜索或重启
- [ ] 高亮在视觉行换行处（同一匹配跨多个视觉行）仍然连贯
