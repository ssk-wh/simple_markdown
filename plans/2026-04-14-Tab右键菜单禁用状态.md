# Tab 右键菜单项禁用状态

## 背景

`src/app/MainWindow.cpp:139-152` Tab 右键菜单的 Close / Close Others /
Close to the Left / Close to the Right 四个项目目前**无条件启用**,导致在无意义场景
下用户仍能点击——虽然操作结果是"无事发生",但体验上像是软件没响应。

典型场景:

- 只有 1 个 Tab:"关闭其他""关闭左侧""关闭右侧" 三项都没有可操作对象,应该全部灰掉
- 右键最后一个 Tab(idx == 最后):"关闭右侧"应灰掉
- 右键第一个 Tab(idx == 0):"关闭左侧"应灰掉

已有 `Open Containing Folder` 根据 `filePath.isEmpty()` 做了 `setEnabled(false)`
(MainWindow.cpp:170),这次的修复沿用同样模式。

## 动作

- [ ] 修改 `src/app/MainWindow.cpp` 的 tab 右键菜单构造逻辑:
  - `Close Others`:`m_tabs.size() > 1` 才启用
  - `Close to the Left`:`idx > 0` 才启用
  - `Close to the Right`:`idx < m_tabs.size() - 1` 才启用
  - `Close`:保持始终启用(当前 tab 本身总可关)
- [ ] 更新 `specs/模块-app/04-窗口焦点管理.md` 或新增 Tab 右键菜单的 Spec 段落:
  加 INV「右键菜单项必须反映真实可执行状态(可关闭对象 > 0 才启用)」+ 对应 T 编号
- [ ] 更新 CHANGELOG.md (Fixed 段或 Improved 段)

## 验收

- 只 1 个 Tab 时,右键菜单的"关闭其他/左侧/右侧"均灰色不可点
- 右键首个 Tab 时"关闭左侧"灰,"关闭右侧"可用(若还有其他 tab)
- 右键末个 Tab 时"关闭右侧"灰
- 中间位置 Tab 四项都可用
- 多语言:中英文 UI 下灰化状态一致
