# pack 脚本自动调用 build

## 背景
当前 `pack_on_win.bat` / `pack_on_linux.sh` 假设用户已执行过 `build_on_*.{bat,sh} release`。多次出现"打包了旧二进制"的问题（CLAUDE.md 已记录）。Spec `specs/30-构建与发布流程.md` 已要求一键打包能力。

## 动作
- [ ] `pack_on_win.bat` 开头调用 `call build_on_win.bat release`，build 失败立即 `exit /b 1`
- [ ] `pack_on_linux.sh` 开头调用 `./build_on_linux.sh release`，失败 `set -e` 终止
- [ ] 收集依赖前再次校验 `installer/dist/SimpleMarkdown.exe` / `SimpleMarkdown` 时间戳新于 build 启动时间
- [ ] 文档 `docs/构建说明.md` 同步说明"打包 = 构建 + 收集 + 安装包生成"

## 验收
- 故意制造编译错误后跑 `pack_on_win.bat`，脚本应在 build 阶段终止，不进行后续打包
- 正常路径下从干净 build 目录直接 `pack_on_*` 能产出可用安装包
- 时间戳校验逻辑能拦截"build 跳过但 pack 继续"的情况
