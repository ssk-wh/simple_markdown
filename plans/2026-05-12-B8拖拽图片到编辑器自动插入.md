# B8 拖拽图片到编辑器自动插入

## 背景

当前 SimpleMarkdown 支持拖拽 `.md` / `.markdown` / `.txt` 到窗口打开文件
（产品愿景 §4 文件操作）。但**拖拽图片**到编辑区现在估计是当文件打开（失败）
或无响应。

期望：拖图片到编辑区光标位置 → 复制到 `./images/` + 插入 `![](images/...)`。
逻辑可复用 B7 的图片保存路径决策。

## 动作

- [ ] Step 1: EditorWidget 接受 `dragEnterEvent` 中 mimeData 含 file URL 且扩展名是图片
- [ ] Step 2: 区分"打开文档"（.md/.markdown/.txt）和"插入图片"（.png/.jpg/.gif/.webp/.svg）
- [ ] Step 3: 调用 B7 的同一组路径决策 + 文件复制（**不是移动**）
- [ ] Step 4: 拖到编辑区/预览区/标签页区的行为分别明确：
  - 编辑区 → 复制+插入 MD 语法
  - 标签页区 → 拒绝（避免被误当作"打开"）
  - 预览区 → 同编辑区行为或拒绝（择一，建议拒绝避免歧义）
- [ ] Step 5: 多文件拖拽 → 依次插入

## 验收

- 从文件管理器拖 PNG 进编辑区光标处 → images/ 目录出现复制文件 + 文档插入 `![](images/<name>.png)`
- 拖 .md 文件到编辑区 → 按"打开文档"逻辑（产品愿景已支持，行为不变）
- 拖 .jpg 到预览区 → 无反应或友好提示
- 拖大图（> 20MB）→ 友好拒绝（沿用 1.0.x 在线图片的 20MB 上限）

## 备注

- 与 B7 共用「图片保存路径决策」helper，避免「同一语义两份独立代码」反模式
- 建议 **B7 先做，B8 复用 B7 的 helper**
