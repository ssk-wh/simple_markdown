# 视图菜单可选中项与普通项文本左侧不对齐

## 背景

用户报告（2026-06-15）：

> 视图菜单中，如"自动换行"这种支持有选中效果（checkable）的菜单项，和"行间距"这种
> 没有选中效果的菜单项，文本的左侧不对齐，我希望他们对齐。

现象：在「视图」菜单里，checkable 菜单项（带勾选标记，如自动换行）与非 checkable 菜单项
（如行间距子菜单入口）的文本左缘没有对齐，视觉参差。期望所有菜单项文本左缘对齐
（checkable 项的勾选标记占位列应在文本左侧统一预留，非 checkable 项同样缩进对齐）。

## 根因方向（待验证）

- Qt 默认：同一 QMenu 内若有 checkable 项，菜单会为勾选标记预留一列，非 checkable 项
  文本本应自动对齐到同一缩进。出现不对齐通常是因为：
  1. 部分项设了 icon，部分没设 → icon 列宽度差异导致文本起点不一
  2. checkable 项与子菜单项（带 ▶ 箭头）混排，样式表覆盖了默认 padding
  3. 自定义 QSS 改了 `QMenu::item` 的 padding/indicator 宽度，未统一
- 需先定位「视图」菜单的构建代码，确认哪些项 checkable、哪些带 icon/子菜单、是否有 QSS。

## 目标（验收条件）

- [ ] 定位视图菜单构建代码与相关 QSS
- [ ] 视图菜单所有项文本左缘对齐（截图 before/after 对比）
- [ ] 不破坏勾选标记、子菜单箭头的正常显示
- [ ] 若属可感知交互样式 → 在 specs/模块-app 下补菜单对齐 Spec/INV

## 进展（补充）

- 2026-06-15 定位 + 修复（截图验证）：
  - 根因：菜单 QSS `QMenu::indicator { ...; margin-right: 8px; }`。Qt(5.12 stylesheet) 把
    indicator 内联绘制在文字前，margin-right 把 **checkable 项文字额外推右**，非 checkable 项
    不受影响 → 文字左缘错位（违反 spec 10 既有 INV-2）。
  - 修复：去掉 indicator 的 margin-right（两处 QSS：浅色 + 深色），改为 `width:16; height:16;`。
    ✓ 与文字间距由 indicator 列宽内部留白提供。
  - 验证：启动应用打开视图菜单截图（tmp/menu_v2.png / menu_toggle.png），主题/自动换行/行间距/
    放大/缩小/重置缩放/切换侧边栏/演示模式/标签栏位置/显示区域 文字左缘均对齐成一条直线。✅
  - Spec：10-菜单栏样式.md INV-1 增补"禁止 indicator margin-right"修订说明。CHANGELOG：1.1.10。
  - 旁注（不在本任务范围）：当前样式下 checkable 项的 ✓ 勾选标记视觉不明显（去 margin-right
    逻辑上不影响 ✓ 渲染，属既有现象）。如需让勾选态更可辨，可后续单开任务加 indicator:checked 图标。
