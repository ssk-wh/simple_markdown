# 预览区支持本地图片与在线图片

## 背景

用户原话：
> 支持本地图片和在线图片

当前预览区对 Markdown 图片语法 `![alt](src "title")` 的处理（查代码确认细节）：
- 本地相对 / 绝对路径：能否直接加载需看 `ImageCache` 是否支持
- `http` / `https` URL：当前很可能未异步下载，显示占位 / 裂图
- `data:image/...` base64：未验证

`specs/模块-preview/05-图片缓存.md` 已定义缓存，但在线图片下载（网络 I/O）要另起一层。

## 动作（待细化）

- [ ] 审计当前 `ImageCache` / `PreviewPainter` 对图片 URL 的识别与加载路径
- [ ] 本地路径：支持绝对路径、相对当前文档路径、`file://` URI
- [ ] 在线图片：`http` / `https` 通过 `QNetworkAccessManager` 异步下载到本地缓存目录（`%APPDATA%/SimpleMarkdown/image_cache/<sha256>`），避免重复下载
- [ ] data URI（`data:image/png;base64,...`）：直接解码成 QImage，不进缓存
- [ ] 下载中：显示占位图（alt 文本 + 加载指示）
- [ ] 失败：显示裂图占位 + tooltip 原因（网络 / 404 / 不支持的格式）
- [ ] 超时：默认 10s，可在设置中改
- [ ] 安全：限制单图最大 20MB，总缓存最大 500MB（LRU），只接受 `image/*` MIME
- [ ] 右键菜单：「复制图片 / 另存为 / 在浏览器打开」
- [ ] 主题化：占位图配色跟随主题
- [ ] Spec：扩展 `05-图片缓存.md`，新增 INV-IMG-NET（在线图片异步不阻塞）、INV-IMG-CACHE-LRU、INV-IMG-SIZE-LIMIT；新增 T 条目覆盖本地/在线/data-URI/超时/失败

## 验收（草稿）

- **T1** 本地相对路径图片正确显示（`![](images/foo.png)`，文档在 `D:/a/b/a.md`，图片在 `D:/a/b/images/foo.png`）
- **T2** 本地绝对路径图片正确显示
- **T3** `file://` URI 正确显示
- **T4** 在线图片异步加载：首次显示占位，下载完成后自动刷新；不阻塞 UI
- **T5** data URI 正确解码显示
- **T6** 网络失败显示裂图 + tooltip 错误原因
- **T7** 断网后从缓存加载仍可显示（曾加载过的图片）
- **T8** 超大图片（>20MB）拒绝加载并提示
- **T9** 总缓存超过上限后自动 LRU 淘汰最久未用
- **T10** 右键菜单三项功能生效

## 风险

- **SSL / 证书**：Qt `QNetworkAccessManager` 需要编译时带 SSL 模块，否则 `https` 请求失败
- **私有资源（401 / 403）**：要不要支持 Cookie / Basic Auth？本版本 V1 不支持，后续可加配置
- **SVG 渲染**：Qt 需要链接 Qt5::Svg 模块
- **GIF 动画**：QMovie 支持但对位图缓存策略要特殊处理
- **内存占用**：大图解码到 QImage 是原始 RGBA，1920×1080 ≈ 8MB；应做缩放（按预览区显示尺寸）
- **线程安全**：下载在网络线程、解码在工作线程、绘制在 UI 线程；QImage 要深拷贝移交
- **隐私**：文档中的在线图片会触发网络请求，可能泄露用户正在阅读的内容（如 GitHub issue 里的图片）；设置中应提供「禁用在线图片」开关
