# 行内代码(inline code)垂直基线偏上

## 背景

预览区渲染含行内代码的段落时,`` ` `` 包裹的部分视觉上比同一行其他文字**偏上**,
未保持在统一基线(baseline)。典型复现文本:

> 版本号从 `CHANGELOG.md` 首条目自动提取

其中 `CHANGELOG.md` 明显比"版本号从""首条目自动提取"更靠上。

## 可疑根因

`src/preview/PreviewLayout.cpp:308` 的 `AstNodeType::Code` 分支:

```cpp
case AstNodeType::Code: {
    InlineRun run;
    run.text = child->literal;
    run.font = m_monoFont;
    run.font.setPointSizeF(currentFont.pointSizeF() * 0.9);  // 字号缩到 90%
    ...
}
```

行内代码同时做了两件事:**换字体**(等宽)**+ 缩字号**(×0.9),后续 PreviewPainter 绘制时
若每个 run 各自按自身 font 的 ascent 绘制,而非统一用行主字体的 baseline,就会出现
小字号文本的底部与主文本 baseline 不对齐——视觉表现就是"偏上"。

需要在 PreviewPainter.cpp 的 inline run 绘制循环里核实:
- 是否用 `painter->drawText(x, baselineY, run.text)`,`baselineY` 是否统一取行主字体 ascent
- 还是每个 run 各按 `run.font` 的 metrics 单独算 y

## 动作

- [ ] 读 PreviewPainter 的 inline run 绘制循环,确认当前 baseline 计算方式
- [ ] 复现:准备一份只有一行 "版本号从 `CHANGELOG.md` 首条目自动提取" 的 md,渲染截图,
  放大对比基线
- [ ] 修复方案预判(任选):
  - (A) 绘制时统一 baselineY = line.top + lineAscent(主字体),run 的 font 只影响字形不影响 y
  - (B) 对 code run,y 偏移 = `fm_main.ascent() - fm_code.ascent()`,让 code 的底部对齐主字体底部
- [ ] 回归确认:
  - 纯文本段落仍在同一基线
  - 粗体/斜体/代码混排段落基线一致
  - 代码块(CodeBlock,非 inline)不受影响
- [ ] 更新 specs/模块-preview/03-绘制管线.md:补 INV 约束「inline run 必须共享行基线」+ 对应 T 编号
- [ ] 更新 CHANGELOG.md (Fixed 段)

## 验收

- 行内代码与周围文字底部对齐(可目视也可截图像素级比较字形底部 y)
- 加粗、斜体、删除线、链接等其他 inline run 与普通文本基线一致
- 新增 Spec INV + T 编号,未来回归可自动检查
