# 打开主题目录为空时引导用户自定义主题

## 背景

用户原话：
> 打开主题目录后，其中是空的，应提供一种让用户自定义的方式，要让用户知道如何自定义，
> 或提供自定义模板或提供说明文档或在线链接的方式

当前行为（`src/app/MainWindow.cpp::openThemeDirectory`）：
- 菜单 视图 → 主题 → 打开主题目录：用系统文件管理器打开 `%APPDATA%/SimpleMarkdown/themes/`
- 首次使用时该目录为空（内置 8 款主题从 `:/themes/*.toml` 资源加载，不落盘）
- 用户打开目录只看到空文件夹，无从下手

## 目标

让"打开主题目录"对新手立即可操作。三条候选路径（可组合）：

- **A. 模板注入**：目录为空时自动复制一份 `template.toml` + `README.md`（写明字段说明）
- **B. 文档链接**：在目录里生成 `HOW_TO.md`（离线）或菜单项里加入「主题开发指南」（打开项目 docs 链接）
- **C. 复制内置主题**：勾选一款内置主题 → 菜单项「复制到主题目录」生成一份可编辑副本

最小可用版（MVP）：**A + B 组合**，无需菜单改动，openThemeDirectory 时侦测空目录并写入 2 个文件。

## 动作

- [ ] 新建 `resources/theme_docs/template.toml`（带中文注释的完整字段示例，含 8 大色组 + frontmatter）
- [ ] 新建 `resources/theme_docs/HOW_TO.md`（300 字说明 + 示例 key/value）
- [ ] 把上述两文件加入 `resources/resources.qrc`（放在 `:/theme_docs/` 前缀）
- [ ] 修改 `MainWindow::openThemeDirectory`：
  - 打开目录前 `QDir(path).entryList(QDir::Files | QDir::NoDotAndDotDot)` 判空
  - 若为空：从 `:/theme_docs/template.toml` 和 `:/theme_docs/HOW_TO.md` 拷贝到目录
  - 写入前提示 QMessageBox::information：
    > 已为你生成主题模板与说明文件（template.toml / HOW_TO.md）。
    > 复制一份并改名重命名为你自己的主题，修改后保存，回到"主题"菜单点"重新扫描主题"即可看到。
- [ ] i18n：新增字符串走完 tr() + lupdate + 补 zh_CN/en_US + lrelease（参照 feedback memory）
- [ ] Spec 更新：`specs/模块-app/12-主题插件系统.md` 新增
  - INV：openThemeDirectory 空目录必须注入模板 + 说明
  - T：首次打开目录后，目录中至少有 template.toml、HOW_TO.md 两个可读文件
- [ ] CHANGELOG 0.2.x 下加一行 Added

## 验收

- **T1** 首次打开主题目录（目录不存在或为空），自动创建 `template.toml` + `HOW_TO.md` 并弹窗说明
- **T2** 已有自定义主题（目录非空）时，打开目录不注入、不弹窗，直接打开资源管理器
- **T3** 跟随 template.toml 改一份复制改名，点"重新扫描主题"，新主题出现在 Theme 菜单下
- **T4** zh_CN/en_US 下弹窗文本全翻译，无英文残留
- **T5** template.toml 内容文字在 UTF-8/GBK 系统上都能正常被 ThemeLoader 解析（验证编码稳定）

## 风险

- **用户反复打开同一空目录**：生成后用户手动删除再打开——应再次注入（空即注入是幂等的，OK）
- **写权限失败**：Windows 受限账户下写 `%APPDATA%` 应该 OK，Linux 下 `~/.config/SimpleMarkdown/themes` 同样 OK；
  如果失败需捕获 QFile::error 并在弹窗中显示原因，不得崩溃
- **模板过时**：Theme schema 扩展（如新增 preview.inline_code_bg 字段）后，template.toml 要同步更新
- **与"在线链接"选项的取舍**：本版本不加在线链接（避免网络请求 + 对离线用户不友好），全部靠离线 HOW_TO.md
