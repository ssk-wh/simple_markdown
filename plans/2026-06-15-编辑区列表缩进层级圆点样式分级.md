# 列表缩进层级使用不同圆点样式分级

## 背景

用户需求（2026-06-15）：

> 对于 " - " 开头的行会渲染一个圆点出来，然后下面的一行再增加两个缩进，输入 "-"，
> 此时还是会渲染一个圆点，但和上面的圆点 x 坐标上有缩进。我觉得最好如果缩进了一级，
> 就别再继续用实心圆点了，比如可以换成空心圆点；再缩进就是实心菱形，之后是空心菱形，
> 接下来三角形这样子。

即无序列表的项目符号应**随嵌套层级变化样式**，而非所有层级都用同一实心圆点。

## 期望的层级符号序列（用户给定）

| 缩进层级 | 符号 |
|---------|------|
| 0（顶层） | ● 实心圆点 |
| 1 | ○ 空心圆点 |
| 2 | ◆ 实心菱形 |
| 3 | ◇ 空心菱形 |
| 4+ | ▶/△ 三角形（再深可循环或保持三角） |

## 待确认/设计

- [ ] 层级超过 5 级后如何处理（循环复用 / 保持最深一级符号）
- [ ] 编辑区与预览区符号是否要完全一致（应一致，避免双区不同源——反模式 B）
- [ ] 符号字形选择：用 Unicode 字符（●○◆◇▶）还是自绘几何图形（自绘更可控对齐/大小）
- [ ] 有序列表不受影响（仅无序列表 bullet 分级）

## 目标（验收条件）

- [ ] 无序列表项目符号按缩进层级呈现 实心圆→空心圆→实心菱形→空心菱形→三角形
- [ ] 编辑区与预览区符号样式一致
- [ ] 符号与文本基线对齐、大小协调（高 DPI 下不偏移）
