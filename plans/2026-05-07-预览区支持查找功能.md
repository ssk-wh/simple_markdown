# 预览区域支持查找功能

## 背景

编辑区已有 Ctrl+F 查找 / 替换栏（`src/editor/SearchBar`）。预览区目前**不支持**查找——
用户在阅读长文档时无法快速定位文本，体感上需要切回编辑区使用查找、跳转、再切回预览。
对于纯阅读场景（演示模式 / 单预览模式）尤其不便。

## 动作

- [ ] 评估 SearchBar 复用策略：
  - 路径 A：抽公共 `SearchBar` 组件到 `src/core` 或 `src/common`，editor / preview 共用
  - 路径 B：预览侧独立 `PreviewSearchBar`（功能子集，不含 replace）
  - 推荐 A，避免 UI 风格漂移
- [ ] 预览侧查找需求范围：
  - 搜索源：`PreviewWidget::m_plainText`（已是 extracted plain text，可直接复用）
  - 高亮：全部匹配高亮 + Enter 下一个 / Shift+Enter 上一个
  - **不**支持替换（预览区不可编辑）
  - 大小写敏感、整词匹配等开关与编辑器侧一致
- [ ] 视觉与标记（荧光笔）区分：
  - 搜索高亮用 theme.searchHighlight 色，与 theme.markerHighlight 不同
  - 当前命中项用更醒目的 currentSearchHit 色
  - 标记 + 搜索高亮重叠时叠加渲染（用透明度区分）
- [ ] 滚动到当前匹配项：复用 `PreviewLayout::sourceLineToY` 或字符偏移 → Y 映射
- [ ] 快捷键：
  - 预览区获焦时 Ctrl+F 弹出查找栏
  - Esc 关闭并清除高亮
  - F3 / Shift+F3 上下跳转（跟编辑器侧对齐）
- [ ] i18n：所有可见字符串 `tr()` 包裹 + 同步 `translations/*.ts` + lrelease

## 涉及模块

- `src/preview/PreviewWidget.h/cpp` 添加 SearchBar 子 widget + 高亮渲染
- `src/preview/PreviewPainter.cpp` 增加搜索高亮绘制（与现有 marker 高亮分两层）
- `src/app/MainWindow.cpp` Ctrl+F shortcut 路由：根据当前焦点决定唤起 editor 或 preview 的查找栏
- `specs/模块-preview/` 新增「预览查找」spec，定义 INV + T 编号
- `translations/*.ts` 新增字符串

## 验收

- 预览区获焦后 Ctrl+F 弹出查找栏，输入文本即时高亮全部命中（防抖参考编辑器侧）
- 上/下匹配跳转 + 滚动到目标位置（命中位于视口中央或合适偏移）
- 关闭后高亮消失，预览内容回归原样
- 大小写、整词开关切换正常工作
- 与现有标记（荧光笔）共存，视觉清晰可区分
- 演示模式 / 单预览模式下均可用
- 中英文 i18n 字符串完整

## 待讨论

- 是否支持正则查找？编辑器侧当前不支持，统一不支持以保持一致
- 当编辑器和预览都打开时 Ctrl+F 唤起哪一侧？基于焦点路由（focusWidget），还是参考 displayMode 默认侧
