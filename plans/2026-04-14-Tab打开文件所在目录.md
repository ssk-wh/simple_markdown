# Tab 打开文件所在目录

## 背景
用户希望在 Tab 右键菜单上增加"打开所在目录"功能，点击后用系统文件管理器打开该 Tab 对应文件所在目录，并且尽量**高亮选中该文件**。对未保存的 Tab（filePath 为空）此菜单项置灰。

现状（`src/app/MainWindow.cpp` ~L130-151）：Tab 右键菜单已存在，包含 Close / Close Others / Close to the Left / Close to the Right 四项。每个 Tab 的文件路径可通过 `m_tabs[idx].editor->document()->filePath()` 获取（同文件 L1426 已有先例用法）。

## 动作
- [ ] 在 `src/app/MainWindow.cpp` 的 Tab 右键菜单 lambda（`tabBar customContextMenuRequested` 连接处）中新增 `Open Containing Folder` 菜单项
  - 放在 Close 系列之前并加一个 `menu.addSeparator()`，视觉上与关闭类操作区分
  - 从 `m_tabs[idx].editor->document()->filePath()` 取 path
  - 若 path 为空 → `action->setEnabled(false)`
- [ ] 新增私有 slot 或内联 lambda `revealInFileManager(const QString& path)`：
  - Windows：`QProcess::startDetached("explorer.exe", QStringList{ "/select,", QDir::toNativeSeparators(path) })`
    - 注意 "/select," 与路径是两个独立参数，不要拼在一起
  - Linux：优先尝试 `dbus-send` 调用 `org.freedesktop.FileManager1.ShowItems`（支持选中），失败则回退到 `QDesktopServices::openUrl(QUrl::fromLocalFile(QFileInfo(path).absolutePath()))`
  - macOS（未来兼容，可选）：`QProcess::startDetached("open", {"-R", path})`
- [ ] 失败时在状态栏显示短暂提示（`statusBar()->showMessage(...)`），不弹窗打断用户
- [ ] `translations/simple_markdown_zh_CN.ts` 补翻译：
  - `Open Containing Folder` → `打开所在目录`
  - 运行 `lrelease` 重新生成 `.qm`（或构建时自动处理）
- [ ] `CHANGELOG.md` 最新未发布版本 `Added` 段增补一条：
  - `Tab 右键菜单新增"打开所在目录"，Windows 下会在资源管理器中高亮选中文件`

## 非目标
- 不改 Tab 标题栏的左键行为
- 不在菜单栏（File 菜单）里加同名项（本次仅右键菜单）
- 不处理沙盒/Flatpak 等特殊发行版的文件管理器调用差异

## 验收
- [ ] 已保存 Tab 右键 → 看到"打开所在目录" / "Open Containing Folder"，可点击
- [ ] Windows：点击后资源管理器打开文件所在目录，并高亮选中目标文件
- [ ] Linux（GNOME/KDE 等主流 DE）：点击后文件管理器打开目标目录；支持 FileManager1 D-Bus 接口的环境下文件被选中
- [ ] 未保存（新建未存盘）的 Tab：该菜单项显示为灰色不可点
- [ ] 中文语言环境下菜单显示"打开所在目录"
- [ ] 路径含空格、中文、Unicode 字符时仍能正确打开与选中
- [ ] 不影响现有四个 Close 菜单项的行为
