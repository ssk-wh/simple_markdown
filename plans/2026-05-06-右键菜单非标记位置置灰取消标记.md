# 渲染区域右键菜单：非标记位置应置灰"取消标记"

## 背景
用户反馈：在渲染区右键，如果点击位置不在任何已标记内容上，"取消标记"action 应当置灰。

**核实代码现状（已读 `PreviewWidget::contextMenuEvent` line 493-551）**：

实际上**这个功能已经实现**——`PreviewWidget.cpp` line 532-539：

```cpp
QAction* clearCurrentAct = menu.addAction(tr("Clear Current Mark"), ...);
clearCurrentAct->setEnabled(hitMarkIdx >= 0);   // ← 命中检查 + 置灰
```

`hitMarkIdx` 由 line 525-530 计算：遍历 `m_highlights`，检查 `textIndexAtPoint(clickPt)` 是否
落入某个标记的 `[first, second)` 区间。空白处 `textIndexAtPoint` 返回 -1，比较自然为 false。

仅记录待办，本次不处理。

## 待复测确认（修复时）

用户报告意味着以下三种情形之一：

1. **功能正常但用户没注意到** —— 可能"取消标记"在用户视角与"清除所有标记"混淆；
   action 文字 `tr("Clear Current Mark")` 是否足够明显？或者用户根本没看到它已置灰
2. **textIndexAtPoint 边界 bug**：某些位置（例如行尾空白、段落间空白带）可能返回非 -1
   的合法 index，导致即便视觉上点在"非标记内容"也错误命中某个 mark
3. **`m_highlights` 越界**：编辑后 char offset 漂移，标记区间可能扩大或重叠到点击位置，
   即便用户感知"不在标记内"，char offset 上确实落入区间

## 动作（修复时）
- [ ] 让用户给一个能复现的样例 + 操作步骤（哪个文档、点哪里、菜单项是否如预期置灰）
- [ ] 写测试：构造文档 + 标记 + 模拟右键不同位置 → 检查 action enabled 状态
- [ ] 如属情形 2/3：定位 `textIndexAtPoint` 在空白区域的返回值，加边界处理
- [ ] 视觉层面也可考虑加 hover 提示（鼠标悬停标记上变手型？），强化"哪些是标记"的可视提示
- [ ] 不进 CHANGELOG（如属情形 1，没行为变化）；如属情形 2/3 修复，按用户视角描述

## 验收
- 在已有标记的文档上：点击标记内容右键 → "取消标记"可点击
- 点击非标记内容（空白 / 普通文本）右键 → "取消标记"明显置灰
