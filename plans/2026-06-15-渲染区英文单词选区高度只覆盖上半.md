# 渲染区英文单词选区高度只覆盖上半

## 背景

用户报告（2026-06-15）：

> 渲染区的英文单词选中后，选中的阴影色只有单词显示区域的上半部分被选中。

现象：在预览区选中英文单词时，选区高亮矩形的**高度不足**，只盖住单词的上半部分，
下半部分（字母 baseline 以下 / 行高下半）没被高亮，视觉上像选区被「腰斩」。

## Spec 现状

`specs/模块-preview/12-选区与拖动.md` 中**未定义选区高亮矩形的高度规则**（grep 无命中）。
→ 按 SDD 黄金法则，这属于「未定义行为」，**先补 Spec 的 INV + T 条目，再改代码**。

## 根因方向（待验证）

- 选区矩形高度可能用了 `ascent`、`fm.height` 不含 leading、或某个字号/字体的度量，
  而非整行的 **line height**（含 ascent + descent + leading）→ 只覆盖到 baseline 上方。
- 英文单词专属现象提示：可能与 InlineRun 的 ascent/descent 计算、或中英文混排时按
  ASCII 字体度量取高有关（CJK 行高 > ASCII 行高时，按 ASCII 高度画选区就矮了）。
- 命中 CLAUDE.md「反模式 A/B」嫌疑：选区矩形高度与文本绘制行高若由两份代码各算一份，
  易走样——需对齐到同一行高来源。

## 复现待补充

| 项 | 待补 |
|----|------|
| 纯英文 vs 中英混排行 | 中文字在同一行时英文选区是否也矮 |
| 字号 | 正文 / 标题 / 行内代码是否都复现 |
| DPI | 100% / 125% / 150% 下高度差异 |
| 单行 vs 跨行选区 | 跨行时每行高度是否都矮 |

## 目标（验收条件）

- [ ] Spec 12 新增 INV：选区高亮矩形高度 = 该行实际行高（与文本绘制同源）
- [ ] 选区高亮完整覆盖单词所在行的整行高度（含中英文混排）
- [ ] 加单元测试覆盖英文/混排/多字号选区高度
