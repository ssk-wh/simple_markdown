# A4 主线程长任务探针（性能可观测性）

## 背景

INV-6「主线程事件处理单次 < 16ms」当前只在 PreviewRenderBenchmark 里测，
日常用户使用时如果遇到卡顿，**没有任何遥测数据**可以定位。

加一个轻量长任务探针（仅 `--debug` 模式启用），把"用户感觉卡"转成可定位的数据，
也为后续 A1/A2/A3 提供量化决策依据。

零额外性能成本（release 模式编译期 disable）。

## 动作

- [ ] Step 1: 写一个 `LongTaskProbe` 工具类，用 QAbstractEventDispatcher 拦截 event 处理时长
- [ ] Step 2: 超过 16ms 的 event 写入 debug 日志（事件类型、widget 名、耗时、调用栈摘要）
- [ ] Step 3: 通过 `--debug` 命令行参数启用（已有该参数，1.0.4 引入）
- [ ] Step 4: 编译期开关：release 模式下整个 probe 类被 dead code elimination
- [ ] Step 5: 文档化使用方法到 `docs/构建说明.md` 或新增 `docs/性能诊断.md`

## 验收

- `--debug` 启动后人工触发卡顿场景（拖大文件 splitter）能看到 ≥ 1 条 > 16ms 的日志
- release 模式 `objdump`/`strings` 看不到 probe 类符号
- 不影响 INV-1「冷启动 < 500 ms」（probe 初始化耗时 < 1 ms）

## 备注

A1/A2/A3 都建议**先做 A4** 再启动——没有探针就没有"改前/改后"对比数据。
