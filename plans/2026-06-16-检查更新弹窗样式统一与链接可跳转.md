# 检查更新弹窗改为自定义 QDialog 风格 + 失败弹窗链接可跳转

## 背景

用户报告（2026-06-16）：

> 检查更新失败后的弹窗做成和更新历史这种界面一样的，尤其是关闭按钮的风格；以及弹窗
> 检查失败时，显示的链接要支持跳转。

当前检查更新的三种反馈（有更新 / 已最新 / 失败）用系统 `QMessageBox`，风格与项目自定义
`ChangelogDialog`（关于/更新历史用的自定义 QDialog）不一致——尤其关闭按钮样式不统一。
且失败时若展示了链接（如手动下载页地址），应可点击跳转。

## 目标（验收条件）

- [ ] 检查更新的弹窗（至少失败弹窗，理想三种都统一）改为自定义 QDialog，复用 ChangelogDialog
      的视觉风格（主题色、关闭按钮样式、布局）
- [ ] 失败/提示文案中的链接（如 releases 页面）以可点击富文本呈现，点击用浏览器打开
- [ ] 关闭按钮风格与「更新历史」对话框一致
- [ ] 所有字符串 tr() 包裹，同步 .ts / lrelease

## 设计方向（待定）

- 参考 ChangelogDialog 的构造（注入当前主题 `m_currentTheme`）+ QLabel 富文本
  （`setTextFormat(Qt::RichText)` + `setOpenExternalLinks(true)` 或 linkActivated 信号）。
- 可抽一个通用「信息对话框」helper（标题 + 富文本 + 关闭按钮），三种反馈共用。

## 实现

新增 `MainWindow::showUpdateDialog(title, richBody)`：自定义 QDialog，复用 ChangelogDialog 模式
（裸 `QPushButton("Close")` 不设样式 → 全局 QSS cascade 统一关闭按钮风格）；正文富文本 QLabel
`setOpenExternalLinks(true)` → `<a href>` 链接点击跳转浏览器。三种反馈（有更新/已最新/失败）统一改用它。
失败弹窗含可点击的 releases 页链接。Spec：23 INV-UPD-DIALOG-STYLE。

## 验证

- 编译 + 20 单测全过；关闭按钮风格与 ChangelogDialog 同模式（同一全局 QSS，逻辑保证一致）；
  链接跳转为标准 Qt setOpenExternalLinks 行为。i18n lupdate 同步 + 填翻译。
- 弹窗实际外观（QSS cascade 后）留用户复验（独立渲染看不到 MainWindow 全局样式）。
