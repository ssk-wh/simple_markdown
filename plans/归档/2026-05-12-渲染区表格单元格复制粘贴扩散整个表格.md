# 渲染区表格单元格复制粘贴扩散整个表格

## 背景

用户报告（2026-05-12）：

> 渲染区表格单元格我复制了之后，粘贴后实际粘贴了整个表格内容

预期：在表格的单个单元格内选中文字（或单元格内全部内容）→ Ctrl+C → 粘贴目标得到
**选中的那部分文字**（plain text 或 markdown 源文）。

实际：粘贴出来的是**整个表格**（所有行 + 所有列，可能含 `|` 表格语法或 tab 分隔
全表内容）。

## 复现条件（待用户/Agent 补充）

- [ ] 单元格内文字部分选中（如只选中一个词）→ 复制 → 粘贴
- [ ] 单元格全部内容选中（双击全选）→ 复制 → 粘贴
- [ ] 粘贴目标的差异：纯文本（终端 / 聊天）vs Markdown 编辑器（GitHub Issue / VS Code）
  - 已知 1.1.2 起预览复制默认写 `text/plain` + `text/markdown` 双 MIME，可能
    问题只出现在某一种 MIME 上
- [ ] 是否仅"渲染区"路径，编辑区源文相应位置选中复制是否正常

## 排查方向（推测，未验证）

1. **选区→字符流映射**：预览渲染区的 selection 通过 sourceLine / segment rect 定位，
   表格内单元格的"选中范围"可能被 selection extractor 误判为"整个 Table block"——
   特别是 1.0.8 修过的"表格内右侧空白拖动扩散到上下行"bug 之后，selection 算法
   有过改造，可能引入回归
2. **Markdown 序列化路径**：1.1.2 让 Ctrl+C 默认写 `text/markdown`，可能 markdown
   序列化把 table 当 atomic unit 输出，无视选区只截取部分
3. **HTML 路径**：右键 "Copy as HTML" 可能仅按 block 粒度 → 表格 block 一锅端
4. CHANGELOG 1.0.2「预览区双击选词和 Ctrl+C 复制内容不准确（含表格的文档下尤其明显）」
   ——历史上已经修过一次类似 bug，可能此次是子情况未覆盖

## 验收

- 表格单元格内选中一个词，Ctrl+C 后粘贴到纯文本编辑器 = 该词（不带 `|` 不带其他行）
- 同样选区粘贴到 Markdown 编辑器 = 该词 markdown 源文
- 选中跨多个单元格时合理（行内 = 多列拼接，跨行 = 多行表格 markdown 片段）
- 选中整个表格时 = 完整表格 markdown
- 不破坏 1.0.2 / 1.1.2 历史修复的现有用例

## 优先级

P1（功能性 bug，可工作日常使用场景受影响——表格复制是高频操作）。
A1 视口剪裁完成后可启动此 bug 修复。
