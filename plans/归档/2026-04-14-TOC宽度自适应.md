# TOC 目录面板宽度自适应

## 背景

用户原话：

> 右侧的目录列表（TOC）面板宽度应该**自适应**：
> - 最大不超过**屏幕宽度的 1/5**（防止占据太多写作空间）
> - 最小不小于**"最长一级目录标题的文本宽度 + 边距"**（保证最长 h1/h2 不被截断省略号）
> - 如果整个文档的所有标题都比较短，TOC 面板应缩窄到只够显示这些标题 + 少量 padding，
>   不浪费屏幕空间
> - 每次打开/切换文档后重新计算一次宽度；窗口 resize 时检查是否需要重新夹紧到 1/5 上限

当前实现（`src/app/MainWindow.cpp:1313-1314`）TOC 宽度写死 220px，`m_tocPanel->setMinimumWidth(160)`
也是硬编码。短文档浪费空间、长标题被裁剪成省略号。需要按实际标题内容动态计算。

## 规则（宽度公式）

令：

- `screen_width`   = 主窗口所在屏幕可用宽度（`QGuiApplication::screenAt(mapToGlobal(...))->availableGeometry().width()`）
- `padding_left`   = TOC 按钮左 padding 8px + 标题层级缩进 `(level-1)*14` + 右 padding 8px（沿用 `TocPanel.cpp:103-120`）
- `scrollbar_w`    = 垂直滚动条占位 5px（`TocPanel.cpp:74`）+ 安全余量 4px
- `extra_margin`   = TOC 容器外边距 8px（`m_listLayout` contentsMargins 左右和）

得：

```
max_width     = floor(screen_width / 5)
text_width_i  = fm_i.horizontalAdvance(entry_i.title) + (entry_i.level-1)*14
content_width = max(text_width_i for all entries) + padding_left + padding_right
              + scrollbar_w + extra_margin
min_width     = 120  // 空文档/极短标题时 placeholder 与标题栏"Contents"至少能完整显示
final_width   = clamp(content_width, min_width, max_width)
```

- `fm_i` 必须用 `QFontMetricsF(font, this)`（**带 QPaintDevice** 的 overload，遵循 `specs/横切关注点/40-高DPI适配.md` INV-DPI-METRICS）
- 字号按层级区分：level==1 用 14px，level>=2 用 13px（与 `TocPanel.cpp:104` 现有渲染保持一致，否则测宽与显示不符）
- 结果以整数像素返回（`ceil`），避免 Qt 布局取整时少 1px 导致省略号

## 验收（T 条目）

- **T1 极短标题窄化**：加载一个只含 `# A`、`## B` 的文档，TOC 宽度收缩到 ≈ `min_width`（120px），不残留 220px 空白
- **T2 极长标题不截断**：加载含 `# 很长的中文标题再加一堆英文 AbcdefghijklmnopqrstuvwXYZ` 的文档，
  TOC 宽度扩张到能完整显示该标题（无省略号），但不超过屏幕 1/5
- **T3 窗口 resize 夹紧**：将主窗口从 1920 宽度缩到 960，TOC 宽度自动从 384 收到 192（960/5），不吞掉编辑区
- **T4 切换 tab 重算**：tab A（短标题）→ tab B（长标题），切换后 TOC 宽度按 B 的标题重算
- **T5 高 DPI 正确**：200% 缩放屏幕下，最长标题文字完整显示不裁剪（验证 `QFontMetricsF` 带 `QPaintDevice` 的 overload 生效）

## 风险

- **与用户拖拽 splitter 的交互**：用户手动拖宽后，新开文档若无条件覆盖用户意图会反感。
  **方案**：记录一个 `m_userDraggedToc` flag（监听 `QSplitter::splitterMoved`），一旦为 true 则
  `preferredWidthChanged` 只更新 `minimumWidth` 不强制 `setSizes`；窗口关闭时清 flag（或提供菜单项「重置 TOC 宽度」）
- **高 DPI fontMetrics**：必须 `QFontMetricsF fm(font, m_scrollArea->widget())`（带 QPaintDevice），
  否则多屏不同 DPI 切换时测宽偏差——这是已知踩坑点（INV-DPI-METRICS）
- **首次加载时序**：`TocPanel` 构造时 AST 尚未就绪，`m_entries` 为空；必须等 `PreviewWidget::tocEntriesChanged`
  信号触发（`MainWindow.cpp:611-615`）后再算宽度，不能在 `showEvent` 中提前计算
- **QSplitter 状态持久化冲突**：`settings` 中存有 `window/mainSplitter`（`MainWindow.cpp:1385`），
  如果恢复的状态里 TOC 宽度 > 屏幕 1/5（比如多屏切到低分屏），需要在 `showEvent`
  restore 后立刻做一次夹紧检查
- **标题字体可能被主题切换改变**：`setTheme` 目前未改字体但未来可能改；`preferredWidth` 计算必须使用
  和 `buildList` 完全相同的 font，最好抽一个 `entryFont(level)` 辅助方法共用
