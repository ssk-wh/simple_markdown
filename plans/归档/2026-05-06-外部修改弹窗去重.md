# 外部修改弹窗去重：最多只弹一个 reload 提示

## 背景
用户反馈：当文档被另一个程序频繁修改时，切换到该文档的 Tab 后会**连续弹出多个**「文件已被另一个程序修改，是否重新加载？」对话框。期望最多只弹一个。

**初步根因**（已读代码确认，待修复时再核实）：

- `MainWindow::promptReloadTab(tabIndex)` 内部**无条件**弹 `QMessageBox::question`，没有"当前 Tab 是否已有 reload 对话框打开"的判定
- 触发路径有两条会叠加：
  1. **当前 Tab + 频繁修改**：`onFileChangedExternally` 中"如果是当前 Tab，立即调 promptReloadTab"——弹窗未关闭时再次触发就会叠一个新窗口
  2. **切换 Tab 时**：tab 切换 handler 用 `QTimer::singleShot(0, ..., promptReloadTab)` 排队，多次切换 / 多次外部修改可能让多个 QTimer 同时进入队列，逐个弹

仅记录待办，本次不修复。

## 动作（修复时）

- [ ] 在 `TabData` 中新增 `bool reloadDialogShowing = false`（或用 weak QPointer 跟踪当前模态对话框）
- [ ] 修改 `MainWindow::promptReloadTab(int)`：进入时若 `reloadDialogShowing == true` 直接 return；展示 QMessageBox 前置 true，关闭后置 false
- [ ] 修改 `MainWindow::onFileChangedExternally`：若 `pendingReload` 已经是 true 或对话框正在显示，则不再重新调度，仅刷新最新文件状态即可
- [ ] 切换 Tab 的 handler 在 schedule QTimer 前先检查是否已有"待弹"调度（可通过 `pendingReload` + 一个 "scheduled" 标志去重），避免多个 QTimer 排队
- [ ] 文件被删除的"File Deleted"提示走另一个路径，本次不动；但需保证两类弹窗不会同时出现（删除对话框开启时压制 reload）
- [ ] 在 Spec `模块-app/03-文件操作与拖拽.md`（或合适位置）补充一条不变量：**任意 Tab 在任意时刻最多只允许一个外部修改/删除提示对话框；文件继续修改不再叠加新对话框**
- [ ] 测试用例：通过 QFileSystemWatcher 模拟连续多次 fileChanged，断言对话框计数 ≤ 1
- [ ] 更新 CHANGELOG（用户视角措辞，例：被其他程序频繁修改的文档不再连续弹出多个"是否重新加载"提示，最多只弹一个）

## 验收
- 通过 PowerShell / `echo > foo.md` 在循环里连续 ≥10 次修改一个已打开但非当前的 Tab 文档，切换到该 Tab 后**只看到一个**对话框
- 在已是当前 Tab 的情况下连续修改 ≥10 次，从开始到结束**只看到一个**对话框；用户响应"是"或"否"后，新一轮修改才能弹下一个
- 用户响应"是"重新加载后，剩余的待重载请求不再继续弹（已是最新内容）
- 用户响应"否"忽略后，下一次外部修改可以再弹（用户重新决策）

## 备注
- 这是 UX 抗烦扰类改动，不涉及数据丢失，但属于用户每次都能感知的"软件不专业"信号
- 若实现时发现根因不止"对话框无去重"（例如 watcher 在某些 OS 上对单次保存触发多次 fileChanged），应在 Spec §已知陷阱中记录
