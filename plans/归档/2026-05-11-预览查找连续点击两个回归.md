# 预览查找：连续点击下的两个回归

## 背景

1.1.2 归档当天用户复测发现两个新 bug：

- **Bug A**：在预览搜索框出搜索结果后**连续多点**"下一个"图标按钮，
  搜索框消失（上一轮已为每条返回路径加了 keepFocus，仍然复现——说明在
  **多次点击的窗口期**内 FocusOut 自动隐藏被触发）
- **Bug B**：搜索结果 7/7 时点"上一个"→ 显示 6/7（正确）→ **再次**点"上一个" →
  **回到 7/7**（错误，期望 5/7）

两者都发生在上一轮 1.1.2 修复之后，是新引入的副作用 / 边界 case。

## 根因

### Bug A：SearchBar FocusOut 自动隐藏 timer 堆叠

`src/editor/SearchBar.cpp:143-148` 实现：

```cpp
if (event->type() == QEvent::FocusOut) {
    QTimer::singleShot(100, this, [this]() {
        if (!m_findEdit->hasFocus() && !m_replaceEdit->hasFocus() && isVisible())
            hideBar();
    });
}
```

每次 m_findEdit 失焦都**启动一个新的 100ms timer**。连续点击按钮（mousePressEvent
让 m_findEdit 失焦 → FocusOut → 启动 timer）→ N 个 timer 排队。

调用序：
1. 用户点击按钮（press）：m_findEdit 失焦 → timer T1 启动（100ms 后到期）
2. emit findNext → PreviewWidget::findNextHit → 最末 keepFocus 让 m_findEdit 重获焦点
3. T1 到期：m_findEdit->hasFocus()=true → 不 hideBar ✓
4. 用户**短时间内**再次点击按钮（press）：m_findEdit 再次失焦 → timer T2 启动
5. **若用户连续点击节奏密**：上一次的 keepFocus 已完成但下一次按钮 press 又让
   m_findEdit 失焦——T1 的检查时机可能正好处于"press → keepFocus"窗口期内
   （press 与 emit/keepFocus 是不同事件循环周期）→ T1 检查时 hasFocus=false → hideBar 触发

编辑器侧同样隐患，只是用户未复现。

### Bug B：反向跳转用 scrollY 找 hit.y < scrollY 在密集 hit 场景下回绕末项

`PreviewWidget::pickHitIndexByScroll(forward=false)`（上一轮新增）：

```cpp
for (int i = m_searchHits.size() - 1; i >= 0; --i) {
    if (hitY[i] >= 0 && hitY[i] < scrollY) return i;
}
return m_searchHits.size() - 1;  // 回绕末项
```

`scrollToCharOffset` 把命中滚到 scrollY = hit.y - vpH/3 处，所以**当前** hit.y 比
scrollY 大 vpH/3。在密集场景（所有命中集中在文档某段）：

- 7/7 时 currentIndex=6，scrollY = hit[6].y − vpH/3
- 点 prev → 找 hit.y < scrollY → 通常 hit[5].y < scrollY 满足 → 返回 5（6/7 ✓）
- 再点 prev：currentIndex 已是 5，scrollY 重设为 hit[5].y − vpH/3
- 找 hit.y < scrollY 的 hit：
  - hit[6].y > scrollY，跳过
  - hit[5].y = scrollY + vpH/3 > scrollY，跳过！
  - 如果 hit[4..0] 都在文档某区域且 y 都 > scrollY（密集场景）→ 全跳过 → 回绕末项 6 → 7/7 ❌

**位置语义不适合"连续跳转"**——连续跳转应基于"我刚选了哪一项"，而非"我看到哪里"。
编辑器侧的语义本质：每次 findNext 把 cursor 移到新命中，下次 findNext 基于**新** cursor
找下一个——等价于索引 +1（cursor 单调推进，下次找的就是下一个）。

## 设计

### Bug A 修复：SearchBar 单实例 timer

`SearchBar.h` 加一个成员：

```cpp
QTimer* m_focusOutTimer = nullptr;  // 单实例化避免多次 FocusOut 堆叠 timer
```

`SearchBar` ctor 初始化：

```cpp
m_focusOutTimer = new QTimer(this);
m_focusOutTimer->setSingleShot(true);
m_focusOutTimer->setInterval(100);
connect(m_focusOutTimer, &QTimer::timeout, this, [this]() {
    if (!m_findEdit->hasFocus() && !m_replaceEdit->hasFocus() && isVisible())
        hideBar();
});
```

`eventFilter` 改为：

```cpp
if (event->type() == QEvent::FocusOut) {
    m_focusOutTimer->stop();   // 重置：之前排队的检查作废
    m_focusOutTimer->start();  // 100ms 后只做一次检查
}
```

`keepFocus()` 调用时主动取消 timer（防御性）：

```cpp
void SearchBar::keepFocus() {
    if (m_focusOutTimer) m_focusOutTimer->stop();
    m_findEdit->setFocus();
}
```

### Bug B 修复：连续跳转回归索引循环

`PreviewWidget::findNextHit / findPrevHit`：

```cpp
if (m_currentSearchIndex < 0) {
    // 首次（或 debounce 刚回填）：按 scrollY 选起点（保留 INV-12 的位置语义）
    m_currentSearchIndex = pickHitIndexByScroll(forward);
} else {
    // 后续连续跳转：索引循环（与编辑器 cursor 单调推进等价）
    if (forward)
        m_currentSearchIndex = (m_currentSearchIndex + 1) % m_searchHits.size();
    else
        m_currentSearchIndex = (m_currentSearchIndex - 1 + m_searchHits.size())
                               % m_searchHits.size();
}
```

同步更新 INV-12 描述，把"单次跳转都基于可视区域"修正为
"**首次**跳转基于可视区域，**后续连续跳转**按索引循环"。

## 风险

- SearchBar 改 timer 影响编辑器侧——编辑器历史未报告此 bug，但行为更稳健（timer 不堆叠），
  属于改进非退化
- 索引循环让"用户滚动后再次按下一个"不会自适应可视区域——但用户期望"我刚按了
  hit[k]，下一个就是 hit[k+1]"，比"我看到哪了就跳到那"更直觉，与编辑器一致
