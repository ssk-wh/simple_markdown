# build_on_win 自动跑单元测试，pack 间接受益

## 背景
用户反馈：
- `build_on_win.bat` 编译完成后**应当自动跑一遍单元测试**（确保编译产物可用）
- `pack_on_win.bat` 在打包前**应当再跑一遍单元测试**（避免打包出有测试失败的版本）

仅记录待办，本次不实现。

## 现状

```
pack_on_win.bat  →  build_on_win.bat release  →  cmake --build  →  windeployqt + Qt DLL mirror
```

`pack_on_win.bat` line 19 已经在打包流程开头调 `build_on_win.bat release`——
**只要 build 脚本末尾加 ctest，pack 路径自动获益**，不必两个脚本各自加。

## 动作（实现时）
- [ ] 在 `build_on_win.bat` 的"Verify output"步骤之前，加 ctest 调用：
  ```bat
  REM ---- Run unit tests ----
  if exist %BUILD_DIR%\CTestTestfile.cmake (
      echo [+] Running unit tests...
      pushd %BUILD_DIR%
      ctest -C %BUILD_TYPE% --output-on-failure
      set CTEST_EXIT=!errorlevel!
      popd
      if not "!CTEST_EXIT!"=="0" (
          echo [ERROR] Unit tests failed.
          exit /b 1
      )
  )
  ```
- [ ] 测试失败时 build 应当返回非 0，让 pack 拒绝打包（fail-fast）
- [ ] 选项控制：加 `--skip-tests` 参数让用户在迭代调试阶段跳过测试；CI / pack 路径默认跑
- [ ] 同步 `build_on_linux.sh`：linux 下也加 ctest 调用，保持平台一致
- [ ] perf 测试（PreviewRenderBenchmark 用 LABELS "perf" 标记）默认参与还是排除？
  - 推荐：build 自动测试**默认排除 perf**（`ctest -LE perf`），减少日常构建时长
  - pack 路径可选择是否跑 perf（性能门槛是否纳入发布质量门）
- [ ] 在项目 CLAUDE.md 中文档化此机制（与 INV-BUILD-QT-MIRROR 同章节）

## 验收
- 任意一个单元测试 fail → `build_on_win.bat release` 返回非 0
- `pack_on_win.bat` 因 build 失败而中断，不进入打包步骤
- 加 `--skip-tests` 时 build 跳过 ctest 直接生成产物（开发迭代场景）
- linux 上行为一致

## 注意 / 风险
- ctest 当前耗时约 5-7 秒（12 个测试），属于可接受的构建尾巴
- 如未来测试规模膨胀，再加 LABELS 分组（如 `unit` vs `integration` vs `perf`），让默认 build 只跑 unit
- 测试需要 Qt DLL 已在 build/tests 目录（已由 build_on_win.bat 中的 mirror 段保证），ctest 才能启动
- Linux 上 ctest 不需要 mirror DLL，但需要 `Display=:0` 或 offscreen Qt 平台插件——已有测试用 `QApplication` 全局 fixture 应当 OK，待实施时实测

## 实施结果（2026-05-06）

- `build_on_win.bat`：编译完成 + Qt DLL mirror 之后自动 `ctest -C %BUILD_TYPE% --output-on-failure -LE perf`
- `build_on_linux.sh`：同步加 `(cd build && ctest --output-on-failure -LE perf)`
- 默认排除 `LABELS "perf"` —— `PreviewRenderBenchmark` 不参与日常构建（11 个 unit test 全跑，约 5-7s）
- 加 `--skip-tests` / `--no-tests` 参数跳过（开发迭代）
- 测试 fail 时 build 返回非 0 → `pack_on_win.bat` 因依赖 build 自动中断，不进入打包步骤
- CLAUDE.md 增 `[INV-BUILD-RUN-TESTS]` 文档化该机制
- 不进 CHANGELOG（构建基础设施改动，用户感知不到）

实测：
- 默认路径：11/11 unit tests PASS
- `--skip-tests`：跳过 ctest，输出 `[!] Skipping unit tests (--skip-tests)`

## 备注
- 这是一种**质量门**机制：CI 流水线已经有，但本地构建/打包流程之前没有强制检测
- 与 plan「2026-04-13-单元测试覆盖已完成」是配套——前者建立测试，后者强制每次构建都跑
- 用户此前提的"编译后自动复制 qt dll 到编译目录"已实施（INV-BUILD-QT-MIRROR），本条是它的下一个自然步骤——
  跑测试需要 DLL 就位，DLL 已就位后跑测试是自然延伸
