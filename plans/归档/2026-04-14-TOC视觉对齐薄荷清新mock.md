# TOC 视觉对齐薄荷清新 mock

## 背景

用户原话：
> 右侧目录（TOC）面板的视觉实现参考 `tmp/theme_mocks/mock_e_mint_breeze.html` 里 TOC 面板的效果，
> 具体要对齐：hover 效果 / 行间距 / 字体大小 / 文章标题前的圆点，以及自行从 mock 里提炼的其它视觉细节
> （圆角、padding、当前项指示、分级缩进等）。

当前 `src/preview/TocPanel.cpp` 的实现方式是 QScrollArea + QVBoxLayout + 每个条目一个 `QPushButton`（flat），
靠 setStyleSheet 的 `QPushButton:hover` 伪类做 hover 效果，**没有圆点 bullet**，面板本身也没有 "card"
视觉（白底 + 圆角 + 阴影），与 mock_e 差距明显。

对比差异（至少 4 条）：

1. **无 bullet 圆点**：mock_e 每个条目左侧有一枚 6×6 的实心圆点（lvl1 #C5DECD / lvl2 5×5 #D8E8DE / active
   换成 #FFDC7A 柠檬黄并带 3px 光晕）。当前 TocPanel 完全没有 bullet，只有纯文本按钮。
2. **面板缺失 card 包裹**：mock_e 把 TOC 列表放在一张白底 card 里（`border-radius:12px`、`padding:12px 14px`、
   `box-shadow: 0 6px 14px -10px rgba(27,59,42,.18)`），上方还有一个 "OUTLINE" 大写小标题 + 小绿条装饰。
   当前 TocPanel 只有一个左边框 + 扁平 QLabel 标题（`font-size:14px font-weight:bold`）。
3. **hover 色彩策略不同**：mock_e hover 仅改背景 `#F1F9F3`（极淡绿灰），**字色不变**；当前实现 hover 时
   把字色也改成了 `accentColor`（主题主色），与 mock 相比显得过于醒目。
4. **行距 / 字号 / padding 比例偏紧**：mock_e lvl1 `font-size:12.5px + padding:6px 8px + gap:2px`，
   lvl2 `font-size:12px + padding-left:20px`；当前实现 lvl1=14px、lvl2=13px、`padding 5px 8px`、
   `spacing:1px`、`indent = (level-1)*14px`，条目字号明显偏大、lvl2 缩进偏小。
5. （补充）**active 指示形态不同**：mock_e active = 淡薄荷底 `#E8F7EC` + 深松针字 `#1F5A3C` + **柠檬黄 bullet**
   + 柔光；当前实现 active (`m_highlightedEntries`) 仅改 normal 背景为 `previewHighlightToc`，
   无 bullet 变色、无光晕、字色与权重也未变化。

## mock 视觉规格提取（来自 mock_e_mint_breeze.html · `.toc` 起）

```
── 面板容器 (.toc) ──────────────────────
├ background: theme.panel-soft   (#F6FBF7 in mock)
├ border-left: 1px solid border-soft
├ padding: 18px 14px
├ gap: 14px (多张 card 之间)
└ 宽度: 260px (mock 固定列宽)

── card 包裹 (.toc .card) ───────────────
├ background: theme.panel  (#FFFFFF)
├ border: 1px solid border-soft
├ border-radius: 12px
├ padding: 12px 14px
└ box-shadow: 0 6px 14px -10px rgba(27,59,42,.18)

── card 小标题 (h4 "OUTLINE") ────────────
├ margin: 0 0 8px
├ font-size: 11px
├ font-weight: 700
├ letter-spacing: .12em
├ text-transform: uppercase
├ color: theme.ink-sub
└ 前缀装饰条: 16px × 2px 薄荷 accent，圆角 2px

── 条目列表 (.toc .tree) ─────────────────
├ display: flex-column
└ gap: 2px  ← 条目之间

── 条目 (a) lvl1 默认 ────────────────────
├ display: flex + align-items:center + gap: 8px
├ font-size: 12.5px
├ line-height: 继承 body 1.75
├ color: theme.ink  (#1B3B2A)
├ padding: 6px 8px
├ border-radius: 7px
├ 无下划线
└ bullet:
    ├ shape: 实心圆 (border-radius:50%)
    ├ size: 6px × 6px
    ├ color: #C5DECD  (lvl1 默认 bullet — 淡薄荷灰)
    └ margin-right: 8px (靠 gap 实现)

── 条目 (a.lvl2) ─────────────────────────
├ padding-left: 20px  (总 padding: 6px 8px 6px 20px)
├ font-size: 12px     (比 lvl1 小 0.5px)
├ color: theme.ink-sub  (#5E7B6B — 次文字)
└ bullet:
    ├ size: 5px × 5px  (比 lvl1 小 1px)
    └ color: #D8E8DE  (比 lvl1 更淡)

── hover (a:hover, 非 active) ────────────
├ background: #F1F9F3  ← 极淡绿灰
├ color: 不变
├ bullet: 不变
├ transition: 无显式 transition（mock 是静态 CSS，Qt 下可用
│   QPropertyAnimation 160ms ease-in-out 近似；MVP 可直接无动画）
└ 圆角跟随条目 border-radius 7px

── active (a.active) ─────────────────────
├ background: theme.mint-soft  (#E8F7EC)
├ color: #1F5A3C  (深松针字)
├ font-weight: 600
└ bullet:
    ├ background: theme.lemon  (#FFDC7A 柠檬黄)
    └ box-shadow: 0 0 0 3px rgba(255,220,122,.28)  ← 柔光环

── 层级缩进 ────────────────────────────────
└ 只有两级在 mock 里呈现:
    ├ lvl1: padding-left 8px (card 内基线)
    └ lvl2: padding-left 20px  → 相当于 level*12 左右
  如果项目 TOC 支持 h3/h4，推荐阶梯 step = 12px：
    padding-left = 8 + (level-1) * 12
```

## 验收（T 条目，至少 4 条）

- **T-VISUAL-1**：以 96 DPI 显示，肉眼核对 TocPanel 的 h1/h2 字号与缩进、
  bullet 大小、card 圆角、hover 背景，与 `mock_e_mint_breeze.html` 的 `.toc`
  区域并排截图对比无明显差距（字号差 ≤0.5px、间距差 ≤2px）。
- **T-VISUAL-2**：鼠标移入任一非 active 条目，背景出现极淡绿灰（或对应主题 hoverBg），
  **字色不变**；鼠标移出恢复透明；连续在多个条目上滑动时无闪烁。
- **T-VISUAL-3**：当前预览滚动到某标题时，对应条目呈 active 状态——底色变亮、
  字色变深加粗、左侧 bullet 由灰变为 accentSecondary（浅主题下为柠檬黄），
  且 bullet 周围有 ≥2px 半透明光晕。
- **T-VISUAL-4**：切换 8 款内置主题（薄荷 / 极光 / 暮色 / 纸雾 / ...），
  TOC card 背景、hover 背景、active 背景、bullet 颜色、文字颜色全部随
  主题变化，无任一处残留硬编码浅绿/深灰；暗色主题下 bullet 光晕
  使用主题自身 accentSecondary 而非固定柠檬黄。
- **T-VISUAL-5**（可选）：高 DPI（175% 缩放）下字号、bullet 尺寸、padding
  均等比例放大，不出现 bullet 位置偏移、文字截断、圆角失真。

## 风险

- **控件方式变更深度**：当前 TocPanel 用 `QPushButton` + stylesheet，如果保留 QPushButton
  靠 stylesheet 画 bullet（用 `::before` 是不行的，Qt 不支持），需要么改成自绘 widget、
  要么用 `QPushButton::setIcon` 放一个预渲染 bullet QPixmap（跨主题要动态生成）。
  自绘方案更干净但改动更大，约 +80 行。
- **Qt stylesheet 不支持 box-shadow**：card 阴影需要用 `QGraphicsDropShadowEffect`，
  或直接在 TocPanel::paintEvent 里绘制一条伪阴影（4-6px 模糊渐变）。
- **transition 动画**：mock 本身是静态 CSS 没 transition 值，Qt 下用
  QPropertyAnimation 对背景色做 120-160ms 缓动；风险是频繁鼠标划过时
  叠加动画导致卡顿，MVP 建议先不做动画。
- **与其它 TOC plan 的冲突面**：
    · `TOC支持折叠.md` 要给每个父节点加 ▸/▾ 图标 → bullet 位置会被折叠箭头占用，
      视觉对齐必须等折叠结构确定后再定 bullet 的 x 基准。
    · `TOC宽度自适应.md` 会影响 card 宽度、padding 是否需要自适应。
    · `TOC垂直对齐编辑预览.md` 可能调整 TocPanel 顶部空余 → card 外边距要让位。
    · `TOC下方文档信息卡.md` 已经是"第二张 card" 样式 → 本 plan 的 card
      外观必须与它保持一致（gap 14px、同圆角、同阴影）。
- **新增 Theme 字段**：若要新增 `tocHoverBg/tocActiveFg/tocBulletColor/tocBulletActive`
  需要同步修改 8 款主题 TOML，否则旧主题加载会缺字段→默认值兜底可能视觉偏差。
  建议先用 fallback（如 `tocHoverBg.isValid() ? tocHoverBg : editorCurrentLine`）
  避免强制破坏旧主题。
