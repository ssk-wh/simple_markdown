# 预览区表格选区拖到右侧空白扩散到上下行

## 背景

用户反馈：在渲染区（PreviewWidget）选中表格内某一行的内容后，按住鼠标继续向**该行右侧的空白区域**拖动，
原本只应该停留在该行的选区会"扩散"——附近**上下若干行**的内容也会被一并加入选区高亮。

预期行为：
- 鼠标在表格右侧空白区域时，选区终点应当 snap 到**当前 y 所在那一行**的行末（或该单元格末尾），
  不应跳到下一行起点 / 上一行末尾。
- 表格场景下，"行末右侧空白"是常见的鼠标终点（用户想"选到行末"），不能让该交互产生跨行误选。

可能涉及的代码反模式（参见 CLAUDE.md「反复踩坑模式」）：
- **反模式 C**：`nearestTextIndex` / 行内 segment 命中函数语义模糊——空白处返回的"最近字符"
  跨越了表格单元格 / 行边界，被选区拖拽当作严格命中使用。
- 表格单元格的 hit-test 区域 y 范围可能与渲染高度不一致，鼠标越过单元格垂直边界时被
  归到上/下一行。

## 复现路径（待实测确认）

1. 打开任意含表格的 Markdown 文档（建议表格 ≥ 3 行）。
2. 在表格中间某一行内按下鼠标左键开始选择。
3. 保持按住，鼠标向**右**移动越过表格右边界进入右侧空白区。
4. 期望：选区高亮仅在当前行；实测：上下相邻几行也被加入高亮。
