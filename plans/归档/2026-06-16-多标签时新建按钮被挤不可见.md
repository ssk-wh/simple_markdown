# 多标签时「新建」按钮被挤出不可见

## 背景

用户报告（2026-06-16）：

> 打开的文档较多之后，tab 上面的「新建」按钮怎么都看不到，非要关闭一些文档才行。

Tab 数量多到撑满 Tab 栏宽度后，跟在 Tab 末尾的「+」新建按钮被挤出可视区域，
用户无法新建文档，只能先关闭部分 Tab。

## 根因方向（待验证）

- TabBarWithAdd 的「+」按钮定位在最后一个 Tab 右侧，Tab 总宽超出栏宽时按钮跟着溢出。
- 需确认 Tab 栏是否启用了滚动/省略策略；「+」按钮应固定可见（如钉在 Tab 栏右端，或
  Tab 区可滚动而「+」常驻）。

## 目标（验收条件）

- [x] 无论打开多少 Tab，「新建」按钮始终可见可点
- [x] Tab 过多时 Tab 区可滚动 / 省略，但不挤掉新建按钮
- [x] 「新建」按钮风格贴合 Tab 栏其他控件（自绘细十字 + hover 半透明圆角底）

## 设计

根因：早期「+」自绘在「最后一个 Tab 右侧」（`addButtonRect = lastTab.right() + gap`）。
Tab 总宽溢出 Tab 栏宽度时，`QTabBar` 启用原生横向滚动，最后一个 Tab 被滚出可视区，
「+」按钮随之被推走 → 用户看不到、只能先关 Tab。

方案（INV-6）：`TabBarWithAdd` 从「`QTabBar` 子类 + 自绘按钮」改为「`QWidget` 容器 =
`QTabBar`（stretch=1，保留原生滚动）+ `QToolButton`（固定宽，钉右端）」。按钮成为 Tab 区的
固定兄弟控件，与滚动解耦。容器对外暴露 `bar()`，宿主沿用全部 `QTabBar` API；可见性切换
（隐藏顶部栏 / 演示模式 / 侧边模式共 5 处）改为作用于容器，避免按钮孤立残留。

按钮风格：用 `QToolButton` 子类自绘细十字线（1.4px RoundCap）+ hover 半透明圆角底，
复刻早期 Chrome/Edge 风格，贴合 Tab 栏其他控件（用户要求）。

## 验证

- 新增 `tests/app/TabBarAddButtonVisibilityTest.cpp`（T-11 / T-12），含「tab 确实溢出」前置
  断言，确保覆盖真实 bug 场景；断言溢出时按钮可见 + 不与 Tab 区重叠 + 在容器内。
- `build_on_win.bat release` 默认路径 ctest 21/21 全过（含新测试）。
- 视觉验证（`tmp/visual_check.ps1` 开 16 个 tab 后截图）：Tab 溢出后原生滚动箭头与「+」
  按钮并存，按钮固定在最右端清晰可见，自绘十字风格与 Tab 栏一致。
