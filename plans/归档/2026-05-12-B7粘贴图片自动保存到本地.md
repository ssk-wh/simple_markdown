# B7 粘贴图片到编辑器自动保存到 ./images/

## 背景

截图笔记工作流必备：截图（Win+Shift+S / macOS Cmd+Shift+4）→ Ctrl+V → 编辑器
自动把剪贴板图片保存到本地 + 插入 `![](images/...)` 语法。

当前粘贴图片估计是无响应或被忽略。Typora / Obsidian / VS Code 都支持。

## 动作

- [ ] Step 1: EditorWidget 拦截 paste 事件，判定 QClipboard::mimeData 含 `image/*`
- [ ] Step 2: 决定保存路径：
  - 当前文档已保存 → 保存到 `<文档目录>/images/` 下
  - 当前文档未保存 → fallback 到用户 Pictures 目录 或 弹窗让用户选目录
- [ ] Step 3: 文件命名：`<文档名>-<YYYYMMDD-HHMMSS>.png`，重名追加 `-1` `-2`
- [ ] Step 4: 写入完成后在光标位置插入 `![](images/<filename>.png)` 相对路径
- [ ] Step 5: 失败兜底（写入失败 / 用户取消选目录）：fallback 到 base64 内嵌 或 提示错误
- [ ] Step 6: 提供 `编辑 → 图片粘贴行为` 设置：自动保存 / 总是 base64 / 总是询问

## 验收

- 截图后 Ctrl+V → 文档同级 `images/` 目录出现 PNG 文件 + 光标处插入 MD 语法
- 同秒连续粘贴两张图 → 文件名带 `-1` 后缀不冲突
- 未保存文档 + 粘贴图片 → 弹出"请先保存文档"或自动选用户目录（按设置决定）
- 撤销（Ctrl+Z）一步还原插入的 MD 语法（注意：物理文件是否同步删除需 plan 内决定，建议**不删**避免误操作）

## 风险

- 自动写文件到当前目录有"看不见的副作用"——按 CLAUDE.md 默认值偏向"用户能看见"，
  应在 statusbar 显示 toast 提示"图片已保存到 ./images/foo.png"
- 同名冲突策略要明确（追加后缀 vs 覆盖），建议总是追加，永不覆盖
