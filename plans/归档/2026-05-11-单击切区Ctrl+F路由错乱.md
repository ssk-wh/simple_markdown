# 单击切区后 Ctrl+F 仍路由到旧区（双击就正常）

## 背景

用户原话：

> 编辑区 Ctrl+F 唤出搜索框之后，点击一下预览区，Ctrl+F 唤出的搜索框还在编辑区，
> 同样在预览区唤出也是一样的问题，但是我双击选中文本之后再唤出就正常了

两个反向：编辑→单击预览→Ctrl+F 仍弹编辑；预览→单击编辑→Ctrl+F 仍弹预览。
双击切区却正常——掩盖了焦点未真正转移的事实。

## 根因（双因素）

### 因素 A：PreviewWidget mousePressEvent 缺 setFocus

QAbstractScrollArea 把鼠标事件分发到 viewport（默认 NoFocus）→ viewport 收到
mousePressEvent → 通过 viewportEvent 转给 PreviewWidget::mousePressEvent。
**Qt 不会自动让 PreviewWidget 获焦**——必须在 mousePressEvent 中显式 `setFocus()`。

- `EditorWidget::mousePressEvent` line 401 ✓ 已有 `setFocus()`
- `PreviewWidget::mousePressEvent` line 436 ✗ 缺失

反模式 B：两侧 QAbstractScrollArea 子类的焦点处理本应一致，编辑器侧做了预览侧
漏了。

### 因素 B：SearchBar::hideBar 无条件 setFocus(parent) 抢回焦点

旧版 `hideBar` 末尾 `parentWidget()->setFocus()`——意图是 Esc 关闭后焦点回到宿主。
但 INV-15"点击外部关闭"场景下：

时序（预览搜索栏 visible，用户单击编辑器）：
1. 用户点击编辑器 viewport
2. 全局 filter（preview SearchBar 装）收到 MouseButtonPress → 点击不在 preview
   SearchBar 矩形内 → `singleShot(0, hideBar)` 排队
3. 事件传到 EditorWidget::mousePressEvent → `setFocus()`（编辑器获焦）✓
4. 用户按 Ctrl+F → 但 singleShot 还没跑，**焦点确实在 editor**
5. MainWindow 路由：focusInEditor=true → openEditor ✓ **本应工作**

但实际反向问题表明步骤 5 没成立——可能因为 singleShot(0) 在 keyPress 之前先跑：
1-2 同上
3'. singleShot(0) 在事件循环 tick 时触发 hideBar → `parentWidget()->setFocus()` 把
    焦点抢回 **preview**（hideBar 的 parent 是 PreviewWidget）
3''. 之后或同时事件传到 EditorWidget::mousePressEvent → setFocus(editor)
4. 用户按 Ctrl+F → 焦点最终在 editor 还是 preview 取决于 3'/3'' 谁先

Qt 事件循环顺序复杂——但**只要 hideBar 在 EditorWidget setFocus 之后跑**，焦点会
被抢回 preview，路由判 focusInPreview=true → 仍弹 preview 搜索栏。

## 修复

### A：PreviewWidget mousePressEvent 加 setFocus

```cpp
void PreviewWidget::mousePressEvent(QMouseEvent* event) {
    if (event->button() == Qt::LeftButton) {
        setFocus();   // ← 新增，对齐 EditorWidget line 401
        // ... 原 DPI 检查 / 选区 / 链接处理 ...
    }
}
```

### B：SearchBar::hideBar 条件 setFocus

```cpp
void SearchBar::hideBar() {
    qApp->removeEventFilter(this);
    // 关键：判定时机在 hide() 之前——hide 后 m_findEdit 必然失焦
    const bool wasFocusInside =
        (m_findEdit && m_findEdit->hasFocus()) ||
        (m_replaceEdit && m_replaceEdit->hasFocus());
    hide();
    emit closed();
    if (wasFocusInside) parentWidget()->setFocus();  // ← 仅 Esc/关闭按钮路径
}
```

Esc / 关闭按钮路径：焦点在 m_findEdit → setFocus(parent) ✓
INV-15 点击外部路径：焦点已在用户新点击的 widget → 不抢回 ✓
