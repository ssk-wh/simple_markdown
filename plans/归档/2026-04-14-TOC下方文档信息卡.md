# TOC 面板下方文档信息卡

## 背景

左侧 TOC 面板目前只显示目录，下方有大片空区。用户希望在 TOC 下方看到当前文档的元信息一览（字数、创建/修改时间、文件大小、frontmatter 标题与 tags），避免另外打开"文档统计"弹窗。与已归档的"文档统计弹窗"互补：**弹窗是按需详细视图，这里是常驻摘要**。

## 动作

- [ ] `TocPanel` 底部新增 `DocInfoCard` 小部件（可折叠：点击标题行切换展开/收起）
- [ ] 展示字段（折叠时只显示字数 + 修改时间；展开时全部）：
    - 字数 / 字符数 / 段落数
    - 标题数（h1/h2/h3 分别计）
    - 创建时间（`QFileInfo::birthTime`，无则降级为修改时间）
    - 修改时间（相对时间 "3 分钟前"）
    - 文件大小（KB / MB 自适应）
    - frontmatter 里的 `title` 和 `tags`（若有，tags 渲染为小徽章）
- [ ] 数据来源：复用现有 AST 与 Document 元数据，**不重复解析**；frontmatter 已由预处理剥离可直接读取
- [ ] 视觉走 Theme：背景色、边框、徽章色均用 Theme 字段；与 TOC 共用圆角/间距
- [ ] 折叠状态持久化到会话（与 Tab 共享）
- [ ] 无文件（未命名 Tab）时显示"尚未保存"占位，字数等仍正常显示
- [ ] Spec 更新：`specs/模块-preview/08-TOC面板.md` 新增 INV（信息卡不重复解析 / 折叠持久化）+ T 条目

## 验收

- 打开一个带 frontmatter 的文件，信息卡展开后显示全部字段且 tags 为小徽章
- 编辑文档后字数 / 段落数 / 标题数实时更新（节流到 ~200ms）
- 切换主题后信息卡配色同步
- 折叠信息卡后重开应用保持折叠

## 风险

- 与"文档统计弹窗"字段重复，需约定：弹窗展示详尽统计（代码块数 / 图片数 / 链接数），信息卡只展示最常看的 7-8 项
- TOC 深度嵌套+折叠（见 `2026-04-14-TOC支持折叠.md`）时要保证信息卡仍可见，必要时 TOC 与卡片间加 splitter
