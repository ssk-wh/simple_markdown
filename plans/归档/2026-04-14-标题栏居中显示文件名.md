# 自定义标题栏（macOS 风格居中标题 + 完整窗口控制 + 主题化）

> 本 plan 原本只聚焦「标题居中」，现扩展为完整的自定义标题栏需求。
> 涵盖标题居中、最大/最小/关闭按钮、双击还原、拖拽、以及标题栏背景跟随主题。

---

## 1. 背景

当前 Windows 下的 `QMainWindow` 使用系统原生标题栏（OS 绘制、OS 控制），
存在 **3 个用户可感知的短板**：

1. **标题对齐**：Windows 原生标题默认左对齐（"fm_test.md - SimpleMarkdown"），
   而 macOS / iOS 26 Liquid Glass mockup（`tmp/theme_mocks/mock_*.html`）的目标视觉
   是标题**居中**、三按钮在左或右。
2. **主题化**：标题栏背景由 Windows DWM 控制，**永远无法跟随 Theme**。
   当前已落地 3 款 Liquid Glass 主题（Arctic Frost / Sunset Haze / Midnight Aurora），
   深色主题下顶部一条灰白系统标题栏，是「全局视觉统一」的**最后一块短板**。
3. **窗口控制交互**：希望在自绘标题栏上保留 macOS / Windows 的标准交互：
   最小化 / 最大化 / 关闭、双击标题条切换最大化/还原、按住标题条拖动窗口。

## 2. 技术路径对比

### 路径 A：Frameless + 完全自绘标题栏

- `setWindowFlags(Qt::FramelessWindowHint)`，自写 `QWidget` 作为标题栏，嵌入到 `QMainWindow` 顶部
  （`setMenuWidget()` 或自己 `setCentralWidget` 外套 `QVBoxLayout`）。
- 优点：
  - **100% 可主题化**：背景 = `theme.editorGutterBg`，所有颜色过 ThemeManager。
  - 完全自由地居中标题、自定义按钮样式（macOS 三圆点 / Windows 方按钮均可）。
  - 跨平台行为可统一（都走 Qt 层），Linux / Windows 代码路径复用度高。
- 缺点（**所有复杂性都来自这些要自己补**）：
  - `WM_NCHITTEST`：Windows 调整窗口大小的 hit-test，`FramelessWindowHint` 默认会丢失「边缘拖拽改变大小」。
  - `WM_NCCALCSIZE`：Windows 11 DWM 阴影和圆角依赖标准非客户区计算，frameless 会丢掉阴影。
  - 双击最大化/还原逻辑（自己写 `mouseDoubleClickEvent`）。
  - **Snap Layout**：Win11 鼠标悬停最大化按钮弹出的 4-grid 布局预览，需要 `HTMAXBUTTON` 命中返回。
  - **系统右键菜单**（Restore / Move / Size / Minimize / Maximize / Close）：需自己唤起 `WM_SYSMENU`。
  - **Aero Shake**、**Alt+Space** 默认路径会走丢，需要手动激活。
  - **任务栏缩略图 / DWM 预览**：Frameless 窗口仍然正常（DWM 复合），但若去除 NCCALCSIZE 可能异常。

### 路径 B：保留系统标题栏 + DWM API 调色（仅 Win11 22H2+）

- 用 `DwmSetWindowAttribute(hwnd, DWMWA_CAPTION_COLOR, &color, sizeof(color))` 设置标题栏背景色，
  `DWMWA_TEXT_COLOR` 设置标题文字色，`DWMWA_BORDER_COLOR` 设置边框色。
- 优点：
  - 代码极少（一个 Windows API 调用即可，加 DPI / 主题切换的信号槽），1 天可以落地。
  - **所有系统级功能保留**：拖拽、调整大小、Snap、Aero、右键菜单、任务栏预览全部不动。
- 缺点：
  - **只支持 Windows 11 22H2+**（构建号 22621），Win10 不可用。
  - **标题文字无法居中**，按钮样式无法自定义（仍是系统原生方按钮）。
  - **Linux 不适用**，X11/Wayland 走另一套装饰协议。
  - 不能实现 macOS 风格的三圆点按钮。

### 路径 C：混合（frameless + `QPlatformWindow` hack）

- 例如 [KDDockWidgets](https://github.com/KDAB/KDDockWidgets) / [QGoodWindow](https://github.com/antonypro/QGoodWindow)
  等第三方库的混合方案：保留系统非客户区行为（拖拽 / 调整 / Snap 全部交还给 OS），
  只覆盖标题栏的绘制。
- 优点：系统功能全保留，视觉可定制。
- 缺点：
  - 平台差异大，每平台实现一套（X11 / Wayland / macOS / Windows）。
  - 需引入第三方 C++ 库（或自写等价 hack）；Qt 主版本升级时易失效。
  - 跨平台测试矩阵成本高，Win10 / Win11 行为不同。

### 推荐：路径 A（Win10 及以上全覆盖），备选路径 B（轻量级）

- 主线按路径 A 实施，接受 **4 周工程量**。
- 关键要点：重写 `nativeEvent` 拦截 `WM_NCHITTEST` / `WM_NCCALCSIZE`，手动支持
  Snap Layout 和系统右键菜单。
- 若时间紧张或觉得工程量不值，**立刻回退到路径 B**（见第 9 节「替代方案」）。
- 测试矩阵：Win10 / Win11 / 高 DPI（125% / 150% / 200%）/ 多显示器 / Linux（X11 + Wayland）。

## 3. Qt Frameless 实现要点（路径 A 细节）

### 3.1 窗口 flags

```cpp
setWindowFlags(Qt::Window
             | Qt::FramelessWindowHint
             | Qt::WindowMinimizeButtonHint
             | Qt::WindowMaximizeButtonHint
             | Qt::WindowCloseButtonHint
             | Qt::WindowSystemMenuHint);
```

保留后三个 hint 是为了任务栏缩略图和 Alt+Tab 时仍然显示系统按钮图标（Qt 内部使用）。

### 3.2 `nativeEvent` 拦截（Windows）

覆写 `bool MainWindow::nativeEvent(const QByteArray& eventType, void* message, qintptr* result)`：

- **`WM_NCHITTEST`**：把鼠标屏幕坐标转本地坐标后分区域返回：
  - 四角 8px 热区 → `HTTOPLEFT / HTTOPRIGHT / HTBOTTOMLEFT / HTBOTTOMRIGHT`
  - 四边 5px 热区 → `HTTOP / HTBOTTOM / HTLEFT / HTRIGHT`
  - 标题栏区域（减去按钮区）→ `HTCAPTION`（让 OS 处理拖拽和双击）
  - 最大化按钮区域 → `HTMAXBUTTON`（Win11 22H2+ 触发 Snap Layout 弹窗）
  - 其他 → `HTCLIENT`（不拦截，交给 Qt 事件）
- **`WM_NCCALCSIZE`**：当 `wParam == TRUE` 时 `*result = 0; return true;`
  告诉 DWM 「整个窗口就是客户区」。
- **`WM_GETMINMAXINFO`**：最大化时修正 `MINMAXINFO`，避免覆盖任务栏
  （Windows 原生 frameless 最大化会遮挡任务栏）。
- **`WM_SYSCOMMAND`**：仅转发，保证系统右键菜单 → Move/Size 正常。
- **`WM_NCRBUTTONUP`**：标题区右键 → 显示系统菜单（`TrackPopupMenu(GetSystemMenu(hwnd, FALSE), ...)`）。

### 3.3 Windows 11 圆角与阴影

```cpp
#ifdef _WIN32
// Win11 圆角
auto pref = DWMWCP_ROUND;
DwmSetWindowAttribute(hwnd, DWMWA_WINDOW_CORNER_PREFERENCE, &pref, sizeof(pref));

// Frameless 要手动恢复 DWM 阴影
MARGINS m = {1, 1, 1, 1};
DwmExtendFrameIntoClientArea(hwnd, &m);
#endif
```

### 3.4 Linux 分支（后置 Phase 3）

- **X11**：用 `_NET_WM_MOVERESIZE` ClientMessage 让 WM 代理拖拽/调整大小。
- **Wayland**：启用 `xdg-decoration` 协议，由 compositor 告知是否用 client-side decoration；
  若 compositor 不支持 CSD 协商（KDE / GNOME 都支持），退化成 server-side。
- Qt 6 的 `QWindow::startSystemMove()` / `startSystemResize()` 封装了两者，优先用这对 API。

## 4. 自绘标题栏组件设计

### 4.1 文件与类

- 新增 `src/app/CustomTitleBar.h` / `CustomTitleBar.cpp`
  - 类：`class CustomTitleBar : public QWidget`
  - 头部注释引用 Spec：`Spec: specs/模块-app/04-窗口焦点管理.md` + `specs/模块-app/12-主题插件系统.md`
- 改动：
  - `MainWindow::MainWindow()`：`setWindowFlags(...)` + `setMenuWidget(new CustomTitleBar(this))`
    或自己组合容器（菜单栏 + 标题栏 + 内容区）。
  - `MainWindow::updateTabTitle()` / `setWindowTitle()`：同步给 `CustomTitleBar::setTitle()`。

### 4.2 布局结构（3 段式）

```
┌──────────────────────────────────────────────────────────────┐
│ [icon] ...          [文件名 - SimpleMarkdown]         - □ ×  │   ← h = 32px (HiDPI 随 fm.height() 缩放)
└──────────────────────────────────────────────────────────────┘
```

- **左段**：app icon + spacer（或 macOS 模式下三圆点）。宽度对称占位。
- **中段**：`QLabel` 显示 `windowTitle()`，`Qt::AlignCenter`，自动省略（`Qt::ElideMiddle`）。
- **右段**：3 个 `QToolButton`：Minimize / Maximize↔Restore / Close。
- 高度：28–32 px（macOS 惯例 28，Windows 惯例 32），通过 `fontMetrics().height() * 1.8` 动态算，
  遵守 **INV-DPI-NO-HARDCODE**。

### 4.3 主题配色接入

- 继承 `ThemeManager` 已有信号 `themeChanged(const Theme&)`，在 `CustomTitleBar` 的槽里更新：
  - 背景色：`theme.editorGutterBg`
  - 文字色：`theme.editorFg`
  - 按钮 hover：`theme.editorCurrentLine`
  - 关闭按钮 hover：硬编码 `#E81123`（Windows 系统惯例）或 `theme.errorRed`（若存在）。
- 失去焦点（`changeEvent(QEvent::ActivationChange)`）时，文字颜色降 40% alpha，模拟系统行为。

### 4.4 鼠标交互

- `mousePressEvent` + 拖拽：用 `window()->windowHandle()->startSystemMove()`（Qt 5.15+）。
  这是**跨平台**的首选 API，替代自己监听鼠标移动算偏移量。
- `mouseDoubleClickEvent`：在标题区域触发 `toggleMaximize()`。
- 按钮点击：
  - 最小化 → `window()->showMinimized()`
  - 最大化/还原 → 根据 `isMaximized()` 切换 `showMaximized() / showNormal()`
    + 同步按钮图标（□ ↔ ❐）
  - 关闭 → `window()->close()`（走 `closeEvent` 保存 geometry、询问保存修改）

## 7. 风险（至少 6 条）

1. **Qt 5 QMenuBar × FramelessWindowHint 冲突**：Alt 键激活菜单的助记符路径在某些 Qt 版本下丢失，
   需验证 Qt 5.15 的 Alt+F/Alt+E 是否仍工作，如有问题改手动 focus 切换。
2. **Snap Layout 依赖 Win11 22H2+（构建号 22621）**：Win10 / Win11 21H2 下鼠标悬停不会弹窗，
   需要检测版本优雅降级（只保留点击最大化，不返回 HTMAXBUTTON 可能破坏 Win10 行为）。
3. **Linux frameless 行为分裂**：X11 走 `_NET_WM_MOVERESIZE`，Wayland 走 `xdg-decoration`；
   KDE / GNOME / Sway / Hyprland 在 CSD 协商上有历史 bug，需要回归测试。
4. **DPI 切换 / 跨屏拖拽**：Windows DPI 切换是异步事件（`WM_DPICHANGED`），
   标题栏尺寸与按钮位置的重绘时机若错开，用户会看到短暂错位；需在槽里 `update() + adjustSize()`。
5. **失焦变灰的一致性**：Windows 11 系统标题栏失焦会变灰，自绘 TitleBar 若不模拟会不一致；
   模拟又与 Liquid Glass 深色主题的暗背景冲突（文字已是白色，再降 alpha 看不清），
   需在 Theme spec 里加「失焦色」字段或显式决定**不模拟**。
6. **菜单栏视觉叠加阴影**：Qt 的 `QMenuBar` 通常在标题栏下方；frameless 后菜单栏的样式表
   （已硬编码 light / 深色分支各一套，见 `MainWindow.cpp:192` 附近）会与 CustomTitleBar 视觉重叠，
   需统一配色避免分界线。
7. **Spec 12 INV-4 弹窗跟主题**：主题插件系统 INV-4 规定「弹窗也要跟主题」。
   `ChangelogDialog` / `ShortcutsDialog` 目前用 `setWindowTitle()` 但保留系统原生标题栏，
   需要决定：**这些弹窗也改 frameless 吗？** 还是只主窗口 frameless、弹窗保留原生？
   （建议 V1 只改主窗口，弹窗 V2 再说。）
8. **焦点模式（演示模式）**：`MainWindow::enterFocusMode()` 目前 `showFullScreen()`；
   frameless 下 `showFullScreen` 行为正常但要确保 CustomTitleBar 被隐藏（新增 `m_titleBar->hide()`）
   和 `exitFocusMode()` 对应 `m_titleBar->show()`。
