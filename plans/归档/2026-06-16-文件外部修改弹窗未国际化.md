# 文件被外部修改弹窗文案未国际化

## 背景

用户报告（2026-06-16）：

> 文件被外部修改后，有一个弹窗，上面的文案没做国际化。

当编辑中的文件在外部（其他编辑器 / 进程）被修改后，应用弹出提示对话框
（询问是否重新加载 / 保留当前内容等）。该弹窗的文案（标题 / 正文 / 按钮）
未用 `tr()` 包裹，或未在 .ts 文件中补对应翻译，导致语言切换时文案不跟随。

违反 [INV-I18N-TR] / [INV-I18N-TS]。

## 目标（验收条件）

- [x] 定位文件外部修改提示弹窗的所有用户可见字符串
- [x] 全部用 `tr()` 包裹（标题、正文已 tr；按钮改自定义 tr 文案）
- [x] 同步补 zh_CN.ts / en_US.ts 的 `<message>` 条目及翻译
- [x] lrelease 生成 .qm，构建验证中文界面下文案均正确

## 设计

根因（实测确认）：弹窗标题/正文本就 `tr()` 且 zh_CN.ts 已翻译，问题在**标准按钮 Yes/No**。
Qt5.12 `QMessageBox` 标准按钮文案取自 `QPlatformTheme` context，而本机可用的 `qt_zh_CN.qm`
（遗留聚合格式）把翻译放在 `QDialogButtonBox` context，二者不匹配 → 按钮回退英文。
匹配的 `qtbase_zh_CN.qm` 本机缺失。诊断输出：
`translate("QPlatformTheme","&Yes")=&Yes`、真实 `QMessageBox` 按钮=`&Yes`。

方案（Spec INV-QMSGBOX-TR）：
1. 弹窗改用**自定义 tr() 按钮**：`tr("Reload")` / `tr("Keep Current")`，响应用 `clickedButton()` 判断。
2. 补 qt_zh_CN.qm 加载（嵌入 qrc，Spec INV-3），覆盖编辑器右键菜单等其他 Qt 内置 UI。
3. zh_CN.ts 补 "Reload"→"重新加载"、"Keep Current"→"保留当前"；en_US.ts 同步占位。

## 验证

- 新增 `tests/app/ReloadDialogI18nTest.cpp`（T-7）：加载 zh_CN.qm 后断言
  `translate("MainWindow","Reload")=="重新加载"`、`"Keep Current"=="保留当前"`，防漏翻译回归。
- `build_on_win.bat release` 默认路径 ctest 22/22 全过（含新测试，lrelease 自动刷新 qm）。
- 视觉验证（`tmp/i18n_check.ps1` 外部改文件触发弹窗截图）：标题「文件已更改」、正文与按钮
  「重新加载 / 保留当前」全部中文。
