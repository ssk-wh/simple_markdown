# A1 预览视口剪裁渲染（virtualized rendering）

## 背景

`PreviewLayout::buildFromAst` 当前一次性 layout 整文档：1k 行已在 60fps 内
（INV-8 基线 P50 ≈ 12 ms），但 10k 行以上文档（笔记+代码片段混排）的拖 splitter / 调字号 / 改主题时
全量 layout + repaint 会突破 16ms 单帧预算。

主流 Markdown 编辑器（Typora、Obsidian）面对长文档都做了视口剪裁。我们的自绘渲染管道有
天然优势——`paintBlock` 已按块逐个绘制，差的是 **layout 阶段**也按需。

## 目标

- 文档行数 ≥ 5000 时，"调宽度 / 改字号 / 改主题"触发的 layout 单帧 ≤ 16 ms（P95）
- 10k / 20k 行文档拖 splitter 不再出现可感知卡顿
- 不破坏现有 INV：保持 INV-3 滚动 60fps、INV-COORD-ABS 子块绝对坐标
- 不破坏滚动同步（INV-4）：visible window 切换时 sourceLine ↔ block 映射仍准确

## 影响范围

- 修改：`src/preview/PreviewLayout.cpp`（核心）、`src/preview/PreviewWidget.cpp`（视口监听 + 触发重排）
- 新增：可能需要 `PreviewLayout::buildVisibleRange(int firstLine, int lastLine)` 接口
- Spec 变更：`specs/模块-preview/02-布局引擎.md` 新增 INV「视口外块允许仅占位高度」、
  `specs/横切关注点/70-性能预算.md` 新增 INV-9「大文件视口剪裁预算」

## 步骤拆分

- [ ] Step 1: 写 benchmark 测量 10k / 20k 行文档当前 layout 耗时（扩展 PreviewRenderBenchmark）
- [ ] Step 2: 设计"块高度快速估算"算法（不真 layout 也能拿到近似高度，用于占位）
- [ ] Step 3: 改造 `buildFromAst` 支持 `(firstLine, lastLine)` 范围参数
- [ ] Step 4: PreviewWidget 监听 scroll/resize，超出视口 ±2 屏时重新 layout
- [ ] Step 5: 处理边界（首/尾 / 章节边界 / 滚动同步 sourceLine 映射）
- [ ] Step 6: 回归 INV-8 基线，确保 1k 行场景没有性能倒退

## 风险与回滚

- 风险 1：占位高度估算不准 → 滚动到视口外区域时高度跳变
  - 缓解：估算偏保守（用最大可能高度），实际 layout 后修正
- 风险 2：滚动同步 sourceLine ↔ block 映射在"未 layout 块"上失效
  - 缓解：保留所有块的 sourceLine 索引但不计算 bounds，映射按行号
- 回滚：feature flag 控制（`view/virtualized_rendering` 配置），有问题关回全量 layout

### 2026-05-12 Step 2：视口剪裁算法设计

#### 数据结构

`LayoutBlock` 新增字段：
```cpp
bool placeholderOnly = false;  // 视口外占位，bounds 是粗估
```

不新增其他字段，inlineRuns/children 在 placeholder 模式保持空 vector。

#### API（向后兼容）

```cpp
class PreviewLayout {
public:
    // 现有：buildFromAst(root) 行为不变——若 viewport range 未设置则走全量
    void buildFromAst(const std::shared_ptr<AstNode>& root);

    // 新增：设置视口像素 Y 范围（widget 计算时含 ±2 屏 buffer）
    // 设置后下次 buildFromAst 走视口剪裁路径
    void setViewportYRange(qreal topY, qreal bottomY);

    // 新增：清除视口剪裁，退回全量模式（首次 layout / 字号改变 / 全量回归测试时用）
    void clearViewportYRange();
};
```

#### 核心算法 quickEstimateHeight(node, maxWidth)

视口外块的高度估算公式（粗估，允许 ±30% 偏差）：

| 块类型 | 估算公式 |
|--------|----------|
| Paragraph | `rawText.length() * baseFm.averageCharWidth() / maxWidth` 上取整 → 行数 × lineHeight |
| Heading | `headingScale[level] × lineHeight`（H1=2.0, H2=1.6, ..., H6=1.0） |
| CodeBlock | `rawLines.size() × m_codeLineHeight + 2*8 px padding` |
| List | `items.size() × baseFm.height() * 1.5` |
| Table | `rows.size() × baseFm.height() * 1.5` |
| BlockQuote | 递归子块 quickEstimate × 1.05 |
| Image | 固定 200px（无法预判图片高度） |
| ThematicBreak | 20px |
| Frontmatter | `entries.size() × baseFm.height() * 1.4 + 16` |

**注意**：`averageCharWidth × N` 是 CLAUDE.md 反模式 A，但**仅在 placeholder 粗估**接受。新 INV
明确边界——精算路径（视口内块）仍按 INV-15 字符级一致。

#### buildFromAst 主循环改造

```cpp
void buildFromAst(const std::shared_ptr<AstNode>& root) {
    m_root.children.clear();
    qreal yCursor = topPadding();  // INV-PREVIEW-TOP-PAD

    for (const auto& childNode : root->children()) {
        // 1. 先 quick estimate 拿一个高度上界，判定是否在视口
        qreal estimatedH = quickEstimateHeight(childNode.get(), m_viewportWidth);
        bool inViewport = !m_viewportYRange.isValid() ||
                          rangesIntersect(yCursor, yCursor + estimatedH, m_viewportYRange);

        LayoutBlock block;
        if (inViewport) {
            block = layoutBlock(childNode.get(), m_viewportWidth);  // 完整精算
        } else {
            block = makePlaceholder(childNode.get(), m_viewportWidth, estimatedH);
        }
        block.bounds.moveTop(yCursor);
        yCursor += block.bounds.height();
        m_root.children.push_back(std::move(block));
    }
    m_root.bounds.setRect(0, 0, m_viewportWidth, yCursor);
}
```

#### PreviewWidget 视口监听 + 防抖

```cpp
void onScroll() {
    qreal newTop = verticalScrollBar()->value();
    qreal newBottom = newTop + viewport()->height();
    // ±2 屏 buffer：避免视口边缘频繁触发 rebuild
    qreal buf = viewport()->height() * 2.0;
    QRectF candidate(0, newTop - buf, m_viewportWidth, newBottom - newTop + 2 * buf);

    // 仅当新 buffer 与上次差超过 1 屏才触发 rebuild
    if (qAbs(candidate.top() - m_lastBuildTop) > viewport()->height()) {
        m_layout.setViewportYRange(candidate.top(), candidate.bottom());
        rebuildLayout();
        m_lastBuildTop = candidate.top();
    }
}
```

#### 滚动同步兼容性

- `sourceLineToY` / `yToSourceLine`：placeholder 块的 bounds 是粗估的，映射会偏差但可用
- placeholder 升级为精算时 bounds.height 可能改变 → 后续块 y 漂移 → **滚动条 maximum 漂移**
- 经典 virtualized scrolling trade-off：用户拖滚动条时可能感觉到"内容跳一下"，但比卡顿 1 秒可接受得多
- 缓解：升级时优先升级当前滚动位置**附近**的块（已经在视口内），其他延后
