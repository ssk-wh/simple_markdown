# 资源管理器隐藏再显示宽度不一致

## 背景

用户报告（2026-06-15）：

> Ctrl+B 隐藏资源管理器之后再打开，这部分区域的宽度和隐藏之前不一致。

现象：Ctrl+B 切换左侧资源管理器面板的显示/隐藏。隐藏前面板是某个宽度（如用户拖过的
自定义宽度），再次 Ctrl+B 显示出来后宽度**变了**（恢复成默认宽度 / 某个估算值 /
被压缩或撑大），没有保持隐藏前的宽度。

## 根因方向（待验证）

- **隐藏时未记住当前宽度**：Ctrl+B 隐藏可能用 `setVisible(false)` 或宽度置 0，
  再显示时拿不到隐藏前的宽度，回退到默认。
- **splitter 状态被覆盖**：显示时若走了 `restoreState` / 重新按百分比布局，会覆盖
  用户拖过的宽度（与历史「restoreState 覆盖面板宽度」同型坑）。
- **显示宽度来自估算**：再显示时按「屏幕宽度 / N」或窗口百分比重算，而非缓存的实际像素。

> 同域历史：项目修过多次面板宽度问题——目录折叠宽度抖动、宽度重启持久化、关闭文件夹
> Tab 栏跳动。本次是 **Ctrl+B 显隐切换** 这条路径上的宽度未保持，需独立定位。

## 复现待补充

| 项 | 待补 |
|----|------|
| 隐藏前宽度来源 | 默认宽度 / 用户拖过的自定义宽度，是否都不一致 |
| 不一致表现 | 变默认 / 变窄 / 变宽 / 每次都变 |
| 是否多次切换累积 | 反复 Ctrl+B 宽度是否持续漂移 |
| 与重启关系 | 仅本次会话切换，还是重启后也错 |

## 目标（验收条件）

- [ ] 复现并量化隐藏前后宽度差
- [ ] Ctrl+B 再次显示资源管理器时，宽度严格等于隐藏前的宽度（含用户自定义宽度）
- [ ] 反复切换不漂移
- [ ] Spec `20-左侧面板.md` 补充显隐切换的宽度保持 INV + T 条目
