# 预览区域支持查找功能

## 背景

编辑区已有 Ctrl+F 查找/替换栏（`src/editor/SearchBar`）。预览区目前不支持查找——
用户在阅读长文档时无法快速定位文本。对于纯阅读场景（演示模式 / 单预览模式）尤其
不便。

## 影响范围

- **修改**：
  - `src/editor/SearchBar.h/cpp`：解耦 ctor 从 `EditorWidget*` 到 `QWidget*`
  - `src/preview/PreviewWidget.h/cpp`：接入 SearchBar + SearchWorker；新增 m_searchHits
    + 跳转 + char offset → Y 映射
  - `src/preview/PreviewPainter.h/cpp`：新增 `setSearchHighlights` + `setCurrentSearchHit`
    渲染路径
  - `src/app/MainWindow.cpp`：Ctrl+F 路由按焦点 widget 派发
- **新增**：
  - `specs/模块-preview/11-预览区查找.md`（已写）
  - `tests/preview/PreviewSearchTest.cpp`
  - `translations/*.ts` 新条目（如有新字符串）
- **不动**：
  - `src/editor/SearchWorker.h/cpp`（已是泛用，直接复用）

## 风险与回滚

- **风险 1**：解耦 SearchBar 可能影响编辑器搜索（接受 EditorWidget* 改 QWidget* 是放宽
  ctor 类型，调用兼容；但要 grep 确认无强转依赖）
  - 缓解：`grep -n "SearchBar\(.*\)"` 确认所有调用处只传 widget pointer
- **风险 2**：char offset → Y 精度到块级——长块（多行段落）跳转可能定位到块顶不是
  匹配项的具体行
  - 接受：V1 简化，用户能感知"跳到大概位置"已够；V2 可加字符级精度
- **风险 3**：异步搜索响应可能在 SearchBar 关闭后到达
  - 缓解：requestId 比对（已是 SearchWorker 内置机制）
- **回滚**：单 commit / 多 commit 提交，任意 commit `git revert` 即可
