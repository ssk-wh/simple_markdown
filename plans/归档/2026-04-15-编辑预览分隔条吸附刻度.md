# 编辑 / 预览分隔条拖动时显示吸附刻度线

## 背景

用户原话：
> 调整编辑区域和预览区域的宽度时，用鼠标拖动中间的分割线，调整后如果我想还原回原来
> 等分的效果，就很难对应上。我希望拖拽过程中加一些标线，分别在编辑区和预览区域的
> 1/4, 1/2, 3/4 处。拖动到这几条垂直虚线附近（水平坐标差距在 5 以内）时，增加一个吸附效果。

当前 `MainWindow::createTab` 里用 `QSplitter(Qt::Horizontal)`，editor | preview 两栏默认等分
（`setSizes({640, 640})`）。用户拖拽后没有视觉参考点，也不能对齐到常用比例。

## 目标

拖拽 editor|preview 之间的 splitter handle 时：

1. 整个 tab 内容区（editor + preview 合计宽度）出现 **3 条水平坐标处的垂直虚线**：
   - 左 1/4（25%）
   - 中 1/2（50%）
   - 右 3/4（75%）
2. 虚线样式：半透明、1-2px 虚线，跟随主题（用 `theme.accentColor` 或 `accentColor.alpha=128`）
3. **吸附阈值 5px**（逻辑像素）：当拖拽位置落入任一虚线 ±5px 时，splitter 自动吸附到精确比例
4. 吸附触发时轻微高亮对应虚线（或加粗 / 变色），提示已吸附
5. 拖拽结束（mouse release）后虚线消失；无拖拽状态下虚线隐藏

## 动作（待实施时细化）

- [ ] 自定义 `TabSplitter` 子类（或用 eventFilter 拦截现有 `tab.splitter`）：
  - 监听 `QSplitter::splitterMoved(pos, index)` 或自行处理 handle 的 mouseMove
  - 在拖拽开始（hover on handle + press）时 `update()` 触发重绘
  - 在拖拽结束（release）清除状态
- [ ] 重绘：在 editor 和 preview 两个 widget 的 `paintEvent` 上覆盖一层透明 widget 画虚线，或者在 splitter 的 widget 区域内直接 `QPainter::drawLine`（`QWidget#tabSplitterOverlay`）
- [ ] 吸附逻辑：`splitterMoved` 收到 `pos` 后，若 `|pos - target| <= 5`，`setSizes` 为精确 `{target, total - target}`
- [ ] 颜色：从当前主题 `accentColor` 派生（alpha ≈ 128），无主题默认 `#88888888`
- [ ] 主题切换实时更新虚线色
- [ ] 新增 Spec：`specs/模块-app/` 下新建「编辑预览分隔条吸附」或追加到主窗口布局相关 Spec；INV：吸附阈值固定 5 逻辑像素；拖拽状态虚线必须可见
- [ ] CHANGELOG `Added`：编辑/预览分隔条拖拽时显示 1/4、1/2、3/4 吸附刻度线（默认开启）

## 验收（T 编号）

- **T1** 静止状态（未拖拽）：tab 内无虚线、无视觉噪音
- **T2** 鼠标按在 handle 上按住不松：editor 和 preview 区各自出现 3 条等距虚线（25% / 50% / 75%）
- **T3** 拖到 50% 附近（水平距离 ≤ 5px）：splitter 自动吸附到精确 50%，同时 50% 虚线高亮
- **T4** 1/4 和 3/4 同样吸附且高亮
- **T5** 拖到非吸附区（离所有虚线 > 5px）：正常跟随鼠标，无吸附
- **T6** 松开鼠标：虚线消失
- **T7** 高 DPI（175%/200%）下虚线位置正确（按逻辑像素对齐到 1/4, 1/2, 3/4）
- **T8** 切主题：虚线颜色同步跟随
- **T9** 演示模式（编辑器隐藏）：分隔条不可见时不画虚线
- **T10** 窗口 resize 中如果恰好在拖拽：虚线位置按新宽度重算

## 风险

- **渲染层级**：虚线如果画在 editor / preview 的子 widget 上，需要 `raise()` 到顶层；如果画在独立 overlay widget 上要确保 hit-test 不拦截鼠标事件（`Qt::WA_TransparentForMouseEvents`）
- **吸附阈值感觉**：5px 在高 DPI 下可能太窄；可考虑 `5 * devicePixelRatio`，或改为 `qMax(5, fm.height() / 4)`
- **与 QSplitter 内部行为冲突**：`QSplitter::splitterMoved` 在鼠标移动过程中不断 emit，`setSizes` 反过来会触发 `splitterMoved`——要防止无限递归，用 `blockSignals` 或 flag
- **多 Tab**：每个 Tab 有自己的 splitter，overlay widget 要 per-tab 创建，切 Tab 时 overlay 跟随显示的 tab
- **用户禁用**：有些用户不喜欢吸附（想精确手动调），可在视图菜单加一个「启用分隔条吸附」开关（默认开）
- **性能**：拖拽时每 mousemove 重绘 overlay，范围是整个 editor+preview 区；可只在 overlay widget 上绘制减少重绘面积
