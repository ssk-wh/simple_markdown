# 表格单元格内 `<br>` 未换行且原文显示

## 背景

用户报告（2026-06-16）：

> 单元格中的内容有 `<br>`，是不是应该换行，当前实际没有换行，而且渲染后 `<br>` 也显示了。

GitHub Flavored Markdown 中表格单元格内 `<br>`（或 `<br/>`）应渲染为**换行**。当前：
1. 没有换行（单元格内容仍在一行）；
2. `<br>` 字面量被当文本显示出来。

## 根因方向（待验证）

- 解析层：`<br>` 被当作 `HtmlInline` 节点，`collectInlineRuns` 的 `HtmlInline` 分支
  （PreviewLayout.cpp:743）按**原始文本**渲染（`run.text = child->literal` = "<br>"）→
  原文显示。需识别 `<br>`/`<br/>`/`<br />` 转为换行 run（text = "\n"，类似 LineBreak）。
- 单元格换行：即使产生换行 run，表格 cell 的 layout/paint 是否支持 cell 内多行？
  需确认 cell 高度计算与绘制对 "\n" run 的处理（paintInlineRuns 对 run.text=="\n" 换行，
  但 cell 高度估算 estimateParagraphHeight 是否计入）。

## 目标（验收条件）

- [x] 单元格内 `<br>` / `<br/>` / `<br />` 渲染为换行，不显示原文
- [x] 单元格高度正确容纳多行（estimateParagraphHeight 的 "\n" 分支自动多记一行，T-BR-3 验证高度增加）
- [x] 普通段落内 `<br>` 也换行（collectInlineRuns 通用入口，段落/表格一并处理）
- [x] 复制语义：换行处复制出 `\n`（extractPlainText 收集 "\n"，run 流断言含 line1\nline2）
- [x] 加测试覆盖 `<br>` → 换行（PreviewBrLineBreakTest T-BR-1..4）

## 实现

根因：`collectInlineRuns` 的 `HtmlInline` 分支统一按 `run.text = child->literal` 渲染原文，
`<br>` 既不换行又显示字面量。

修复（最小、复用成熟机制）：在 `HtmlInline` 分支归一化 `<br>` 变体
（`literal.simplified().remove(' ').toLower()` == `<br>` / `<br/>`）→ 产出 `text == "\n"` 的
InlineRun，与 `LineBreak` 同走换行通路。`estimateParagraphHeight`（"\n" → ++totalLines）、
`paintInlineRuns`（"\n" → 换行）、`extractPlainText`（收集 "\n"）三处已成熟支持，零新增排版逻辑。

Spec：02-布局引擎 新增 INV-BR-LINEBREAK + T-BR-1..4。

## 验证

  - T-BR-2 变体 `<br/>` `<br />` `<BR>` `<br  />` 全换行
  - T-BR-3 含 `<br>` 段落高度 56.2 > 单行 30.7（确实占两行）
  - T-BR-4 表格单元格 `first<br>second` → `first\nsecond`、不含 `<br>`
