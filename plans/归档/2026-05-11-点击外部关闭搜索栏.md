# SearchBar 增加"点击外部自动关闭"机制（坐标判断版）

## 背景

用户原话：

> 搜索框现在不会主动消失，我希望增加一条消失的逻辑，就是收到了鼠标事件之后，
> 然后这个事件不是发给搜索框的，那么搜索框应该消失，这表示用户点击了搜索框外部
> 的应用内部其他区域，可以用事件发生坐标进行区分

上一轮（INV-13 v3）取消了 FocusOut 自动隐藏因为延迟检查 hasFocus() 在密集按钮交互
下不可靠。用户希望恢复"点外部就关"的语义，但用**鼠标坐标判断**替代——避开 focus
的事件链时序问题。

## 设计

### 全局事件过滤器 + 全局坐标判断

`SearchBar` 在 `showSearch()` / `showReplace()` 末尾 `qApp->installEventFilter(this)`，
`hideBar()` 顶部 `qApp->removeEventFilter(this)`。

`eventFilter` 中新增分支：

```cpp
if (event->type() == QEvent::MouseButtonPress) {
    auto* me = static_cast<QMouseEvent*>(event);
    QPoint gp = me->globalPosition().toPoint();   // Qt 6 API；Qt5 用 globalPos()
    QRect myGlobal(mapToGlobal(QPoint(0,0)), size());
    if (!myGlobal.contains(gp)) {
        // 点击在 SearchBar 几何范围之外 → 关闭
        QTimer::singleShot(0, this, &SearchBar::hideBar);
        // 不消费事件——目标 widget 正常收到点击，SearchBar 在下一个事件循环 hideBar
    }
}
return QWidget::eventFilter(obj, event);
```

**关键点**：
1. 用 `globalPosition().toPoint()`（Qt 5.14+ 的新 API）—— 项目 Qt 版本若 < 5.14
   用旧的 `globalPos()`。看构建脚本 / CMakeLists 决定。
2. `QTimer::singleShot(0, ...)` 延迟到下个事件循环执行 hideBar——避免在事件分发
   过程中销毁/隐藏 widget（safer Qt pattern）
3. 不调 `event->accept()`——让点击事件正常传到目标 widget（用户点编辑器一行内容
   仍能定位光标）
4. 全局 filter 在 hideBar 时立即 `removeEventFilter`——避免隐藏后还监听

### 为什么坐标判断比 FocusOut 更稳健

- FocusOut 是异步事件 + 延迟检查 → 密集按钮交互时 m_findEdit 的 hasFocus 可能
  短暂 false → 误触发
- MouseButtonPress 是用户**真实操作**的同步事件 → 点击位置确定，判断结果确定
- SearchBar 自绘按钮区域 = SearchBar widget 几何范围内 → 自然不被误触发

### 互斥与现有路径

- 用户 Esc / 关闭按钮 → 调 hideBar → 顶部 removeEventFilter ✓
- 用户 Ctrl+F 切到另一侧 → 走 MainWindow 互斥路由 → 调非目标侧 hideSearchBar →
  hideBar → removeEventFilter ✓
- 用户点击外部（编辑器内容 / 预览内容 / TOC / 工具栏 / 标题栏…）→ 全局 filter
  捕获 MousePress → 不在 SearchBar globalRect → hideBar ✓
- 用户点击搜索栏自己（输入框 / 工具按钮 / 间隙）→ 全局 filter 捕获 MousePress →
  在 globalRect 内 → 不关 ✓

## 验证

- 编辑器或预览搜索栏打开 → 点击编辑器内容/预览内容 → 搜索栏关闭，搜索高亮清除

- 搜索栏打开 → 点击应用工具栏 / 菜单栏 / TOC 面板 → 关闭
- 连续快速点击 30 次"下一个" → 不关闭（点击位置都在 SearchBar 内）

## 风险

- 应用启动后某些异常路径让 SearchBar 不可见但 filter 未 remove → 内存上 install
  着 filter 但 isVisible=false 时点击外部也不触发实际 hideBar（hideBar 顶部判 isVisible
  返回 no-op）——可加 `if (!isVisible()) return QWidget::eventFilter(obj, event);`
  在 filter 入口做防御
- 异常情况：用户点击系统对话框 / 子窗口 → 全局坐标在 SearchBar 矩形外 → 关闭。
  这可能不是用户期望——但与"点击外部即关"语义一致，可接受
