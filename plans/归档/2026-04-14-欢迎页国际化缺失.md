# 欢迎页国际化缺失

## 背景

首次启动欢迎对话框(MainWindow::onShowWelcome, src/app/MainWindow.cpp:2046)的文案在中文 UI 下仍显示英文。
源码使用了 `tr()`,`translations/simple_markdown_zh_CN.ts` 也有对应翻译条目,但运行时未生效。

**历史重复问题**:0.2.4 新增「快捷键面板」时也出现相同现象,当时已修一次。这次欢迎页又踩同一个坑,
说明新增 UI 时 i18n 流程没有形成稳定 checklist,应固化规则防止第三次再犯。

## 根因

- `resources.qrc` 引用的路径是源码 `translations/simple_markdown_zh_CN.qm`(7.5K 旧文件,含最后一次手动 lrelease 的 0.2.3 内容)
- CMake `qt5_create_translation` 生成新 qm(10.4K,含欢迎页翻译)但**输出到 `build/src/app/`**,qrc 根本不会引用它
- 开发者在 334a10f / f2c23c1 两次提交改 .ts 但没手动 `lrelease`,也没提交新 .qm
- 结果:`tr()` + .ts 翻译齐全,但运行时加载的 .qm 是旧的,欢迎页走 fallback 显示英文源串

## 动作

- [x] 定位根因:qm 输出路径与 qrc 引用路径不一致
- [x] 立即用 `lrelease` 生成最新 qm 到 `translations/`(10.4K,lconvert 反解确认含欢迎页翻译)
- [x] 改 `src/app/CMakeLists.txt`:通过 `set_source_files_properties(... OUTPUT_LOCATION ${CMAKE_SOURCE_DIR}/translations)`
  让 `qt5_create_translation` 直接输出到源码 `translations/`,每次 build 自动刷新,永久消除"忘记 lrelease"可能
- [x] 完整重建:删所有 qm,build 日志确认 `Generating D:/.../translations/simple_markdown_zh_CN.qm` 在 qrc_resources.cpp 编译前
- [x] 启动 exe,PrintWindow 截欢迎对话框,目视确认标题「欢迎使用 SimpleMarkdown」+ 全部 7 条特性 + 提示句均为中文
- [x] 补 `specs/横切关注点/60-国际化.md`:新增 INV-1.1(qm 与 qrc 路径一致)、T-5/T-6 验收条件、第 9 节新 UI 字符串 Checklist(6 步),
