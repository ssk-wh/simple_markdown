# A8 编辑器行号与文本绘制合一

## 背景

行号与文本绘制当前是**两个独立 paint loop**：
- `EditorWidget::paintEvent` 顶部 loop 画行号，每行 `lineY(line)`
- `EditorPainter::paint` 内部 loop 画文本，每行 `lineY(line)`

两者**约定共享 yCache**——任何时候 yCache 不准（估算 vs 精算混合 / wrap 多行 /
新增行漂移 / 边界误差 / paint 中间态）→ 立刻错位。

近期累计的补丁均针对此契约：
1. A7 EditorPainter::paint 入口预热 ensureLayout
2. EditorWidget::paintEvent 顶部预热 + ±3 行 buffer
3. 行号矩形高度从 lineHeight(line) 改为 singleLineH（修 wrap 多行行号居中）

用户实测仍有偏移："行号还是有偏移，本来就没有对应上"。补丁方向回报递减——
契约本身脆弱，继续打补丁价值不大。

## 目标

- 行号与文本绘制 Y 坐标**像素级一致**（无论 wrap 模式 / 新增行 / 视口位置）
- yCache 不再是行号/文本绘制的依赖——它只服务于 hitTest / cursor / search highlight 等
  辅助路径
- 删除 A7/A8 累积的补丁代码（预热、±3 buffer、ensureLayout 末尾 invalidateYCache）
- 不破坏拖动卡顿修复（A7 性能收益保留）

## 影响范围

- 修改：
  - `src/editor/EditorWidget.cpp`（paintEvent 主 loop 重构）
  - `src/editor/EditorPainter.h/cpp`（paint 拆出 paintLine + 装饰函数）
- 新增：
  - `EditorPainter::paintLine(painter, layout, doc, line, cy, ...)` 单行绘制
  - `EditorPainter::paintCursor(painter, layout, ...)`（已隐式存在，提取为函数）
  - `EditorPainter::paintSearchHighlights(painter, layout, ...)`
- 删除：
  - 补丁累积的预热 loop + ±3 buffer + 重算视口范围逻辑
  - 可选：`EditorLayout::ensureLayout` 末尾的 `invalidateYCache`（如不再需要）
- Spec 变更：
  - `specs/模块-editor/03-绘制管线.md` 写 INV「行号与文本必须在同一 per-line loop 中
    用本地 cursor Y 推进绘制」

## 风险与回滚

- 风险 1：cursor / search highlight 当前在 paint loop **之前**画，依赖 yCache 拿到
  位置。改成 per-line loop 后这些装饰可能需要重新组织
  - 缓解：cursor 行高亮可在 per-line loop 内"该行 == cursorLine 时画背景"；
    search highlight 同理"该行有匹配时画背景"
- 风险 2：搜索匹配可能跨 wrap 多行，原逻辑用 textLine.y() 在 QTextLayout 内部定位
  - 缓解：保持原逻辑，只是把 baseY 改为本地 cy（不再 lineY）
- 风险 3：IME 预编辑使用 cursorRect → positionToPoint → lineY，仍依赖 yCache
  - 缓解：保留 A7 的 ensureLayout 末尾 invalidateYCache 让 yCache 维持精算
