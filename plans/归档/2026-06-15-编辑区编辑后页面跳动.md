# 编辑区内容编辑后页面跳动

## 背景

用户报告（2026-06-15）：

> 编辑区域内容编辑后，页面内容会跳动好几下才会停止，严重影响编辑体验。

现象：在编辑区敲字/改内容后，页面（编辑区内容、可能含滚动位置）会**连续跳动好几下**
才稳定下来。属于明显可感知的视觉抖动，用户明确反馈「严重影响编辑体验」→ **P1**。

## 根因方向（待验证）

- 编辑触发的**多次重排/重绘连环**：一次编辑可能依次触发 文档高度重算 → 滚动条范围变化
  → 滚动位置被夹取(clamp)调整 → 又触发重排……形成几轮收敛震荡。
- **滚动锚点丢失**：编辑后没有以光标/可视行为锚点保持视口稳定，导致内容上下跳。
- **异步重排回灌**：编辑器视口剪裁渲染（参见 `A7 编辑器视口剪裁渲染`）下，真实高度
  分几帧补足，每帧补足都改一次总高 → 滚动条/内容位置逐帧跳。
- 与预览侧滚动同步可能形成反馈循环（编辑→同步→预览重排→回灌编辑滚动）。

> 关联历史：CHANGELOG 已修过若干「分隔条拖动重排」「行内代码基线抖动」类问题，但
> 「编辑输入后内容连跳」是新的体感问题，需独立定位。

## 复现待补充

| 项 | 待补 |
|----|------|
| 文档规模 | 小文档是否也跳？多少行明显 |
| 触发动作 | 输入单字 / 回车换行 / 粘贴 / 删除 行为差异 |
| 跳动内容 | 是内容上下位移，还是滚动条跳，还是光标行漂移 |
| 光标位置 | 文档顶部/中部/底部编辑时跳动是否不同 |
| 预览开关 | 仅编辑区（关预览）时是否仍跳（排除滚动同步反馈） |
| 跳动次数 | "好几下"≈ 几次，是否与视口剪裁补帧次数相关 |

## 目标（验收条件）

- [ ] 复现并录屏/计数跳动次数，定位是重排连环 / 滚动 clamp / 补帧回灌
- [ ] 编辑输入后视口保持稳定，内容不再多次跳动（一次到位）
- [ ] 不引入输入延迟（性能预算：输入响应、滚动一屏 < 16ms）
- [ ] 若涉及编辑器交互行为 → 在 specs/模块-editor 下补 Spec + INV

## 进展（补充）

- 2026-06-15 定位 + 修复：
  - 根因：编辑器 A7 视口剪裁懒布局下，未进视口的行用估算行高；`onTextChanged` 全量 rebuild
    把所有行重置为估算，之后空闲预加载逐帧精算视口上方行的真实高度（wrap 模式 > 估算），
    每帧改 Y 缓存却不补偿 scrollY → 视口内容多帧逐行位移 = "跳动好几下"。命中 plan「滚动锚点丢失」。
  - 修复（滚动锚定）：① onTextChanged rebuild 前记录首个可见行屏幕偏移，rebuild 后恢复；
    ② 预加载精算视口上方行时，比较首个可见行精算前后 Y、差值补偿到 scrollY。
    blockSignals 包裹补偿 setValue 避免重入 rebuild。typewriter 模式由 ensureCursorVisible 接管不锚定。
  - Spec：新增 specs/模块-editor/15-编辑滚动稳定.md（INV-EDIT-SCROLL-ANCHOR）。CHANGELOG：1.1.10。
  - 验证：编译通过。跳动是多帧时序+视觉现象，难离屏单测忠实复现；按黄金法则 #8 例外采用
    手动验证（长文档中部编辑/回车/删除 → 视口一次到位不跳；关预览复测排除同步反馈）。
