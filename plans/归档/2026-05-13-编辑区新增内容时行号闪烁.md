# 编辑区新增内容时行号闪烁

## 背景

用户报告（2026-05-13）：

> 编辑区我新增内容时，行数总是先闪一下再恢复

发现时间点：A7 编辑器视口剪裁（lineHeight 估算 + ensureLayout invalidateYCache +
EditorPainter paint 入口预热）落地后。

可能与 A7 改动相关——输入字符触发 `Document::insert` → `updateLines` →
`invalidateYCache` → `paintEvent` → 预热 ensureLayout（多次再 invalidate yCache）→
ensureYCache 重建 → 渲染。这条链路有可能让 GutterRenderer 短暂看到不一致的 yCache
或 lineCount 中间态。

## 复现条件

- [ ] 打开任意文档（建议长文档让行号视觉变化更明显）
- [ ] 在编辑器输入一个字符
- [ ] 观察左侧行号列：是否瞬间闪一下（位置 / 数字消失 / 颜色变化）后恢复

## 排查方向

1. **GutterRenderer 渲染时机**：是否在 ensureLayout 完成前调 lineY，拿到估算值，
   然后下一帧用精算值——视觉上行号位置跳一格
2. **A7 paint 预热的中间态**：预热 ensureLayout 多次 invalidate yCache，paint 内可能
   ensureYCache 被多次调用，每次结果可能在重建过程中被 GutterRenderer 误读
3. **行号渲染独立 paintEvent vs 一体 paintEvent**：若行号绘制时机与文本绘制不在同一帧
   可能出现"行号先闪"
4. **是否 A7 引入还是更早**：在 A7 commit 前回滚 build 实测确认

## 动作（实施时）

- [ ] 用 `--debug` 启动确认是否 lineCount 在帧间变化
- [ ] 检查 GutterRenderer 是否独立 paintEvent；若是，强制 gutter 与文本同步重绘
- [ ] 如果 A7 是回归源 → 给 ensureLayout 加批量模式（一次性预热全部视口内行后才
  invalidate yCache 一次，而非每行一次）

## 验收

- 编辑器单字符输入后行号无可感知闪烁
- 长文档拖动 splitter / 滚动等场景行号显示稳定
- 不破坏 A7 拖动卡顿修复

## 优先级

P2（视觉打磨级别，不影响功能；但属于"持续输入时的小烦扰"，长期累积影响体验）

## 进展

- 2026-05-13 启动，多轮补丁尝试：
  1. EditorPainter::paint 入口预热 ensureLayout（A7 改动延伸）
  2. EditorWidget::paintEvent 顶部加预热 + ±3 行 buffer
  3. 行号矩形高度从 lineHeight(line) 改为 singleLineH（解决 wrap 多行行号居中问题）
- 用户实测："行号还是有偏移，本来就没有对应上"——补丁方向不可持续：
  根因是"两个 paint loop（行号 / 文本）共享 yCache 契约脆弱"，任何中间态（估算 vs 精算
  混合 / wrap 多行 / 新增行漂移 / 边界误差）都让两 loop 错位
- 2026-05-13 用户提议合一方案——把行号视为每行的一部分，与文本在同一 per-line loop
  中用本地 cursor Y 推进绘制——一劳永逸
- **本 plan 废弃**，由 `plans/2026-05-13-A8编辑器行号与文本绘制合一.md` 继续
