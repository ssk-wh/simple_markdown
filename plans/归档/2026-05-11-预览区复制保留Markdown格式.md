# 预览区复制内容希望保留 Markdown 格式

## 背景

用户原话（2026-05-11）：

> 再预览区复制内容希望复制 markdown 格式的内容

当前行为（已实现）：
- `PreviewWidget::copySelection`：复制纯文本（plain text，去 Markdown 标记）
- `PreviewWidget::copyAsHtml`：复制 HTML（含格式但是 HTML 而非 Markdown 源）

用户期望：预览区选中内容后复制（Ctrl+C 默认动作）→ 剪贴板里是**原始 Markdown 源文**，
便于粘贴到其他 Markdown 编辑器 / Issue / PR 描述等场景。

## 设计要点（待细化）

### 1. 字符流映射 = 该 plan 是双区同步跳转 plan 的子问题

预览侧从 plain text 映射回 Markdown raw source 不是 1:1 的——需要：
- 编辑器侧持有 raw source（`m_doc->text()`），按行/字符索引
- 预览侧选区是 `m_plainText`（去标记）的 charOffset 范围
- 需要 preview charOffset ↔ source line/column 双向映射

这个映射在 `specs/模块-sync/` 中已经为"滚动同步"建立——但精度通常到**行级**，
不到字符级。复制场景需要：用户选中预览 5 行正文中的 3 行 → 拿到对应 source
range（行起到行末），切出 raw markdown 文本子串。

### 2. 三种粒度方案

| 粒度 | 实现 | 优点 | 缺点 |
|------|------|------|------|
| **行级**（推荐 V1） | 选区起止行 → editor 对应行起止 → 切 source 子串 | 实现简单，能覆盖 80% 场景 | 用户在一行中部选半行得到整行 Markdown |
| **块级** | 选区落在某 LayoutBlock 范围内 → 取该 block 对应的 sourceLine 范围 → 切 source | 与渲染语义对齐 | 块边界跨越复杂结构（如表格单元格内选词）时仍取整块 |
| **字符级** | preview plain charOffset → editor source charOffset 精确映射 | 选什么得什么 | parse 阶段需要保留 inline span 的 raw range；改动大 |

### 3. UX 选项

- 方案 A：**默认**复制 Markdown，右键菜单提供 "Copy as Plain Text" / "Copy as HTML"
  作为额外选项（与现有两项并列）
- 方案 B：保留 Ctrl+C 复制纯文本（向后兼容），新增 Ctrl+Shift+C 或右键 "Copy as
  Markdown" 选项

倾向方案 A——用户的"复制"意图在 Markdown 编辑器中自然指向 Markdown 源文。

### 4. 关联

- 与 `plans/归档/2026-05-11-预览编辑双区同步跳转.md` 共享"跨域字符映射"基础设施
- 实施前可先做基础映射，让两个 plan 一并受益

## 验收

- 预览区选中某段正文复制 → 粘贴到任意 Markdown 编辑器，看到的是原始 `**bold**` /
  `[link](url)` 等标记
- 选中代码块复制 → 含围栏 ``` 和语言标签
- 选中表格行复制 → 含 `|` 分隔符
- 选中跨多块内容（如标题 + 段落）复制 → 顺序保留，结构正确

## 风险

- 字符级精确映射改动大（parser 改造），V1 先行级即可
- 复制行为变化是 breaking change，向后兼容性需在 CHANGELOG / 右键菜单中显式说明
