# Tab 空白区右键菜单（新建 / 关闭所有），可替代固定「+」按钮

## 背景

用户建议（2026-06-16）：

> 在 tab 的空白区域右键可以弹出菜单，支持新建文件、关闭所有文档，这样就先不需要
> 固定显示一个新建按钮了。

现状：
- Tab **上**右键已有菜单（关闭 / 关闭其他 / 关闭左侧 / 关闭右侧等，见 INV-5）。
- Tab 栏**空白区域**（无 tab 处）右键目前无菜单。
- 固定「+」新建按钮见 INV-6（2026-06-16 刚实现，容器化后钉在 Tab 栏右端）。

## 目标（验收条件）

- [ ] Tab 栏空白处右键弹出菜单
- [ ] 菜单含「新建文件」「关闭所有文档」（文案走 tr()，同步 .ts）
- [ ] 评估是否移除固定「+」按钮（与 INV-6 取舍）——**需用户确认后再定**

## 设计要点（待细化）

- 容器化后 Tab 区是内部 `QTabBar`；空白区右键需在 `QTabBar` 的 contextMenu 中判断
  `tabAt(pos) < 0`（点在空白处）走「新建 / 关闭所有」分支，命中 tab 走既有 INV-5 菜单。
- 若决定移除「+」按钮，需同步回退 INV-6 的容器结构取舍（保留容器仍可，仅隐藏按钮）。
