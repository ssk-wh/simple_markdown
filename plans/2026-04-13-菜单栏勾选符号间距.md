# 菜单栏勾选符号与文字间距修复

## 背景

用户报告：视图菜单下的主题选项组，选中"跟随系统"时，前方的勾选符号（✓ 或 radio dot）紧贴文字，没有视觉间距，显得拥挤。

排查后发现根因在 `src/app/MainWindow.cpp` 的 `applyTheme()` 方法中，深色和浅色主题都通过 `setStyleSheet` 注入了自定义 QMenu 样式：

```cpp
"QMenu::item { padding: 6px 24px 6px 12px; }"
"QMenu::indicator { width: 16px; height: 16px; margin-right: 8px; }"
```

问题：

1. `QMenu::item` 左 padding 为 **12 px**
2. `QMenu::indicator` 占位 = 16 + 8 = **24 px**
3. padding-left 小于 indicator 占位，在 Fusion / WindowsVista 样式下 indicator 会被挤进文字区域，造成 ✓ 紧贴文字

此外，所有 checkable 菜单项都受同一 QSS 影响，不只是主题组。受影响的菜单包括：

- 视图 → 主题（3 个 radio）
- 视图 → Word Wrap（1 个 checkbox）
- 视图 → 行间距子菜单（5 个 radio）
- 视图 → Focus Mode（1 个 checkbox）
- 视图 → Restore Last File（1 个 checkbox）
- 设置 → 语言子菜单（2 个 radio）

## 目标

满足 `specs/模块-app/10-菜单栏样式.md` 的不变量：

- **INV-1** 勾选符号与文字间距 ≥ 1 个字符宽度
- **INV-2** 同一菜单内文本 X 起点列对齐
- **INV-3** 深浅主题视觉对等
- **INV-4** 不硬编码脱离字体度量
- **INV-5** 跨 Qt 样式一致
- **INV-6** 右键菜单不被破坏

## 影响范围

- **修改**：`src/app/MainWindow.cpp`（`applyTheme` 内的 QMenu / QMenu::indicator / QMenu::item 相关 QSS）
- **新增**：可能新增一个私有方法 `buildMenuStyleSheet(const Theme&)`
- **删除**：无
- **Spec 变更**：`specs/模块-app/10-菜单栏样式.md` 实施完成后 `status` 从 draft 升为 stable
- **不影响**：菜单功能逻辑、主题切换流程、其他自定义样式（TabBar、ScrollBar 等）

## 候选方案

### 方案 A（推荐）：调整 QSS 参数 + 注释

最小改动。直接修改 `applyTheme` 中 QMenu 相关的 QSS 字符串：

```cpp
// indicator 占位 = width(16) + margin-right(8) = 24
// padding-left 必须 >= indicator 占位 + 约 1 字符宽(~8) = 32
"QMenu::item { padding: 6px 24px 6px 32px; }"
"QMenu::indicator { width: 16px; height: 16px; margin-left: 8px; margin-right: 8px; }"
```

把 `padding-left` 从 12 加到 32，为 indicator 预留 24 + 8（间距）。

**优点**：
- 改动最小，风险可控
- 深浅主题同步修改即可
- 不引入新抽象

**缺点**：
- 仍然硬编码像素值（可以加注释说明推导）
- 字号变化时不会自适应（但当前菜单字号固定，问题不大）

### 方案 B：运行时根据 QFontMetrics 派生

抽出私有方法：

```cpp
QString MainWindow::buildMenuStyleSheet(const Theme& theme) const {
    QFontMetricsF fm(font());
    qreal charW = fm.horizontalAdvance(QChar('M'));
    qreal lineH = fm.height();

    int indicatorW = qRound(lineH * 0.8);
    int indicatorMR = qRound(charW * 0.6);
    int padLeft = qRound(indicatorW + indicatorMR + charW);  // INV-1
    int padRight = qRound(charW * 2);
    int padVert = qRound(lineH * 0.3);

    return QString(
        "QMenu::item { padding: %1px %2px %1px %3px; }"
        "QMenu::indicator { width: %4px; height: %4px; margin-right: %5px; }"
        // ... 其它规则
    ).arg(padVert).arg(padRight).arg(padLeft).arg(indicatorW).arg(indicatorMR);
}
```

**优点**：
- 满足 INV-4，随字号自适应
- 深浅主题共用同一结构，只切颜色

**缺点**：
- 改动较大
- 需要在字号变化时重新注入样式表（`applyTheme` 已经是主题切换时调用，字号变化暂不触发，但未来国际化换字体时可能需要）

### 方案 C：QProxyStyle 重写 PM_*

```cpp
class MenuStyle : public QProxyStyle {
    int pixelMetric(PixelMetric m, const QStyleOption* o, const QWidget* w) const override {
        switch (m) {
            case PM_SmallIconSize: return baseFontSize * 1.0;
            case PM_MenuHMargin: return baseFontSize * 0.5;
            // ...
        }
        return QProxyStyle::pixelMetric(m, o, w);
    }
};
QApplication::setStyle(new MenuStyle);
```

**优点**：
- 最纯净，完全脱离 QSS
- 作用于所有 QMenu 和 QMenu-like 控件

**缺点**：
- 和现有 QSS 冲突（QSS 会覆盖 pixelMetric）
- 需要删除现有菜单 QSS 或协调两套机制
- 改动面最大

### 决策建议

**优先方案 A**（快速修复 + 加注释说明像素推导）；若字号国际化需求推进再升级到方案 B。方案 C 暂不考虑。

## 风险与回滚

### 风险 1：padding-left 加大后非 checkable 菜单项看起来左边空太多

- **影响**：Exit、Undo、Zoom In 等没有勾选符号的项，前面会空出 32 px
- **缓解**：这正是 INV-2 要求的"文本 X 起点列对齐"，设计上就该如此
- **复核**：用户对比 VSCode、Notepad++ 等成熟编辑器的菜单，它们也是所有项左侧对齐

### 风险 2：右键菜单被连带改变

- **影响**：编辑器 / 预览 / TabBar 的右键菜单样式同步变化
- **缓解**：这是 INV-6 的明确要求，全部回归即可
- **复核**：手动打开每个右键菜单对比 before / after

### 风险 3：高 DPI 屏下 padding 过大或过小

- **影响**：Qt 对 QSS 的 px 值可能有缩放行为
- **缓解**：在 1x / 1.5x / 2.0x DPI 屏分别验证；若失真再切换到方案 B
- **复核**：覆盖 T-12

### 回滚方案

`git revert` 单个 commit 即可。改动局部于 MainWindow.cpp applyTheme 内两段 QSS 字符串，无数据迁移风险。
