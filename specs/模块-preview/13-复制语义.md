# 预览区复制语义

## 1. 目的

预览区选区支持三种复制粒度（剪贴板格式），覆盖不同的下游场景：

| 粒度 | 剪贴板内容 | 典型下游 |
|------|-----------|----------|
| **Plain Text** | 纯文本（去 Markdown 标记） | 邮件、聊天、纯文本编辑器 |
| **HTML** | 富文本 HTML | Word、邮件富文本、网页粘贴 |
| **Markdown** | **原始 Markdown 源文** | GitHub Issue、其他 Markdown 编辑器、Wiki |

## 2. 行为契约

### [INV-1] Markdown 复制使用块级精度

`copyAsMarkdown()` 的实现策略：

1. 找出 m_selStart..m_selEnd 覆盖的所有 LayoutBlock（用 plain text 字符流位置反查，
   策略与 `scrollToCharOffset` 一致）
2. 收集这些块的 `sourceStartLine` 最小值和 `sourceEndLine` 最大值
3. 从 PreviewWidget 持有的 `m_sourceText`（编辑器 raw markdown source 的拷贝）
   切出 [minLine, maxLine] 行
4. `\n` 连接后写入剪贴板

**为何块级而非字符级**：字符级精确映射需要在 parser 阶段保留每个 inline span 的
raw range，改动大。块级 V1 覆盖 80% 场景（用户复制整段或多段时正确，复制半行只
会拿到整行——可接受的近似）。

### [INV-2] 默认 Ctrl+C = Markdown 源（多 MIME 智能粘贴）— 修订二，2026-05-12

右键菜单两项（在 "Mark" 之前）：

```
Copy                 Ctrl+C    ← 默认即 markdown 源
Copy as HTML
Select All           Ctrl+A
```

**Ctrl+C 默认复制 Markdown 源**——同时写入剪贴板的 `text/plain` 和 `text/markdown`
两个 MIME slot：
- `text/plain` 字段就是 markdown 原文（含 `**bold**` / `[link](url)` / 围栏代码块等
  标记字符）
- `text/markdown` 字段也是同样的 markdown 原文（让按 MIME 协商的接收方明确识别）

**粘贴方按 MIME 协商自动选格式**：
- 支持 markdown 的目标（GitHub Issue、Typora、VS Code 等）渲染出 markdown
- 纯文本目标（终端、聊天框、邮件普通输入）粘出含标记的字符——用户可见 markdown 源

用户原话（2026-05-12）："如果是直接 md 格式的输入区域，我粘贴后就是 md，如果是纯
文本接收区域，我粘贴后就是纯文本"——本设计满足这一行为。

**为何不再有单独的 "Copy as Markdown" 菜单项**：默认 Ctrl+C 已是 markdown，
再单列重复入口反而增加心智负担。

**为何不再保留 "Copy as Plain Text"（去标记的纯文本）入口**：用户在 markdown 编辑
器场景下复制更常需要的是源文（粘到任何地方都能保留语义），去标记的纯文本反而是
少数场景——若未来用户提出需求再加 `copyAsStripped` 入口。

### [INV-3] m_sourceText 必须与 m_currentAst 同步

`m_sourceText` 必须等于"生成当前 m_currentAst 的那一份 markdown 源"。同步策略：

- MainWindow 把 `ParseScheduler::astReady` 信号桥接为 lambda：
  ```cpp
  connect(scheduler, &ParseScheduler::astReady,
          preview, [preview, doc](std::shared_ptr<AstNode> root) {
              preview->setSourceText(doc->text());   // 与 AST 同步
              preview->updateAst(std::move(root));
          });
  ```
- 即使用户在解析期间继续编辑，`m_sourceText` 也仍对应 AST 解析的那一帧——这对
  "复制 Markdown"语义正确：复制的是用户**看到的预览内容**对应的源文。

### [INV-4] 选区空时复制 no-op

若 m_selStart < 0 / m_selEnd <= m_selStart（无选区或 collapsed），三个 Copy 入口
全部 no-op，右键菜单项 disable。

## 3. 接口

```cpp
class PreviewWidget : public QAbstractScrollArea {
public:
    void setSourceText(const QString& source);  // INV-3 入口
    void copySelection();                        // 已有：Plain Text
    void copyAsHtml();                           // 已有：HTML
    void copyAsMarkdown();                       // 新增：原始 Markdown 源
private:
    QString m_sourceText;                        // INV-3 持有
};
```

## 4. 验收条件

- **[T-1]** 选中预览中一段 `**bold** 正文 [link](url)` → 右键 Copy as Markdown →
  剪贴板内容包含原始 `**bold**` / `[link](url)` 字符（不是去标记的纯文本）
- **[T-2]** 选中预览中一个代码块 → Copy as Markdown → 剪贴板内容含 ``` 围栏和
  语言标签
- **[T-3]** 选中跨多个块（如标题 + 段落） → Copy as Markdown → 顺序保留，行间结构正确
- **[T-4]** 无选区时三个 Copy 菜单项 disable
- **[T-5]** Ctrl+C 仍调 copySelection（Plain Text），保持向后兼容

## 5. 已知限制

- 块级精度——半行选区会复制整行 Markdown 源（V1 妥协）
- 跨表格单元格选区取整张表格的 source 行
- 不支持"选区起止位置在 source 中精确切片"
