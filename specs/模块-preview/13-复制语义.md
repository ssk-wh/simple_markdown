# 预览区复制语义

## 1. 目的

预览区选区支持两种复制粒度（剪贴板格式），覆盖不同的下游场景：

| 粒度 | 入口 | 剪贴板内容 | 典型下游 |
|------|------|-----------|----------|
| **Plain Text** | Ctrl+C / 右键 "Copy" | 选区对应的预览纯文本切片（所见即所得） | 邮件、聊天、纯文本编辑器、Markdown 编辑器 |
| **HTML** | 右键 "Copy as HTML" | 选区对应的富文本 HTML | Word、邮件富文本、Notion |

## 2. 行为契约

### [INV-1] 默认 Ctrl+C = 纯文本切片（所见即所得）— 修订三，2026-05-13

`copySelection()` 的实现极简：

```cpp
QString plain = m_plainText.mid(start, end - start);
QMimeData* mime = new QMimeData();
mime->setText(plain);
QApplication::clipboard()->setMimeData(mime);
```

剪贴板写入的就是预览区**正在渲染**的纯文本（`m_plainText` 由 `extractBlockText`
按渲染顺序拼接得到），不含任何 Markdown 标记字符。

**为何放弃 markdown 源切片路径**：1.1.2 ~ 1.1.5 期间曾尝试两种方案：
1. 修订一：单 MIME 输出去标记的纯文本（用户报告"复制 [link](url) 时被剥光标记"）
2. 修订二：双 MIME 输出原始 markdown 源（同时写 `text/plain` 与 `text/markdown`）
   ——多轮迭代（Heading/Table 子集特化、块级 vs 字符级精度）后仍无法稳定
   满足"复制时所见即所得"的体感，块级精度还会让"半行选区"扩展到整行 markdown
   源（与用户视觉选区不符）

修订三（本版）放弃 markdown 路径，统一为预览纯文本切片——**逻辑极简、行为可预测、
与用户视觉选区严格一致**。富文本场景由 `Copy as HTML` 独立路径覆盖。

### [INV-2] 复制选区为空时 no-op

若 `m_selStart < 0 || m_selEnd < 0`，或 `start == end`（collapsed 选区），
copySelection 直接 return，**不写剪贴板**——剪贴板保持上次内容。

**重要协同关系**（与 specs/模块-preview/12-选区与拖动.md INV-3/INV-4）：双击选词
**必须**严格命中且找到有效 [wordStart, wordEnd) 范围，否则 m_selStart/m_selEnd
会维持 mousePress 第二次按下时的 collapsed 状态，紧接着的 Ctrl+C 走本 INV 早
返回路径——用户感知为"复制完粘贴出旧内容"。该场景的修复在选区入口（12-选区与
拖动 INV-3）而非复制入口。

右键菜单的 "Copy" / "Copy as HTML" 在选区空时 disable（防止用户误触发 no-op）。

### [INV-COPY-PLAINTEXT-SYNC] 复制源 m_plainText 必须与当前剪裁的索引空间同步（2026-06-16 新增）

复制源 `m_plainText` 由 `extractPlainText()` 遍历**视口剪裁后的 layout** 构建——视口外
块是 placeholder（空 inlineRuns），故 m_plainText 只含当前剪裁可见区的文本，其字符索引
空间与 paint 的 `m_charCounter`（同样跳过 placeholder）一致。

**因此：任何重新剪裁 layout（`applyLayoutViewportCrop` + `buildFromAst`）的地方，都必须
在其后同步 `m_plainText = extractPlainText()`**，否则选区索引（新剪裁空间）切到旧 m_plainText
→ 复制粘贴为空或错乱。

历史教训（2026-06-16 用户报告："滚动到表格后复制单元格粘贴为空/错乱"）：
`PreviewWidget::scrollContentsBy` 在滚动超过半屏时重剪裁 layout，但**漏了重建 m_plainText**
→ 滚动后 m_plainText 停留在旧剪裁状态，选区在新剪裁空间切片 → 落到错误/越界区间 → 空白。
修复：scrollContentsBy 重剪裁后补 `m_plainText = extractPlainText()`。

> 命中 CLAUDE.md 反模式 B（同一字符流两处构建走样）：选区索引空间（paint charCounter）
> 与复制源（extractPlainText）必须始终基于**同一 layout 剪裁状态**。
> 测试 `PreviewCopyConsistencyTest` T-COPY-CONSIST-3 验证重剪裁后二者仍逐字节一致。

### [INV-3] Copy as HTML 单独路径（保留）

`copyAsHtml()` 流程：
1. 把选区对应的 `m_plainText` 片段当成临时 markdown 源
2. 走 `MarkdownParser::renderHtml` 渲染为 HTML
3. 写入剪贴板的 `text/plain`（原片段）和 `text/html`（渲染结果）双 MIME

下游富文本程序（Word / Notion / 邮件客户端）按 `text/html` 协商粘贴出富文本。

### [INV-4] m_sourceText 保留供未来扩展，当前复制路径不使用

`m_sourceText` / `setSourceText` 历史上为修订二的 markdown 源切片路径而存在
（参见 INV-1 历史段）。修订三后两个复制入口均改用 `m_plainText`，`m_sourceText`
目前**仅供未来需要原始 markdown 源时复用**（如"导出选区 markdown"独立入口）。

MainWindow 在 astReady 信号桥接 lambda 中仍同步赋值（保留同步链路、不退化）：
```cpp
connect(scheduler, &ParseScheduler::astReady,
        preview, [preview, doc](std::shared_ptr<AstNode> root) {
            preview->setSourceText(doc->text());
            preview->updateAst(std::move(root));
        });
```

## 3. 接口

```cpp
class PreviewWidget : public QAbstractScrollArea {
public:
    void setSourceText(const QString& source);   // INV-4 同步入口（保留）
    void copySelection();                         // 默认 Ctrl+C —— 纯文本切片
    void copyAsHtml();                            // 右键 "Copy as HTML" —— 富文本路径
private:
    QString m_sourceText;                         // INV-4 持有（当前复制路径未使用）
    QString m_plainText;                          // INV-1 / INV-2 实际切片来源
};
```

## 4. 验收条件

- **[T-1]** 选中预览中一段 `bold 正文 link` → Ctrl+C → 剪贴板纯文本就是
  "bold 正文 link"，**不**含 `**bold**` / `[link](url)` 等 markdown 标记字符
- **[T-2]** 选中预览中一段代码块内容 → Ctrl+C → 剪贴板内容是代码原文（不含
  围栏 `` ``` `` 标记）
- **[T-3]** 选中跨多个块 → Ctrl+C → 剪贴板顺序与预览渲染顺序一致，块间以换行分隔
- **[T-4]** 无选区时右键菜单 "Copy" / "Copy as HTML" 项 disable，Ctrl+C 走早返回
  路径**不写剪贴板**（剪贴板维持上次内容）
- **[T-5]** Copy as HTML：选中含 `**bold**` 的段落 → Copy as HTML → 粘到 Word 时
  显示为加粗
- **[T-6]** 双击选词后立即 Ctrl+C → 剪贴板内容等于 `m_plainText.mid(selStart, selEnd-selStart)`
  （与 specs/模块-preview/12 INV-3/INV-4 配合，确保选区永远不为 collapsed 时进入复制）

## 5. 已知限制

- 选区跨表格时复制结果是预览纯文本（cells 间以渲染顺序拼接），不保留表格视觉
  结构——下游程序粘贴出来是一行行 cell 文本，需要表格结构请用 Copy as HTML
- 选区跨图片时图片不进剪贴板（仅 alt 文本可能进 m_plainText）
