# 预览区选区与鼠标拖动

## 1. 目的

预览区（PreviewWidget）支持鼠标拖动选区。选区终点（m_selEnd）由
`textIndexAtPoint(QPointF)` 计算——把鼠标坐标映射到 m_plainText 中的字符索引。

## 2. 行为契约

### [INV-1] 鼠标在某一视觉行右侧空白处拖动时，选区终点 snap 到**该行**末尾

`textIndexAtPoint` 在鼠标位置不直接命中任何 segment 时（如表格右侧空白 / 段落
行末右侧空白）使用 fallback 找最近字符索引。**优先**取"鼠标 y 落在 segment
垂直范围（rect.top() ~ rect.bottom()）内"的 segment 作为候选，这一层候选中
按水平距离 dx 取最近——保证用户在某行内拖到右侧空白时，选区不会跨到上下行。

仅当鼠标 y **不**在任何 segment 垂直范围内时（如行间空白），fallback 才使用
2D 欧氏距离 `dy² + dx²` 找最近 segment。

历史教训（2026-05-09 用户报告 / 2026-05-11 修复）：在表格行 X 内拖到右侧空白
时，原算法用 2D 距离找最近——dx 相同（鼠标远离所有列），dy 决定胜负；表格上下
行的 segment dy 微小差异可能让算法选到上一行/下一行 segment，导致 selStart→
selEnd 跨过多行，**整段被高亮**而非"只选当前行"。

### [INV-2] 表格单元格内的拖动不跨列

（待补——目前 V1 把行内 cells 视为同行；如需"拖动只在 hover 列内 snap"另议）

## 3. 接口

```cpp
class PreviewWidget : public QAbstractScrollArea {
private:
    int textIndexAtPoint(const QPointF& point) const;  // INV-1 实现入口
    // m_selStart / m_selEnd 由 mousePress / mouseMove 调用 textIndexAtPoint 更新
};
```

## 4. 验收条件

- **[T-1]** 表格行右侧空白拖动：含 ≥ 3 行的表格中，从第 2 行某 cell 内开始
  按下鼠标，拖动到该行右侧空白区（鼠标 y 仍在第 2 行垂直范围内）→ 选区只覆盖
  第 2 行从起点到行末的字符，**不**扩散到第 1/3 行
- **[T-2]** 普通段落行右侧空白：在段落某行内按下，拖到该行右侧空白处 → 选区
  止于该行末，不跨到下一段
- **[T-3]** 行间空白处拖动：鼠标 y 落在两个 segment 之间的空白处（无 segment
  垂直覆盖）→ fallback 到 2D 距离选最近 segment，行为与历史一致

## 5. 已知限制

- 不支持表格"列内 snap"（拖到列右侧未到下一列时仍可能选到下一列起点）—— INV-2
  占位
- char offset 精度到字符级，但块级排版（如代码块/段落）的行内字符宽度估算依
  hitTestSegment 单字符 horizontalAdvance 累加，不一定百分百对齐绘制结果
