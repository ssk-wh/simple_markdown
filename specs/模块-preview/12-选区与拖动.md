# 预览区选区与鼠标拖动

## 1. 目的

预览区（PreviewWidget）支持鼠标拖动选区和双击选词。
- 拖动选区：选区终点（m_selEnd）由 `textIndexAtPoint(QPointF)` 计算——把鼠标
  坐标映射到 m_plainText 中的字符索引。
- 双击选词：在 `mouseDoubleClickEvent` 中走严格命中 + Word 边界分词路径。

## 2. 行为契约

### [INV-1] 鼠标在某一视觉行右侧空白处拖动时，选区终点 snap 到**该行**末尾

`textIndexAtPoint` 在鼠标位置不直接命中任何 segment 时（如表格右侧空白 / 段落
行末右侧空白）使用 fallback 找最近字符索引。**优先**取"鼠标 y 落在 segment
垂直范围（rect.top() ~ rect.bottom()）内"的 segment 作为候选，这一层候选中
按水平距离 dx 取最近——保证用户在某行内拖到右侧空白时，选区不会跨到上下行。

仅当鼠标 y **不**在任何 segment 垂直范围内时（如行间空白），fallback 才使用
2D 欧氏距离 `dy² + dx²` 找最近 segment。

历史教训（2026-05-09 用户报告 / 2026-05-11 修复）：在表格行 X 内拖到右侧空白
时，原算法用 2D 距离找最近——dx 相同（鼠标远离所有列），dy 决定胜负；表格上下
行的 segment dy 微小差异可能让算法选到上一行/下一行 segment，导致 selStart→
selEnd 跨过多行，**整段被高亮**而非"只选当前行"。

### [INV-2] 表格单元格内的拖动不跨列

（待补——目前 V1 把行内 cells 视为同行；如需"拖动只在 hover 列内 snap"另议）

### [INV-3] 双击选词必须严格命中某个 segment（2026-05-13 新增）

`mouseDoubleClickEvent` 在调用 `textIndexAtPoint` 之前**必须**先用 `seg.rect.contains(pt)`
做一轮严格命中判定——只要鼠标位置未落入任一 TextSegment 矩形，立即 no-op 退出。

历史教训（2026-05-13 用户报告 / 反模式 C）：`textIndexAtPoint` 是为"拖动选区"
设计的 snap 函数，在 segment 外的位置（行末空白、段间间距、表格右侧空白等）
仍会返回**最近** segment 的字符 index（永不返回 -1）。如果双击直接消费该
结果，会出现两种症状：

1. 双击点在段间空白 / 行末空白 → snap 到邻近段落的某个字符 → isWordChar 向
   两侧扩展 → 选中"邻近段落里某个无关的词"，用户感知"双击选不到我点的位置"
2. 第二次 mousePress 已把 m_selStart = m_selEnd 设为 collapsed（同位置），
   若 doubleClick 因 `idx < 0 || idx >= length` 不进入选词分支，选区维持
   collapsed 状态 → 紧接着的 Ctrl+C 在 `start == end` 早返回，剪贴板维持
   旧内容 → 用户感知"复制后粘贴出非预期内容"

严格命中后，双击在 segment 外的"中间地带"统一不动选区（与 contextMenuEvent
的处理对齐）。

### [INV-4] 双击选词使用 `QTextBoundaryFinder::Word` 边界（2026-05-13 新增）

双击选词的"词"边界**必须**使用 Qt 内置的 `QTextBoundaryFinder::Word` 计算，
不得用 `QChar::isLetterOrNumber()` 简单扩展。

历史教训（2026-05-13 用户报告）：原实现 `isWordChar = c.isLetterOrNumber() || c == '_'`
对 CJK（汉字、假名、谚文）也返回 true，向两侧扩展时**把整段中文当成一个"词"
吞掉**。例如双击 "今天天气真不错" 中的"天"，会一直扩展直到段落首尾——选中整段，
用户期望的"双击选词"语义（点哪个词选哪个词）失效。

`QTextBoundaryFinder::Word` 内部走 Unicode UAX#29 分词规则，与 QTextEdit /
浏览器 / 主流编辑器一致：
- ASCII 单词按 letter/digit run 切分（`hello world` → `hello` / `world`）
- CJK 按 ICU 字典/规则切分（`今天天气` → `今天` / `天气`，单字时退化为单字）
- 标点 / 空白单独成段（boundary 切出"非词项"区间）

算法：
1. `setPosition(idx)`；若不在 StartOfItem 上 → `toPreviousBoundary()` 直到找到
2. 从词首 `toNextBoundary()` 直到 EndOfItem 作为词尾
3. 退化：若 `[wordStart, wordEnd)` 不含 idx（说明切出的是非词项空白/标点段），
   选当前单字符 `[idx, idx+1)`

### [INV-SEL-HEIGHT] 选区/标记/搜索高亮矩形高度 = 行主字体高度（2026-06-15 新增）

预览区行内文本的选区高亮、内容标记高亮、搜索命中高亮，其矩形**高度必须统一使用
「行主字体高度」`defaultFm.height()`**（`defaultFm` = 该行第一个 InlineRun 的字体，
即 `paintInlineRuns` 中决定基线的同一来源），矩形顶部对齐行槽位顶 `curY`。

**禁止**用当前被高亮 run 自身的 `fm.height()` 作为矩形高度。

历史教训（2026-06-15 用户报告：「英文单词选中后阴影只覆盖上半部分」）：行内所有 run
共享统一基线 `baseline = curY + lineAscent`（INV-10，`lineAscent = defaultFm.ascent()`）。
若高亮矩形高度取**当前 run 自身**的 `fm.height()`，当该 run 字体与行主字体度量不一致时
（行首为 inline code 字号更小拉低基线、或中文字体回退后字形比 ASCII 度量更高），矩形
相对基线偏矮/偏移，导致单词只被盖住一部分。改用行主字体高度后，同一行所有 run 的高亮
高度一致、与基线同源对齐，紧贴文字且完整覆盖；且不含行距乘数（不取 `lineHeight`），
避免文字下方出现大片空白选区带。

> 命中 CLAUDE.md 反模式 B（同一语义两份来源走样）：选区矩形高度此前由「各 run 自测」，
> 与文本基线「行主字体定」两套来源，跨字体即走样。INV 强制对齐到单一来源 `defaultFm`。

注：命中测试用的 TextSegment.rect 仍保留各 run 自身高度（hit-test 精度），仅**视觉高亮**
矩形统一为行主字体高度。

## 3. 接口

```cpp
class PreviewWidget : public QAbstractScrollArea {
private:
    int textIndexAtPoint(const QPointF& point) const;  // INV-1 实现入口
    // m_selStart / m_selEnd 由 mousePress / mouseMove 调用 textIndexAtPoint 更新

protected:
    void mouseDoubleClickEvent(QMouseEvent* event) override;  // INV-3/INV-4 实现入口
};
```

## 4. 验收条件

### 拖动选区
- **[T-1]** 表格行右侧空白拖动：含 ≥ 3 行的表格中，从第 2 行某 cell 内开始
  按下鼠标，拖动到该行右侧空白区（鼠标 y 仍在第 2 行垂直范围内）→ 选区只覆盖
  第 2 行从起点到行末的字符，**不**扩散到第 1/3 行
- **[T-2]** 普通段落行右侧空白：在段落某行内按下，拖到该行右侧空白处 → 选区
  止于该行末，不跨到下一段
- **[T-3]** 行间空白处拖动：鼠标 y 落在两个 segment 之间的空白处（无 segment
  垂直覆盖）→ fallback 到 2D 距离选最近 segment，行为与历史一致

### 双击选词（2026-05-13 新增）
- **[T-DC-1]** 双击在 ASCII 单词字符上 → 选中整个 ASCII 单词（letter/digit run）
- **[T-DC-2]** 双击在 CJK 字符上 → 选中 BoundaryFinder 切出的"词项"（通常 1~2 字符），
  **绝不**吞掉整段 CJK 文本
- **[T-DC-3]** 双击在词与词之间的空白 / 标点上 → 选中**单个**字符（不蔓延到邻词）
- **[T-DC-4]** 双击在段间空白 / 行末空白处（鼠标坐标不命中任何 segment）→ **no-op**，
  m_selStart/m_selEnd 维持原值，紧接着的 Ctrl+C 不写剪贴板（或写入 collapsed 选区被
  copySelection 早返回拒绝）
- **[T-DC-5]** 双击后立即 Ctrl+C → 剪贴板包含 [m_selStart, m_selEnd) 的纯文本

### 选区高亮高度（2026-06-15 新增）
- **[T-SEL-HEIGHT-1]** 选中单词（纯英文 / 中英混排 / 加粗后接英文 / 行首 inline code 后接
  英文等场景）→ 离屏光栅化后，选区高亮矩形的垂直范围必须完整覆盖该单词字形墨迹的上下沿
  （`selTop ≤ inkTop`，`selBottom ≥ inkBottom`，允许 1px 抗锯齿容差）。
  实现见 `tests/preview/PreviewSelectionHeightTest.cpp`（须用 windows 平台 + Qt SDK 运行时
  渲染，offscreen 平台不光栅化文字）。

## 5. 已知限制

- 不支持表格"列内 snap"（拖到列右侧未到下一列时仍可能选到下一列起点）—— INV-2
  占位
- char offset 精度到字符级，但块级排版（如代码块/段落）的行内字符宽度估算依
  hitTestSegment 单字符 horizontalAdvance 累加，不一定百分百对齐绘制结果
- 双击选词的 CJK 分词依赖 Qt 的 `QTextBoundaryFinder::Word`——Qt 5.x 在没有 ICU
  library 时退化为基于 Unicode UAX#29 的简化规则，对中文可能切出"单字"，但
  绝不会再"整段一个词"。该限制可接受。
