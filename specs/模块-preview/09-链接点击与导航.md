# 预览区链接点击与导航

## 1. 目的

让用户在预览区通过 **Ctrl + 左键** 直接打开 Markdown 源文本中的链接（`[text](url)` 或裸 URL）：本地 Markdown 文件在应用内新 Tab 打开、可执行文件禁止自动打开、其他文档交给系统默认应用、`http(s)://` URL 走系统浏览器。目标是提供「可点击文档」体验，同时不破坏现有的选区/拖选/双击选词交互，并在安全边界上拒绝被链接数据诱导执行任意进程。

## 2. 输入 / 输出

### 输入

- `LayoutBlock` 树中的 `InlineRun`（已带有 `linkUrl` 字段，见 `PreviewLayout.h`）
- `QMouseEvent`（press / move / release）
- `Qt::KeyboardModifiers`（检测 `Qt::ControlModifier` 是否按下）
- 当前激活 Tab 对应的 Markdown 文件路径（用于解析相对路径基准目录）

### 输出

- 新 Tab 打开本地 Markdown 文档（通过 `MainWindow::openFile(QString)` 信号调用）
- 系统默认应用打开其他扩展名的本地文件（通过 `QDesktopServices::openUrl(QUrl::fromLocalFile(...))`）
- 系统默认浏览器打开 `http(s)://`、`mailto:`、`ftp://` 等标准 scheme（通过 `QDesktopServices::openUrl(QUrl)`）
- 悬停时的光标状态（`Qt::PointingHandCursor` vs `Qt::IBeamCursor`）
- 悬停时的视觉强调（链接底色/下划线在 Ctrl 按下时高亮）
- 错误弹窗或状态栏提示（文件不存在、被拒绝的可执行文件）

## 3. 行为契约

### [INV-1] 仅 Ctrl+左键触发跳转

触发条件严格是：`event->button() == Qt::LeftButton` **且** `event->modifiers().testFlag(Qt::ControlModifier)` **且** `mousePressEvent` 命中的位置在某个带 `linkUrl` 的 `InlineRun` 矩形内。

普通左键点击**不得**触发跳转，必须走现有的选区逻辑（光标定位、拖选起点记录）。这样避免用户在文本上随手一点就误打开文件。

### [INV-2] 相对路径基准是当前文件所在目录

对于非绝对、非 URL 的 `linkUrl`，解析基准是**当前 Tab 对应 Markdown 文件的目录**（通过 `QFileInfo(currentFilePath).absoluteDir()`）。若当前 Tab 是未保存的新文档（无路径），相对路径链接应当被视为无效并给出提示，而不是使用应用工作目录或用户 home 目录作为基准。

### [INV-3] 本地 Markdown 文件在应用内新 Tab 打开

指向扩展名为 `.md` / `.markdown` 的本地文件（相对或绝对），必须通过 `MainWindow::openFile()` 作为新 Tab 加载，而非交给系统默认应用。依据现有 MainWindow 的逻辑，该方法已负责去重（同一路径已打开则切换到对应 Tab）、窗口提升等。

### [INV-4] 其他文档类扩展名交给系统默认应用

对于扩展名在白名单内的本地文件（`.txt`、`.pdf`、`.png`、`.jpg`、`.jpeg`、`.gif`、`.bmp`、`.svg`、`.webp`、`.html`、`.htm`、`.csv`、`.json`、`.xml`、`.log`、`.yml`、`.yaml`、`.toml`、`.ini`、`.rst`、`.org`），调用 `QDesktopServices::openUrl(QUrl::fromLocalFile(absPath))`，让系统决定打开方式。

### [INV-5] `http(s)://` / `mailto:` / `ftp://` 走 QDesktopServices

直接 `QDesktopServices::openUrl(QUrl(link))`，不做任何修改。允许 scheme 列表：`http`、`https`、`mailto`、`ftp`、`ftps`。其他 scheme（如 `javascript:`、`data:`、`file:` 由用户复制粘贴得到的情况）**不得**自动打开，必须静默忽略并在状态栏提示。

### [INV-6] 禁止打开可执行文件（安全红线）

扩展名在以下黑名单的**本地文件**必须**禁止自动打开**，改为弹出提示「禁止打开可能的可执行文件 XXX」：

- Windows：`.exe` `.bat` `.cmd` `.com` `.ps1` `.vbs` `.msi` `.scr` `.reg` `.pif` `.lnk`
- Linux/macOS：`.sh` `.bash` `.zsh` `.fish` `.run` `.py` `.pl` `.rb` `.app`

黑名单优先级高于白名单。任何未在白名单且未在黑名单中的扩展名，默认归为「未知类型」，**不自动打开**，给出提示。这是对 `INV-SEC-EXT-FILES` 的具体实现。

### [INV-7] 防御路径穿越

解析出的绝对路径必须通过 `QFileInfo::canonicalFilePath()` 规范化。如果规范化后的路径指向**不存在**的位置，或无法读取 `QFileInfo::exists()` 返回 false，给出「文件不存在」提示，**禁止**继续调用 `QDesktopServices::openUrl`。如此即便链接文本里写了 `../../../../../windows/system32/cmd.exe` 也会因为黑名单 INV-6 先被拦截，路径穿越本身也因为规范化后不一定落在白名单而被拒。

### [INV-8] 悬停光标手型

`mouseMoveEvent` 在鼠标命中 `linkUrl != ""` 的 InlineRun 矩形时，把 `viewport()->setCursor(Qt::PointingHandCursor)`；离开链接区域恢复 `Qt::IBeamCursor`。**不得**依赖 `QApplication::setOverrideCursor`，避免影响其他控件。

### [INV-9] 悬停视觉强调分两级

链接文本的视觉分三种状态：

| 状态 | 视觉 |
|------|------|
| 正常 | 链接色 + 无下划线（现状） |
| 悬停（不按 Ctrl） | 链接色 + 下划线（提示这是可点击链接） |
| 悬停（按 Ctrl） | 链接色加深或加亮 + 下划线（提示此时点击会跳转） |

状态转换必须触发 `viewport()->update(hitRect)` 仅重绘命中矩形所在的行或块，避免全窗口重绘。

### [INV-10] 按键状态变化必须刷新悬停强调

用户按下/释放 `Ctrl` 时若鼠标正悬停在链接上，必须重绘该链接区域，使强调状态立即切换。实现途径：在 `PreviewWidget::keyPressEvent` / `keyReleaseEvent` 中检测 `Qt::Key_Control`，触发 `viewport()->update(m_lastHoverLinkRect)`。

### [INV-HOVER-TARGET] 悬停可跳转内容显示目标地址（2026-06-15 新增）

鼠标悬停在任意可跳转内容（Markdown 链接、裸 URL、本地文件路径等带 `linkUrl` 的 InlineRun）
上时，必须给出**它指向哪里**的可见反馈：在悬停位置弹出 tooltip 显示该 `linkUrl` 原文
（URL 显示原始地址；本地文件显示其相对/绝对路径，即 link 的 target 原样）。

实现：`PreviewWidget::mouseMoveEvent` 命中带 `linkUrl` 的 segment 时，
`QToolTip::showText(globalPos, linkUrl, viewport())`；离开链接时 `QToolTip::hideText()`。
用成员 `m_hoverLinkUrl` 做去抖——仅当悬停的 URL 变化时刷新，避免每次鼠标移动闪烁。
`linkUrl` 是数据非 UI 文案，不需 `tr()` 包裹（INV-I18N-TR 例外：URL/路径原文）。

与 INV-8（手型光标）协同：手型提示"可点"，tooltip 提示"点了去哪"。

### [INV-11] 跨行链接命中检测

同一个 `linkUrl` 可能因为换行被拆成多个 `InlineRun` 矩形（例如一个很长的 URL 跨两行）。命中检测必须对**整组属于同一个链接**的 InlineRun 分别测试；视觉强调时也必须整组一起高亮，不能只亮其中一行。

实现途径：每个 `InlineRun` 记录 `linkUrl`，查询命中时找到所有相邻的 `InlineRun`（相同 linkUrl、同一段落）作为一个逻辑链接返回。

### [INV-12] 与选区/拖选/标记高亮互斥

- 正在拖选时（`m_isDragging == true`）**不**触发悬停光标/链接高亮
- Ctrl+左键按下触发跳转后**不**进入拖选状态（跳过 `m_isDragging = true`）
- 链接视觉强调的绘制顺序：在文本之后、在内容标记高亮之前，避免链接下划线被标记色覆盖（标记色是半透明荧光，底色让下划线透出）

### [INV-13] 错误提示策略

- 文件不存在：状态栏临时提示 `无法打开：文件不存在 <path>`，超时 5 秒消失
- 被黑名单拦截：状态栏临时提示 `禁止打开可执行文件 <name>`，超时 5 秒消失
- scheme 不支持：状态栏临时提示 `不支持的链接类型 <scheme>`
- `QDesktopServices::openUrl` 返回 false：状态栏临时提示 `系统拒绝打开 <url>`

**不**使用模态对话框，避免在连续点击时打断用户。

### [INV-14] 非主线程禁入

所有链接解析、路径规范化、`QDesktopServices::openUrl` 调用都在 GUI 主线程发生；不得放进 `QtConcurrent::run` 或其他 worker，因为相对路径基准涉及当前 Tab 状态，读取必须在主线程。

## 5. 核心算法

### 命中测试

```
linkHitAt(point):
    block = blockAtPoint(point)  // 已有逻辑
    if block is null: return nullopt
    for run in block.inlineRuns:
        if run.linkUrl.isEmpty(): continue
        runRect = computeRunRect(run, block, device)  // 与 textIndexAtPoint 同一度量
        if runRect.contains(point):
            // 收集同段内所有相邻 linkUrl 相同的 run（跨行）
            siblings = collectAdjacentRunsWithSameLink(block, run)
            return LinkHit{ run.linkUrl, boundingRect(siblings), rectsOf(siblings), blockIndex }
    return nullopt
```

命中测试使用 `QFontMetricsF(run.font, viewport())`，符合 `横切关注点/40-高DPI适配.md` INV-2。

### 链接分类与分发

```
activateLink(rawUrl):
    if rawUrl.isEmpty():
        emit linkActivationError("空链接")
        return false

    QUrl url(rawUrl)
    if url.scheme() in {"http", "https"}:
        QDesktopServices::openUrl(url)
        return true

    if url.scheme() in {"mailto", "ftp", "ftps"}:
        QDesktopServices::openUrl(url)
        return true

    // 其他外来 scheme：拒绝
    if hasScheme(rawUrl) and url.scheme() not in 白名单:
        emit linkActivationError("不支持的链接类型：" + url.scheme())
        return true  // 已处理

    // 否则当作本地路径
    resolved = resolveLocalPath(rawUrl)  // 相对 → 绝对 + canonical
    if resolved.isEmpty() or !QFileInfo::exists(resolved):
        emit linkActivationError("文件不存在：" + rawUrl)
        return true

    ext = QFileInfo(resolved).suffix().toLower()
    if isExecutableExtension(ext):
        emit linkActivationError("禁止打开可执行文件：" + QFileInfo(resolved).fileName())
        return true

    if isMarkdownExtension(ext):
        emit requestOpenMarkdownFile(resolved)
        return true

    if isSafeDocExtension(ext):
        QDesktopServices::openUrl(QUrl::fromLocalFile(resolved))
        return true

    // UnknownFile：保守拒绝
    emit linkActivationError("未知文件类型：" + ext)
    return true
```

### 相对路径解析

```
resolveLocalPath(raw):
    QString candidate = raw
    // 解 URL encoding（例如 %20 空格）
    candidate = QUrl::fromPercentEncoding(candidate.toUtf8())
    // 去掉锚点（# 之后的部分暂不支持滚到章节，先忽略）
    int hash = candidate.indexOf('#')
    if hash >= 0: candidate = candidate.left(hash)

    QFileInfo fi(candidate)
    if fi.isAbsolute():
        return QFileInfo(candidate).canonicalFilePath()

    // 相对：以当前 Tab 的文件目录为基准
    if currentTabFilePath.isEmpty():
        return ""  // 未保存的新文档，无法解析相对路径
    QDir base = QFileInfo(currentTabFilePath).absoluteDir()
    return QFileInfo(base.absoluteFilePath(candidate)).canonicalFilePath()
```

### 悬停重绘节流

- 命中同一个链接时不触发重绘（比较 `m_hoverLink->url` 和新命中的 url）
- 离开链接时只重绘旧的矩形集合
- 进入新链接时只重绘新的矩形集合
- 不要 `viewport()->update()` 全量重绘

## 8. 已知陷阱

### 8.1 Qt::PointingHandCursor 不随 Ctrl 变化

`QApplication::keyboardModifiers()` 在 `mouseMoveEvent` 里读到的 Ctrl 状态是**事件产生时**的状态，不是实时的。用户按住 Ctrl 但不移动鼠标时，不会触发 `mouseMoveEvent`，因此必须靠 `keyPressEvent`/`keyReleaseEvent` 检测 Ctrl 按下/释放并手动刷新视觉状态（见 INV-10）。

### 8.2 QFileInfo::canonicalFilePath 对不存在路径返回空

这是 Qt 规定的行为：canonical 路径只对存在的文件有效。因此「不存在」判断必须先检查 `exists()`，再在存在的前提下调用 `canonicalFilePath()`，避免把「空字符串」错误地识别为「路径为空」。

### 8.3 Windows 大小写不敏感 vs Linux 大小写敏感

同一份 Markdown 在 Windows 上 `[x](A.md)` 可以打开 `a.md`，在 Linux 上不行。本 Spec 不对此做 workaround，保持平台原生语义；用户应按平台习惯写链接。

### 8.4 URL 编码

链接 `[space](a%20b.md)` 必须先 `QUrl::fromPercentEncoding` 再解析。否则会找不到带空格的文件。

### 8.5 锚点 / fragment

`[section](doc.md#heading-id)` 当前只打开 `doc.md`，**不**滚到锚点位置。这是后续增强项，本 Spec 显式不包含。但解析时必须正确去掉 `#...` 部分，不要把它当作文件名的一部分查找。

### 8.6 拖选起点与链接点击冲突

普通左键点击在链接上时需要同时作为「拖选起点」和「可能的链接点击」。要点：
- **按下**时只记录拖选起点，**不**打开链接
- **释放**时如果发现：按下和释放在同一点 + Ctrl 被按住 → 此时不做打开（这已在 press 时处理）；否则走正常拖选/选区
- 正确做法：Ctrl+Left **press** 直接触发打开，return 之前 `event->accept()` 吃掉该事件；非 Ctrl 或非 Left 一律交还基类

### 8.7 file:// scheme 的边界

用户可能写 `[x](file:///C:/Windows/System32/cmd.exe)`。按 INV-5 中的白名单，`file://` **不**在自动 scheme 白名单里，但可以被拆解为 `QUrl::toLocalFile()` 后走本地路径分支，此时 INV-6 的可执行扩展名黑名单会再拦截一次。双重防线。

### 8.8 Markdown 图片 `![alt](url)` 不应触发跳转

预览区图片是独立块，图片 `LayoutBlock::Image` 不走 InlineRun，其 `imageUrl` 字段不是链接点击入口。图片点击是否另行支持（例如用系统查看器打开原图）由后续 Spec 决定，本 Spec 只处理文本链接。

### 8.9 代码块内的链接样式

代码块 ```` ```cpp ``` ```` 内的 `http://...` 不应被当作可点击链接——这类文本不在 InlineRun 中（CodeBlock 用 `codeText` 单独存储），无 `linkUrl` 字段，天然不会命中。这条记录于此以便后续如果实现代码块的内嵌 URL 识别时不要踩坑。

### 8.10 `QDesktopServices::openUrl` 返回 false 不一定是真失败

在 Linux 某些桌面环境下即使 xdg-open 成功启动应用，`openUrl` 仍可能先返回，随后子进程出错。我们**只**根据返回值给出提示；真实的应用启动失败由操作系统层面负责显示。
