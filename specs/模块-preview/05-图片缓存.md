# 05 — 图片缓存

## 1. 职责
异步加载、缓存、渲染 Markdown 文档中的图片（含光栅图片和 SVG 矢量图），为布局引擎提供尺寸信息，为绘制管线提供 `QPixmap`。

## 2. 对应源码
`src/preview/ImageCache.h`, `src/preview/ImageCache.cpp`

## 3. 接口

### 3.1 公有接口

| 方法 | 用途 |
|------|------|
| `get(url)` | 返回缓存中的 `QPixmap*`，未就绪返回 `nullptr`（可能触发异步加载） |
| `getSize(url)` | 返回图片原始尺寸（不解码像素，仅读头部或 SVG 元信息） |
| `setDocumentDir(dir)` | 设置当前文档目录，用于解析相对路径 |
| `isFailed(url)` | 图片是否已加载失败 |
| `isLoading(url)` | 图片是否正在网络下载中 |
| `isNetworkUrl(url)` | URL 是否为 HTTP/HTTPS 协议 |
| `isDataUri(url)` | URL 是否为 data: URI |
| `clear()` | 清空缓存 |

### 3.2 信号
| 信号 | 触发时机 |
|------|----------|
| `imageReady(url)` | 图片加载完成（成功或失败） |

## 4. 数据流

```
Markdown: ![](diagram.svg)
    ↓ AST Image node (url)
    ↓ PreviewLayout: getSize(url) → 获取尺寸算高度
    ↓ PreviewPainter: get(url) → 获取 QPixmap* → drawPixmap
              ↕ (异步)
         ImageCache
    ├─ 本地文件: QFile + QSvgRenderer / QPixmap::load
    ├─ data URI: QByteArray → QSvgRenderer / QPixmap::loadFromData
    └─ 网络 URL: QNetworkAccessManager → QSvgRenderer / QPixmap::loadFromData
```

## 5. 不变量 (INV)

### INV-IMG-SIZE-LIMIT
单文件最大 20MB（`kMaxImageSize`），超过即标记为失败。

### INV-IMG-NET
网络图片异步加载，不阻塞主线程布局/绘制。

### INV-IMG-SVG-RENDER
SVG 图片使用 `QSvgRenderer` 渲染到 `QImage(ARGB32_Premultiplied)` 再转为 `QPixmap`，
不依赖 `QPixmap::load()` 的 SVG 插件路径。

### INV-IMG-SVG-SIZE
SVG 尺寸优先使用 `QSvgRenderer::defaultSize()`（来自 SVG 的 viewBox 或 width/height 属性）。
无 intrinsic size 时 Qt 内部返回固定值（Qt 5.12 为 100×100），此时仍可正常渲染。
该尺寸缓存到 `m_sizeCache` 避免重复解析。

### INV-IMG-SVG-DETECT
SVG 检测规则：
- 本地文件：扩展名 `.svg`（不区分大小写）
- data URI：Content-Type 包含 `image/svg+xml`
- 网络响应：Content-Type 包含 `image/svg+xml`

## 6. 线程模型
所有方法在 UI 主线程调用。网络下载在 Qt 事件循环中异步完成，回调也在主线程。

## 7. 验收条件 (T)

| 编号 | 描述 |
|------|------|
| T1 | 本地 `.svg` 文件正常渲染，尺寸与 viewBox 一致 |
| T2 | 网络 SVG 异步下载后渲染 |
| T3 | data URI `data:image/svg+xml;base64,...` 解码渲染 |
| T4 | 无 intrinsic size 的 SVG 使用 800×600 默认尺寸 |
| T5 | SVG 加载失败时显示错误占位（复用现有 failed 分支） |
| T6 | getSize 对 SVG 文件返回正确尺寸 |
