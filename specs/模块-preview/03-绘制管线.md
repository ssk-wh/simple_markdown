# 预览绘制管线（PreviewPainter）

## 1. 目的

把 `PreviewLayout` 产生的 `LayoutBlock` 树绘制到 QPainter 上。所有视觉细节（字体度量、颜色、坐标）的**最终执行方**。

## 2. 输入 / 输出

- **输入**：
  - `QPainter* p` — 目标画布
  - `const LayoutBlock& root` — 布局树
  - `const Theme& theme` — 颜色
  - `viewport rect` — 可视区（用于裁剪）
  - `selection / highlights` — 选区和标记
- **输出**：像素绘制到 QPainter

## 3. 行为契约

### [INV-1] 绘制阶段 QFontMetricsF 必须带 p->device()

```cpp
// ✓ 正确
QFontMetricsF fm(monoFont, p->device());
QFontMetricsF defaultFm(font, p->device());
```

### [INV-2] 递归子块必须计算绝对坐标

遵守 `横切关注点/20-坐标系统.md` 的 INV-1 和 INV-2：

```cpp
for (const auto& child : block.children) {
    qreal childAbsX = absX + child.bounds.x();
    qreal childAbsY = absY + child.bounds.y();
    paintBlock(p, child, childAbsX, childAbsY, ctx);
}
```

### [INV-3] 反引号 padding 必须按字体度量比例

```cpp
// ✗ 硬编码
qreal segW = fm.horizontalAdvance(run.text) + 4;

// ✓ 按比例
qreal hPad = fm.height() * 0.2;
qreal segW = fm.horizontalAdvance(run.text) + hPad * 2;
```

### [INV-4] 绘制顺序

对每个块：
```
1. 背景色
2. 边框
3. 内容标记高亮（marking）
4. 选区高亮
5. 文本/代码内容
6. 光标（preview 无光标，此步跳过）
```

**标记高亮必须先于选区高亮**，否则选区会覆盖标记色。

### [INV-5] 只绘制可视区内的块

使用 viewport 裁剪，跳过不可见块（性能）。

### [INV-6] 容器块绘制完自身后再绘制子块

先绘制容器块的背景/边框/装饰（如引用条、序号），再递归子块。

### [INV-7] 序号与内容使用同一 Y 坐标

列表序号和对应列表项的内容必须用同一个 `itemAbsY`：

```cpp
qreal itemAbsY = listAbsY + item.bounds.y();
drawNumber(p, numberText, itemAbsX, itemAbsY);          // ← 同一个 itemAbsY
paintBlock(p, item, itemAbsX + indent, itemAbsY, ctx);  // ← 同一个 itemAbsY
```

### [INV-8] 禁止硬编码 QFont 字号

遵守 `横切关注点/80-字体系统.md` 的 INV-8。`PreviewPainter.cpp` 中**任何** `QFont` 对象都必须从 `PreviewLayout::baseFont()` / `PreviewLayout::monoFont()`（或通过 `LayoutBlock` / `RenderContext` 字段间接）派生，**禁止**出现 `QFont("<family>", <literal_size>)` 这种把字号常量写死的构造。

**强制规则：**

```cpp
// ✗ 禁止（无论是代码块、表格、图片占位符、列表序号、图标字符，全部禁止）
QFont monoFont("Consolas", 9);
QFont cellFont("Segoe UI", 12);
QFont iconFont("Segoe UI", 16);
QFont baseFont("Segoe UI", 12);

// ✓ 允许：从 layout 的 getter 读取基础字体后按比例派生
const QFont& base = m_layout->baseFont();   // 通过 ctx 拿到 layout 指针
const QFont& mono = m_layout->monoFont();

QFont cellFont = base;                        // 表格正文 == 正文
QFont monoFont = mono;                        // 代码块 == 等宽
QFont iconFont = base;
iconFont.setPointSizeF(base.pointSizeF() * 1.33);   // 图标按比例派生
QFont errorLabelFont = base;
errorLabelFont.setPointSizeF(base.pointSizeF() * 0.83);  // 小字标签按比例派生
```

**例外清单**（必须在代码中加注释 `// [INV-8 例外] 原因：…`）：

- 当前无例外。图标字符（`\u2717` / `\u25A3`）不算例外——它们仍是 `QFont`，字号必须随缩放变化。

### [INV-9] 绘制阶段不拥有字体数据

`PreviewPainter` 是纯粹的绘制执行者，所有字体信息必须来自：

1. `LayoutBlock::inlineRuns[i].font`（布局阶段写入）
2. `PreviewLayout::baseFont()` / `PreviewLayout::monoFont()` getter（通过 RenderContext 或构造注入）
3. `LayoutBlock` 上新增的专用字段（例如 `LayoutBlock::cellFont` 用于表头加粗；`LayoutBlock::monoFont` 用于代码块）

绘制阶段**不得**独立计算字号、不得访问 `font_defaults::*`、不得读取 `m_fontSizeDelta`。

### [INV-10] 同一行的 inline run 必须共享统一基线

段落/标题/表格单元格内 `inlineRuns` 在绘制时，所有 run **必须使用同一 baseline Y** 来绘制文本。

```cpp
// ✓ 正确：统一基线
QFontMetricsF defaultFm(block.inlineRuns[0].font, p->device());
qreal lineAscent = defaultFm.ascent();   // 行主字体 ascent
qreal lineHeight = defaultFm.height() * 1.5;
// ...
for (const auto& run : block.inlineRuns) {
    QFontMetricsF fm(run.font, p->device());
    // baselineY 基于行主字体，不是 run 自身
    p->drawText(QPointF(curX, curY + lineAscent), seg);
    // 背景矩形仍用 curY + lineHeight
}

// ✗ 错误：每个 run 按自身 ascent 放置，小字号 run 会偏上
QFontMetricsF fm(run.font, p->device());
p->drawText(QPointF(curX, curY + fm.ascent()), seg);  // ← fm 是 run 自己的
```

**场景**：行内代码（``` `text` ```）通常把字号缩到周围正文的 90%。若用 run 自身 ascent 定位 y，小字号 run 的 baseline 会比主字体高，视觉上 code 浮在正文上方（底部不对齐）。

**作用域**：凡是循环绘制 `block.inlineRuns` 的地方（`paintInlineRuns` 及其同构代码）都必须遵守。

## 4. 接口

```cpp
class PreviewPainter {
public:
    void paint(
        QPainter* p,
        const LayoutBlock& root,
        const RenderContext& ctx  // theme, selection, highlights, viewport
    );

private:
    void paintBlock(
        QPainter* p,
        const LayoutBlock& block,
        qreal absX,
        qreal absY,
        const RenderContext& ctx
    );

    // 按块类型分发
    void paintParagraph(...);
    void paintHeading(...);
    void paintCodeBlock(...);
    void paintList(...);
    void paintBlockQuote(...);
    void paintTable(...);
    void paintInlineCode(...);  // 反引号
};
```

## 5. 核心算法

### 递归绘制骨架

```
paintBlock(p, block, absX, absY, ctx):
    if not inViewport(block, absX, absY):
        return

    // 1. 背景 + 边框
    drawBackground(p, block, absX, absY, ctx)
    drawBorder(p, block, absX, absY, ctx)

    // 2. 标记高亮（marking）
    drawMarkings(p, block, absX, absY, ctx.highlights)

    // 3. 选区高亮
    drawSelection(p, block, absX, absY, ctx.selection)

    // 4. 内容（文本 / 代码 / 装饰）
    switch (block.type):
        case Paragraph:  paintParagraph(...)
        case Heading:    paintHeading(...)
        case CodeBlock:  paintCodeBlock(...)
        case List:       paintList(...)
        case BlockQuote: paintBlockQuote(...)
        case Table:      paintTable(...)

    // 5. 递归子块
    for each child in block.children:
        childAbsX = absX + child.bounds.x()
        childAbsY = absY + child.bounds.y()
        paintBlock(p, child, childAbsX, childAbsY, ctx)
```

### 反引号渲染

```cpp
void PreviewPainter::paintInlineCode(QPainter* p, const Run& run, qreal x, qreal y) {
    QFontMetricsF fm(run.font, p->device());
    qreal hPad = fm.height() * 0.2;
    qreal segW = fm.horizontalAdvance(run.text) + hPad * 2;

    p->fillRect(QRectF(x, y, segW, fm.height()), ctx.theme.inlineCodeBackground);
    p->setPen(ctx.theme.inlineCodeForeground);
    p->drawText(QPointF(x + hPad, y + fm.ascent()), run.text);
}
```

## 6. 性能预算

- 绘制一屏内容 < 16 ms（60fps）
- 反引号绘制 < 0.1 ms

## 7. 验收条件

- **[T-1]** 容器块（List / BlockQuote / Table）的子块位置正确（序号与内容对齐）
- **[T-2]** 反引号在 1x/1.25x/1.5x DPI 下 padding 比例一致
- **[T-3]** 标记高亮不被选区覆盖
- **[T-4]** 可视区外的块不被绘制（用计数探针验证）
- **[T-5]** `grep -rn "QFontMetricsF(" src/preview/PreviewPainter.cpp` 所有调用都带 `p->device()`
- **[T-6]** `grep -rn "paintBlock.*absX.*absY" src/preview/PreviewPainter.cpp` 的递归调用都有 `childAbsX`/`childAbsY` 计算
- **[T-7]** `grep -En 'QFont[[:space:]]*\("[^"]+"[[:space:]]*,[[:space:]]*[0-9]' src/preview/PreviewPainter.cpp` 必须返回 0 行
- **[T-8]** Ctrl+滚轮缩放 4 次向上后，以下元素均按 delta 放大：代码块文本、表格单元格、表头加粗、列表序号/圆点、图片失败的 X 图标、错误标签、文件名标签、已加载图片的占位图标、文件名、尺寸文字、未加载图片的占位图标。用逐元素像素对比验证
- **[T-9]** `grep -En 'font_defaults::|m_fontSizeDelta' src/preview/PreviewPainter.cpp` 必须返回 0 行（painter 不感知全局字号状态）
- **[T-10]** 行内代码（反引号包围片段，字号为正文的 90%）与周围正文共享同一基线：目视段落 `前文 \`code\` 后文` 时 code 的字形底部与前后文字底部对齐，不浮在上方

## 8. 已知陷阱

### 8.1 QFontMetricsF 缺 device

历史上多处 `QFontMetricsF(font)` 导致布局与绘制不一致。解决方案：所有调用都加 `p->device()`。

### 8.2 容器块子块 X 坐标（commit 4073b22）

见 `横切关注点/20-坐标系统.md` 第 8.1 节。`BlockQuote` 和 `List` 递归时必须加 `child.bounds.x()`，Table 早已正确。

### 8.3 标记与选区绘制顺序

选区画在标记之后会覆盖标记色，用户看不到标记。修复：`drawMarkings` 在 `drawSelection` 之前。

### 8.4 反引号硬编码 padding

早期 `+4` 在 1.5x 屏视觉太紧。改为 `fm.height() * 0.2`。

### 8.5 硬编码 QFont 字号导致缩放失效（2026-04-13 发现）

`PreviewPainter.cpp` 历史上散落着 11 处 `QFont("Segoe UI", <N>)` / `QFont("Consolas", 9)` 硬编码，覆盖了代码块、表格、列表序号、图片占位符及其装饰文字。用户 Ctrl+滚轮后 `MainWindow::applyFontSize()` 只更新了 `PreviewLayout::m_baseFont`，这些硬编码字号完全绕过。

**完整清单**见 `横切关注点/80-字体系统.md` §8.6 表格。

**修复原则**（由 INV-8/INV-9 强制）：

1. 绘制阶段不创建带字号字面量的 QFont
2. 从 `PreviewLayout` 的 getter 或 `LayoutBlock` 字段读取字体
3. 装饰元素（图标、小标签）通过 `base.pointSizeF() * <比例>` 派生，而不是独立字号

### 8.6 inline run 按自身 ascent 定位导致基线漂移（2026-04-14 发现）

`paintInlineRuns` 旧版本对每个 run 都用 `fm.ascent()`（`fm` 基于 run 自己的 font）作为 drawText 的 y 偏移。对于行内代码（字号 = 正文 * 0.9），小字号 run 的 ascent 小于正文，baseline 被抬高 → 视觉上 code 浮于正文上方。

修复：见 §3 INV-10，以行主字体（`block.inlineRuns[0].font`）的 ascent 作为该行统一基线，所有 run 共享。
