# 编辑区滚动稳定（编辑后不跳动）

## 1. 目的

编辑区采用按需（视口剪裁 A7）懒布局：未进入视口的行返回**估算行高**
（`defaultLineHeight * lineSpacingFactor`），进入视口或被预加载时才精算真实高度。
wrap 模式下一行可能换成多视觉行，真实高度 > 估算。本规范约束：**编辑内容后页面
不得连续跳动**。

## 2. 行为契约

### [INV-EDIT-SCROLL-ANCHOR] 编辑与预加载期间锚定首个可见行

任何会改变行高估算↔精算状态的操作，必须保持**首个可见行在视口中的屏幕位置不变**：

1. **编辑触发的全量 rebuild**（`onTextChanged`）：rebuild 会把所有行重置为估算高度，
   若编辑前视口上方的行已精算（真实高度 ≠ 估算），Y 缓存随之变化 → 同一 `scrollY`
   映射到不同行 → 跳动。修复：rebuild 前记录首个可见行相对视口顶的屏幕偏移，rebuild +
   updateScrollBars 后用该偏移恢复 `scrollY`。
2. **空闲预加载精算视口上方行**（`m_idlePreloadTimer`）：精算上方行的真实高度会抬高
   Y 缓存 → 视口内容下移。修复：所有影响锚点的行（视口上方 + 首个可见行）在一个 tick
   内批量精算，只做一次 scroll 补偿。后续行（视口内及下方）逐行精算不做补偿。

### [INV-EDIT-SCROLL-DEBOUNCE] 编辑后防抖启动预加载

`onTextChanged` 不直接启动 `m_idlePreloadTimer`，而是启动 300ms 单次防抖定时器
`m_preloadDebounce`。编辑过程中连续文本变化持续重置防抖 → 预加载完全不启动 → 零滚动。
编辑停止 300ms 后防抖到期再启动预加载。

历史教训：

- （2026-06-15）编辑后"页面内容跳动好几下才停"——根因是逐行精算视口上方行时每次都改
  Y 缓存却不补偿滚动位置。
- （2026-06-24）逐行补偿虽消除了跳动，但每 50ms 补偿一次导致持续约 5 秒的"微滚"。
  修复：防抖启动 + 批量预加载 + 单次补偿。

例外：typewriter（打字机）模式由 `ensureCursorVisible` 居中接管，不做锚定（否则双重调整）。

实现：`verticalScrollBar()->blockSignals(true/false)` 包裹补偿性 `setValue`，避免触发
`scrollContentsBy` 重入再次 rebuild。

## 3. 接口

```cpp
// EditorWidget
void onTextChanged(int offset, int removedLen, int addedLen);  // rebuild + 锚定恢复
int firstVisibleLine() const;
qreal scrollY() const;
// EditorLayout::lineY(line) 返回该行文档 Y（基于当前 估算/精算 混合状态）
```

## 4. 验收条件

- **[T-EDIT-SCROLL-1]**（手动）wrap 模式下打开较长文档，向下滚动若干屏，在视口中部
  输入字符 / 回车 / 删除：视口内容**一次到位**，不出现连续上下跳动。
- **[T-EDIT-SCROLL-2]**（手动）关闭预览（排除滚动同步反馈）后重复 T-EDIT-SCROLL-1，
  确认跳动消失（隔离编辑器自身路径）。
- **[T-EDIT-SCROLL-3]**（手动）在长文档视口中部快速输入 10 个以上字符：编辑过程中
  视口不微滚。
- **[T-EDIT-SCROLL-4]**（手动）编辑停止后观察 1 秒：最多一次补偿后视口稳定不滚动。

> 说明：跳动和微滚是多帧时序 + 视觉现象，依赖真人观察滚动稳定性，难以用离屏单测
> 忠实复现；按 CLAUDE.md 黄金法则 #8 例外条款采用手动验证，并在此 Spec 固化 INV
> 与复现步骤。
