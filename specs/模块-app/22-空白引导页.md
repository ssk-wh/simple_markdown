# 空白引导页（WelcomePanel）

## 1. 目的

应用启动且没有需要显示的 Tab（无会话恢复、无命令行文件）时，**不再自动创建一个 `Untitled` 空白文件标签**，而是显示一个**空白引导面板**（WelcomePanel），引导用户主动选择动作（打开文件 / 打开文件夹 / 新建文件）。

把"空白 Tab"作为强制初始状态会污染会话恢复语义、增加用户关闭无意义文件的负担，且首次启动看到一个空白编辑器对新用户没有任何引导价值。

## 2. 范围

涉及：
- 启动时无可恢复内容的初始状态（restoreSession 末尾兜底）
- 关闭最后一个 Tab 后的回退状态（onCloseTab 兜底）
- Tab 栏右侧 "+" 按钮的显隐与可点击性
- WelcomePanel 自身的 UI（文案、按钮、主题响应、i18n）

不涉及：会话恢复中实际有 saved tab 的路径（行为不变）。

## 3. 不变量

### [INV-EMPTY-NO-AUTO-CREATE]
当启动后会话恢复结果为空（无 saved tabs 且无命令行 requestedFile），**禁止**自动创建 `Untitled` Tab。`MainWindow::restoreSession` 末尾的兜底从 `newTab()` 改为切换到 WelcomePanel。

### [INV-EMPTY-CLOSE-LAST]
关闭最后一个 Tab 后，**禁止**自动新建 `Untitled` Tab。`MainWindow::onCloseTab` 中 `if (m_tabs.isEmpty()) newTab();` 改为切换到 WelcomePanel。

### [INV-EMPTY-WELCOME-MUTUAL]
"WelcomePanel 可见" 与 "m_contentStack 可见" 互斥：
- `m_tabs.isEmpty()` → WelcomePanel 可见、m_contentStack 隐藏
- `!m_tabs.isEmpty()` → WelcomePanel 隐藏、m_contentStack 可见

由统一函数 `MainWindow::updateEmptyState()` 维护，所有改变 `m_tabs` 大小的代码路径必须最终调用此函数（addPage / removePage / restoreSession 兜底）。

### [INV-EMPTY-TOC-HIDE]
当 `m_tabs.isEmpty()` 时，右侧 TocPanel（含目录条目和文档信息卡片）必须 `hide()`——
没有打开的页面就没有内容支撑这个面板，挂在窗口右侧的空目录 / 空文档信息卡片
对用户是无意义的 chrome。任何 Tab 出现时（`m_tabs` 从空变非空）必须 `show()`。

由 `updateEmptyState()` 与 INV-EMPTY-WELCOME-MUTUAL 同位置维护，行为：
```cpp
if (m_tocPanel && !m_focusMode) m_tocPanel->setVisible(!empty);
```

**演示模式例外**：F11 演示模式下 TocPanel 显隐由 `m_savedTocVisible` 机制
独立控制（enterFocusMode 强制 hide / exitFocusMode 还原）。`!m_focusMode`
守卫保证两套机制互不污染。按 T-9，演示模式与空 Tab 不会同时成立。

### [INV-EMPTY-PLUS-VISIBLE]
Tab 栏右侧 "+" 按钮**始终可见且始终可点击**，无论 `m_tabs` 是否为空。点击 "+" 触发 `onNewFile()`，创建一个 `Untitled` Tab；这是用户从 WelcomePanel 状态显式进入编辑状态的入口之一。

### [INV-EMPTY-OPEN-EXITS]
任何会让 `m_tabs.size() > 0` 的操作（点 "+"、菜单"打开文件"、拖拽文件、文件夹面板单击、最近打开）必须在新增 Tab 后**自动退出 WelcomePanel 状态**——通过 `updateEmptyState()` 自然达成（addPage 末尾调用）。

### [INV-EMPTY-I18N]
WelcomePanel 中所有用户可见字符串**必须**用 `tr()` 包裹。`zh_CN.ts` 和 `en_US.ts` 必须包含对应翻译条目。

### [INV-EMPTY-THEME]
WelcomePanel 的背景、前景、按钮、accent 颜色**必须**派生自 `Theme`，禁止硬编码。`MainWindow::applyTheme` 中调用 `m_welcomePanel->setTheme(theme)` 同步。

### [INV-EMPTY-NO-CURRENT-TAB-DEREF]
`m_tabs.isEmpty()` 是合法状态，所有引用 `currentTab()` / `m_tabs[i]` 的代码必须先判空。Tab-依赖的 UI（保存状态栏、字数统计、TOC）在空状态下应**显示空/默认值或隐藏**而非崩溃。

## 4. 接口

```cpp
// src/app/WelcomePanel.h
class WelcomePanel : public QWidget {
    Q_OBJECT
public:
    explicit WelcomePanel(QWidget* parent = nullptr);
    void setTheme(const Theme& theme);  // 主题响应

signals:
    void openFileClicked();    // 触发 MainWindow::onOpenFile
    void openFolderClicked();  // 触发 MainWindow::onOpenFolder
    void newFileClicked();     // 触发 MainWindow::onNewFile

protected:
    void paintEvent(QPaintEvent*) override;  // 主题色背景

private:
    QPushButton* m_openFileBtn;
    QPushButton* m_openFolderBtn;
    QPushButton* m_newFileBtn;
    QLabel* m_titleLabel;
    QLabel* m_subtitleLabel;
    QLabel* m_hintLabel;
    Theme m_theme;
};
```

## 6. 集成点

`MainWindow`：
- 新增成员 `WelcomePanel* m_welcomePanel`
- 在 ctor 中创建并 `m_centralLayout->addWidget(m_welcomePanel, 1)`，初始 `hide()`
- 信号转 slot：openFileClicked → onOpenFile，openFolderClicked → onOpenFolder，newFileClicked → onNewFile
- 新增 helper `updateEmptyState()`：根据 `m_tabs.isEmpty()` 切换 WelcomePanel/contentStack 可见性
- 修改：
  - `restoreSession` 末尾从 `newTab(); reapplyTheme();` 改为 `updateEmptyState(); reapplyTheme();`
  - `onCloseTab` 中 `if (m_tabs.isEmpty()) newTab();` 改为 `updateEmptyState();`
  - `addPage` 和 `removePage` 末尾调 `updateEmptyState()`（防御）
- `applyTheme` 中调 `m_welcomePanel->setTheme(theme)`

## 7. 验收（T 编号）

- [T-1] 清空 QSettings 首次启动 → 无 Tab + WelcomePanel 可见 + tabBar 仅显示 "+" 按钮
- [T-2] 在 WelcomePanel 状态点 "+" 按钮 → 创建 Untitled Tab + WelcomePanel 隐藏 + 编辑区可见
- [T-3] 在 WelcomePanel 状态点"打开文件"按钮 → 弹出文件对话框，选文件后打开（Tab 出现 + WelcomePanel 隐藏）
- [T-4] 在 WelcomePanel 状态点"打开文件夹"按钮 → 弹出目录对话框，选目录后侧栏出现（Tab 仍空，WelcomePanel 仍可见，等用户单击文件再退出）
- [T-5] 有 saved session 启动 → 正常恢复 Tab，WelcomePanel 不显示
- [T-6] 关闭最后一个 Tab → WelcomePanel 重现
- [T-7] 切换主题 → WelcomePanel 配色跟随
- [T-8] 切换语言（中文 ↔ 英文）→ WelcomePanel 文案随之变化
- [T-9] 演示模式（F11）下 WelcomePanel 可见性遵循"无 Tab → 不进入演示模式"或"有 Tab → 正常切演示"
- [T-10] 关闭最后一个 Tab → m_tocPanel 立即隐藏，仅 WelcomePanel 占据中央 + tabBar 区域，右侧不再有空目录卡片
- [T-11] 在空状态点 "+" 或"打开文件" → Tab 出现的同时 m_tocPanel 重新显示并填充对应目录

## 8. 已知陷阱

- **`currentTab()` 解引用**：原代码假设 m_tabs 至少 1 个，多个槽函数（保存、字数统计、自动保存定时器）会取 `currentTab()->editor`。本规范让 `m_tabs.isEmpty()` 成为合法状态后，所有解引用必须 nullptr-check
- **状态栏空状态**：编辑器/字数统计 / 保存状态在 WelcomePanel 状态应显示空/dash，不可崩溃
- **会话恢复时机**：`updateEmptyState()` 必须在 `restoreSession` 处理完所有路径后调用，避免中间态闪烁
- **+按钮不可禁用**：即使在空状态，"+" 按钮必须可点击——它是用户进入编辑状态的主入口之一
