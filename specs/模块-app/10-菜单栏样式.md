# 菜单栏样式

## 1. 目的

定义主窗口菜单栏（QMenuBar）与下拉菜单（QMenu）的视觉规范，保证：

- checkable 菜单项的勾选符号（✓ / radio dot）与文本之间有足够可辨识的间距，不拥挤
- 同一菜单内所有项的文本起始 X 坐标列对齐，不因为勾选符号的有无抖动
- 深色 / 浅色主题下 padding、margin、indicator 比例一致
- 样式不硬编码像素值脱离字体度量，保证在高 DPI 屏和字号变化时保持视觉比例

本 Spec 只聚焦样式层面，菜单项的业务逻辑见 `02-主窗口与多Tab.md`、主题切换见 `横切关注点/30-主题系统.md`。

## 2. 输入 / 输出

- **输入**：
  - 当前主题（`Theme::isDark`）
  - 系统字体（QApplication::font）
  - 平台默认 QStyle（Windows: Fusion / WindowsVista；Linux: Fusion / KDE / GNOME）
- **输出**：
  - 通过 `MainWindow::setStyleSheet` 注入的 QMenuBar / QMenu 样式规则
  - 视觉效果：菜单项对齐、间距合理、勾选符号清晰可见且与文字分离

## 3. 行为契约（不变量）

### [INV-1] 勾选符号与文本必须有可辨识间距

所有 checkable 菜单项（radio 和 checkbox 两种）的勾选符号与其后的文字之间，必须留有 **不小于 1 个字符宽度**（以系统菜单字体的平均字符宽度为基准）的视觉间距。禁止出现勾选符号紧贴文字首字的情况。

具体 QSS 规则要求：`QMenu::item` 的 `padding-left` 必须 ≥ `QMenu::indicator` 的 `width` + **至少 1 个字符宽**的额外间距。

> ⚠️ [2026-06-15 修订] **禁止给 `QMenu::indicator` 设 `margin-right`**。Qt（5.12 stylesheet）
> 把 indicator 内联绘制在文字前，`margin-right` 会把 **checkable 项的文字额外推右**，而
> 非 checkable 项不受影响 → 违反 INV-2（文字左缘错位，用户 2026-06-15 报告）。✓ 与文字的
> 间距应由 indicator 列宽（width）内部留白提供，而非 margin。当前实现：`QMenu::indicator { width: 16px; height: 16px; }`（无 margin）。

### [INV-2] 文本 X 起点列对齐

同一 QMenu 内，不论菜单项是否 checkable、是否带图标，所有文本的起始 X 坐标必须一致。用户从上往下看菜单时，文字必须形成一条直线。

实现原则：`QMenu::item` 的 `padding-left` 应当为所有菜单项预留 indicator / icon 占位空间，即使该项没有 checkable 状态。

### [INV-3] 深浅主题视觉对等

深色主题和浅色主题的 QMenuBar / QMenu 样式规则在以下维度必须完全相同：

- `QMenu::item` 的 padding（四边）
- `QMenu::indicator` 的 width / height / margin
- `QMenu::separator` 的 height / margin
- `QMenuBar::item` 的 padding

两种主题只允许在**颜色**（background / color / border-color）上有差异，结构度量必须一致。

### [INV-4] 不得硬编码像素值脱离字体度量

菜单样式中的 padding、margin、indicator 尺寸**不得**直接写死像素字面量而不留推导依据。允许两种做法：

1. 基于 `QFontMetricsF(menuFont, device).height()` 或 `horizontalAdvance("M")` 在运行时派生像素值，然后格式化进 QSS 字符串
2. 硬编码像素值但**注释说明**该数值对应字体度量的哪个比例（例如 `// indicator 宽 = 菜单字号 * 1.0`）

**反面例子**：`"QMenu::item { padding: 6px 24px 6px 12px; }"`，既没有说明 6 / 24 / 12 的出处，也没有在字号变化时自适应。

### [INV-5] 跨 Qt 样式一致

样式规则必须在以下 QStyle 下视觉表现一致（文字 X 起点误差 ≤ 2 px，indicator 大小相同）：

- Windows: Fusion、WindowsVista
- Linux: Fusion（项目默认）、KDE (Breeze)、GNOME (Adwaita)

如果某个 QStyle 行为异常（例如 indicator 不走 QSS 规则），必须在 `QApplication::setStyle("Fusion")` 级别强制统一，或在 Spec 中记录例外。

### [INV-6] 样式表作用域最小化

菜单样式通过 `MainWindow::setStyleSheet` 注入时，其 QMenu / QMenu::item 选择器会同时作用于：

- 菜单栏下拉菜单
- 所有上下文（右键）菜单
- Tab 右键菜单
- 预览区右键菜单

不允许出现「只改菜单栏但连带破坏右键菜单」的情况。改动菜单样式必须同时回归所有右键菜单场景。

## 4. 接口（API 签名）

```cpp
class MainWindow : public QMainWindow {
private:
    // 应用主题时同步更新菜单样式（当前实现入口）
    void applyTheme(const Theme& theme);

    // 可选：将来抽出专门函数，根据字体度量生成菜单 QSS 片段
    // QString buildMenuStyleSheet(const Theme& theme, const QFont& menuFont, QPaintDevice* device) const;
};
```

关键 QSS 选择器：

- `QMenuBar`、`QMenuBar::item`、`QMenuBar::item:selected`
- `QMenu`、`QMenu::item`、`QMenu::item:selected`、`QMenu::item:disabled`
- `QMenu::indicator`、`QMenu::indicator:checked`、`QMenu::indicator:non-exclusive`、`QMenu::indicator:exclusive`
- `QMenu::separator`

## 5. 核心算法

### 度量派生（建议实现）

```
menuFont    = QApplication::font("QMenu")  // 系统菜单字体
fm          = QFontMetricsF(menuFont, device)
charW       = fm.horizontalAdvance('M')     // 平均字符宽
lineH       = fm.height()                   // 行高

indicatorW  = round(lineH * 0.8)            // 勾选符号宽，约 0.8 倍行高
indicatorMR = round(charW * 0.6)            // indicator 右 margin，约 0.6 字符宽
padLeft     = indicatorW + indicatorMR + charW  // 左 padding = indicator 占位 + 至少 1 字符间距
padRight    = round(charW * 2)              // 右 padding 给快捷键留位
padVert     = round(lineH * 0.3)            // 上下 padding = 0.3 倍行高
```

### 样式表注入顺序

`MainWindow::applyTheme` 流程：

1. 计算菜单度量（基于当前字体）
2. 格式化 QSS 字符串（深色 / 浅色共用同一结构，只换颜色常量）
3. `setStyleSheet(qss)` 注入
4. 由于 Qt 样式表会级联到所有子 QMenu，无需逐个菜单设置

### 子菜单和右键菜单

QMenu 样式通过全局选择器（不带对象名）应用，自动覆盖：

- 主菜单栏（File / Edit / View / Settings / Help）
- 子菜单（View → Line Spacing、Settings → Language）
- 右键菜单（编辑器、预览区、TabBar）

## 6. 性能预算

- `applyTheme` 内菜单 QSS 生成 + 注入：< 5 ms
- 菜单打开到显示首帧：< 50 ms（用户感知即时）
- 菜单切换主题不触发重新布局：必须在同一 setStyleSheet 内原子完成

## 8. 已知陷阱

### 8.1 `QMenu::item` padding-left 小于 indicator 总宽度

当前实现（MainWindow.cpp 约 721 / 773 行）：

```cpp
"QMenu::item { padding: 6px 24px 6px 12px; }"
"QMenu::indicator { width: 16px; height: 16px; margin-right: 8px; }"
```

- `QMenu::item` 左 padding = **12 px**
- `QMenu::indicator` 占位 = width 16 px + margin-right 8 px = **24 px**

`padding-left (12)` < `indicator 占位 (24)`，indicator 会被绘制在 padding 区内但文字仍从 padding-left 之后起步，在某些 Qt 样式下 indicator 和文字会重叠或紧贴。典型症状就是"跟随系统"前的 ✓ 几乎贴着"跟"字。

**修复方向**：`padding-left` 必须 ≥ `indicator.width + indicator.margin-right + 一个字符宽`。

### 8.2 硬编码像素值在高 DPI 下失真

6 / 24 / 12 / 16 / 8 这些数值在 1x DPI 屏上看似合理，但高 DPI 屏上 Qt 会对 QSS 的 px 值做缩放（取决于 Qt 版本和 `QT_ENABLE_HIGHDPI_SCALING`），容易造成 indicator 和文字的比例失调。

### 8.3 非 checkable 菜单项文字起点抖动

Fusion 样式下，如果某菜单同时包含 checkable 和非 checkable 项，且 `QMenu::item` 的 padding-left 不足以容纳 indicator，非 checkable 项的文字会向左靠（因为不预留 indicator 空间），而 checkable 项的文字起点被 indicator 推到右边，造成同一菜单文字左边缘参差不齐。

**修复方向**：无论是否 checkable，所有 QMenu::item 的 padding-left 必须相同且足够大。

### 8.4 一份样式表同时影响右键菜单

`MainWindow::setStyleSheet` 注入的 `QMenu` 选择器会级联到所有子 QMenu，包括右键菜单。改菜单样式时必须回归：编辑器右键、预览区右键、TabBar 右键。历史上曾出现过"只调了菜单栏但把预览右键菜单边框改坏"的情况。

### 8.5 深浅主题两份字符串漂移

当前深色和浅色主题各自维护一份 QSS 字符串，两份字符串的 padding / indicator 参数**必须手动保持同步**。如果某次只改了一份，两主题会出现结构差异（深色 padding 6px / 浅色 padding 8px 这种）。

**修复方向**：抽出共用的结构常量，只让颜色部分按主题切换。或者改为运行时根据字体度量派生。

### 8.6 QProxyStyle 覆盖 QSS 的陷阱

如果未来为了进一步控制菜单使用 `QProxyStyle` 重写 `pixelMetric(PM_SmallIconSize)` / `PM_MenuHMargin` / `PM_MenuVMargin`，要注意 QSS 的 `QMenu::indicator { width: N }` 会**覆盖** QProxyStyle 的 `PM_IndicatorWidth`，不要两套机制同时用。建议选一套：要么纯 QSS，要么纯 QProxyStyle。
