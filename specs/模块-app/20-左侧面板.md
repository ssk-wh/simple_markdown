# 左侧面板

## 1. 目的

管理左侧面板（`m_leftPaneSplitter`）的组成、显隐规则和宽度策略。左侧面板包含文件夹资源管理器和可选的侧边 Tab 栏，是用户浏览项目文件和切换标签页的核心区域。

## 2. 输入 / 输出

- 输入：用户打开/关闭文件夹、Ctrl+B 快捷键、Tab 栏位置菜单切换、演示模式切换
- 输出：左侧面板的可见性和宽度

## 3. 行为契约（不变量）

### 显隐规则

- [INV-LP-FOLDER-ONLY] 左侧面板**仅在有打开的文件夹时显示**，无论 Tab 栏位置设置如何。没有打开任何文件夹时，左侧面板始终隐藏
- [INV-LP-FOCUS-HIDE] 演示模式（F11）下左侧面板始终隐藏，优先级最高
- [INV-LP-CTRLB] Ctrl+B 手动隐藏侧边栏，状态为会话级（不持久化），重启后默认显示
- [INV-LP-AUTO-SHOW] 以下操作自动取消 Ctrl+B 的隐藏状态，恢复显示：
  - 打开新文件夹（菜单、拖拽、最近文件列表）
  - 切换 Tab 栏位置菜单项（Show All / Top Only / Side Only / Hide All）

### 判断优先级

```
演示模式？ → 隐藏（最高优先级）
Ctrl+B 手动隐藏？ → 隐藏
有打开的文件夹？ → 显示
否则 → 隐藏
```

### 宽度策略

- [INV-LP-WIDTH-DEFAULT] 默认宽度 = **应用所在屏幕宽度 / 8**（不是窗口宽度，因为窗口在最大化几何过渡期 `m_mainSplitter->width()` 可能仍为初始 1280，导致默认值或夹紧上下限错位）
- [INV-LP-WIDTH-PERSIST] 用户调整后的宽度通过 `QSettings("view/leftPanelWidth")` 持久化；启动时若该键存在，**忠实还原 saved 值**（不做 1/8~1/5 夹紧），否则使用 INV-LP-WIDTH-DEFAULT
- [INV-LP-WIDTH-RESTORE] 仅当 `sizes[0] <= 0`（即面板被隐藏/挤压到 0 宽度）时，从隐藏→可见转换时补充宽度（saved 优先，否则 INV-LP-WIDTH-DEFAULT）
- [INV-LP-WIDTH-USER] 用户拖拽 splitter 手动调整的宽度不受任何自动修正覆盖

### Tab 栏位置

- [INV-LP-TAB-DEFAULT] 默认 Tab 栏位置为"全部显示"（侧边 + 顶部同时显示）
- [INV-LP-TAB-SIDE] 侧边 Tab 栏（`m_sideTabBar`）作为 `m_leftPaneSplitter` 的子 widget，位于文件夹面板下方
- [INV-LP-TAB-RATIO] 侧边模式下文件夹面板与 Tab 栏的默认高度比为 2:1

## 4. 接口（API 签名）

```cpp

// Tab 栏位置切换
void MainWindow::setTabBarPosition(bool onSide, bool hideTopBar = true);
```

### 关键成员


## 5. 核心算法

### updateLeftPaneVisibility 逻辑

```
if 演示模式 → hide, return
if m_sidebarHidden → hide, return
if 有打开的文件夹 → show（并修正宽度）
else → hide
```

### 宽度修正（隐藏→可见转换时）

```
if shouldShow && wasHidden && sizes[0] <= 0:
    leftW = saved("view/leftPanelWidth") if exists else screenW / 8
    sizes[0] = leftW
    sizes[1] = totalW - leftW - sizes[2]
```

### showEvent 初始化（统一逻辑：saved 优先，否则 screen/8）

```
screenW = appScreen.geometry.width    # 应用所在屏幕宽度，不是 splitter 宽度
defaultPanelW = screenW / 8

savedLeftW = QSettings("view/leftPanelWidth")  # -1 表示无
savedTocW  = QSettings("view/tocPanelWidth")   # -1 表示无
leftShouldShow = (rootPaths != ∅ && !sidebarHidden)

leftW = savedLeftW if savedLeftW > 0 else (defaultPanelW if leftShouldShow else 0)
tocW  = savedTocW  if savedTocW > 0  else defaultPanelW

# 若有 savedTocW：标记为"用户意图"，避免 TOC content-aware 自适应覆盖（INV-TOC-WIDTH-USER-OVERRIDE）
if savedTocW > 0: m_userDraggedToc = true

setSizes({leftW, totalW - leftW - tocW, tocW})
```

## 6. 性能预算

- 显隐切换：< 16ms（一帧内完成）
- 无额外内存分配（仅修改 QSplitter sizes）

## 7. 验收条件（测试用例清单）

- [T-1] 全新安装（无 QSettings）启动，左侧面板不显示（无文件夹打开）
- [T-2] 打开文件夹后，左侧面板出现，宽度在窗口宽度 1/8 ~ 1/6 之间
- [T-3] Ctrl+B 隐藏侧边栏，再按 Ctrl+B 恢复
- [T-4] Ctrl+B 隐藏后，打开新文件夹自动恢复显示
- [T-5] Ctrl+B 隐藏后，切换 Tab 栏位置菜单项自动恢复显示
- [T-6] 演示模式下左侧面板隐藏，退出演示模式后恢复
- [T-7] 关闭所有文件夹后左侧面板自动隐藏
- [T-8] 重启应用后 Ctrl+B 隐藏状态不保留（默认显示）
- [T-9] 用户拖拽调整宽度后，**关闭重启**忠实恢复同样宽度（不被夹紧、不被自动修正覆盖）
- [T-10] 清空 `view/leftPanelWidth` 后启动，左侧面板宽度 ≈ 应用所在屏幕宽度 / 8（不依赖 `m_mainSplitter->width()` 的瞬时值）

## 8. 已知陷阱

- **showEvent 时机的 width 不准**：`setWindowState(Qt::WindowMaximized)` 在 ctor 中调用后，showEvent 触发时窗口几何转换可能尚未完成，`m_mainSplitter->width()` 仍可能是 ctor 中 `resize(1280, 800)` 设置的初始值。**禁止用 `m_mainSplitter->width()` 计算默认值或 minW/maxW 上下限**，否则用户在大屏（如 1920+）保存的 350 px 会被错误的 maxW=256 截断（INV-LP-WIDTH-DEFAULT 的设计动机）
- **setSizes 参数个数**：`m_mainSplitter` 有 3 个子 widget（左面板、内容、TOC），`setSizes` 必须传 3 个值，传 2 个会导致未定义行为
- **会话级 vs 持久化**：`m_sidebarHidden` 不写入 QSettings，避免用户"上次隐藏了侧边栏"后重启仍然隐藏造成困惑
- **TOC 内容自适应会覆盖 saved 值**：`TocPanel::preferredWidthChanged` 信号在文档加载时触发 `applyTocPreferredWidth`，若 `m_userDraggedToc=false` 会自动夹紧到 `screenW/8`，覆盖用户保存的 `view/tocPanelWidth`。修复方式：showEvent 中读到有效 `view/tocPanelWidth` 时强制 `m_userDraggedToc=true`
