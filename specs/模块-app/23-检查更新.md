# 检查更新（Check for Updates）

## 1. 目的

让用户得知有新版本发布，并便捷跳转到下载页自行下载安装。客户端侧对比**当前版本**
（编译期由 CHANGELOG 头条注入的 `APP_VERSION` → `QApplication::applicationVersion()`）与
**远端最新版本**（GitHub `releases/latest`，无 release 时回退 `tags`）。

## 2. 范围与非目标

- **做**：帮助菜单「检查更新…」手动触发、异步查询远端最新版本、语义化比较、有更新时提示并
  提供「打开下载页」。
- **不做**：启动时自动检查（2026-06-16 用户要求仅手动触发，避免每次启动联网）、自动下载安装包、
  静默安装、自动重启、"跳过此版本/稍后提醒"调度。自动下载涉及 exe 落地 + 提权安全风险，
  留作后续演进（见 INV-UPD-NO-AUTODOWNLOAD）。

## 3. 行为契约

### [INV-UPD-SEMVER] 语义化版本比较

`UpdateChecker::compareSemVer(a, b)` 按 `major.minor.patch` 三段整数逐级比较：
- 去掉前导 `v`/`V`；
- 截断首个 `-`（预发布）或 `+`（构建）后缀，只比三段数字；
- 缺位补 0（`"1.2"` == `"1.2.0"`）；非数字段按 0；
- `patch` 按**数值**比较（`1.2.10 > 1.2.9`，非字典序）。

返回 `>0`（a 新）/ `0`（相等）/ `<0`（a 旧）。`isUpdateAvailable(current, latest)` 当且仅当
`compareSemVer(latest, current) > 0` 为 true（相等不算更新）。

### [INV-UPD-MANUAL-ONLY] 仅手动检查，启动不自动联网

检查更新只在用户主动点击帮助菜单「检查更新…」时触发（`checkForUpdates(true)`）。应用启动时
**不**自动联网检查（2026-06-16 用户要求：避免每次启动联网，按需手动）。`UpdateChecker` 仍保留
`manual` 参数与 `manual=false` 的静默契约（见 INV-UPD-SILENT-AUTO）以便将来需要，但当前 UI
**不**触发任何自动检查，也不提供"启动时自动检查"开关。

### [INV-UPD-SILENT-AUTO] manual=false 路径静默（保留契约）

`checkForUpdates(manual)` 的 `manual=false` 路径下：网络失败 / 无网 / 超时 / 「已是最新」
**均不打扰用户**（不发 `checkFailed`/`upToDate`，UI 无弹窗）。只有 `manual=true`（用户主动
点「检查更新」）才在「已是最新」给信息框、失败给警告框；`updateAvailable`（发现新版本）两种
模式都提示。当前 UI 只调用 `manual=true`，此 INV 描述的是 API 契约（为将来可能恢复自动检查保留）。

### [INV-UPD-NO-AUTODOWNLOAD] 不自动下载/静默安装

发现新版本时**仅**提供「打开下载页」（`QDesktopServices::openUrl` 打开 release 页 / 仓库
releases 列表），由用户在浏览器自行下载安装。**禁止**在客户端自动下载安装包、禁止静默安装/
提权。此约束为安全底线，任何"自动下载"演进必须显式用户确认且不静默提权。

### [INV-UPD-SSL-GUARD] HTTPS 依赖 OpenSSL，缺失明确反馈

GitHub API 走 HTTPS，Qt5 的 `QSslSocket` 需要 **OpenSSL 1.1 x64 运行时**
（`libssl-1_1-x64.dll` + `libcrypto-1_1-x64.dll`）。`windeployqt` **不**部署 OpenSSL，故
`build_on_win.bat`（部署到 exe 同目录 + 镜像到 build/tests）与 `installer/collect_dist.py`
（打包进安装包）负责收集（从 Git mingw64 等 x64 OpenSSL 位置）。

`requestLatestRelease` 发请求前先 `QSslSocket::supportsSsl()` 预检：若不支持，手动检查时
`emit checkFailed`（提示"安全连接不可用，请确认已安装 OpenSSL"），**不静默挂起**。

历史 bug（2026-06-16 用户报告"点了没反应"）：首版未部署 OpenSSL → `supportsSsl()==false`
→ HTTPS 请求失败/挂起且无反馈。修复：部署 x64 OpenSSL（命令行 `curl` 自带 SSL 不受影响，
故误以为网络无问题）+ SSL 预检 + 超时。

### [INV-UPD-TIMEOUT] 请求超时兜底

Qt 5.12 无 `setTransferTimeout`，用 `QTimer`（15s）兜底：超时 `reply->abort()` → 触发
`onReplyFinished` 的 error 分支 → 手动检查时给「检查失败」反馈，绝不无限等待。

### [INV-UPD-API-FALLBACK] release 优先、tags 回退

优先 `GET /repos/{owner}/{repo}/releases/latest`（含 `tag_name` / `body` 更新说明 /
`html_url`）。该接口在仓库**尚无 release** 时返回 404 → 回退 `GET /repos/{owner}/{repo}/tags`
取首个 `name`（无更新说明，下载链接用仓库 releases 列表页）。请求必须带 `User-Agent` 头
（GitHub API 强制，否则 403）。

## 4. 接口

```cpp
class UpdateChecker : public QObject {
public:
    void checkForUpdates(bool manual);                          // 发起一次检查
    static int  compareSemVer(const QString& a, const QString& b);     // INV-UPD-SEMVER
    static bool isUpdateAvailable(const QString& cur, const QString& latest);
    static bool parseLatestRelease(const QByteArray& json,
                                   QString& tag, QString& notes, QString& url);  // INV-UPD-API-FALLBACK
signals:
    void updateAvailable(const QString& latestVersion, const QString& notes, const QString& url);
    void upToDate();                  // 仅 manual
    void checkFailed(const QString& message);  // 仅 manual
};
```

## 5. 验收条件

- **[T-UPD-1]** 基本大小比较 + `v` 前缀归一（`v1.2.3` == `1.2.3`）
- **[T-UPD-2]** 缺位补 0 + 逐段优先级 + `patch` 数值比较（`1.2.10 > 1.2.9`）
- **[T-UPD-3]** 预发布/构建后缀被忽略（`1.2.3-beta.1` == `1.2.3`）
- **[T-UPD-4]** `isUpdateAvailable`：latest 严格新于 current 才 true；空/非法版本按 0.0.0 不误判
- **[T-UPD-5]** JSON 解析兼容 releases（对象）与 tags（数组）两形态；非法/空 → 失败
- **[T-UPD-UI]**（手动验证）帮助菜单「检查更新…」可触发；有更新弹窗「打开下载页」跳转浏览器；
  已是最新弹「已是最新」；失败/无网/SSL 缺失弹错误提示——手动检查成功/失败/无更新均有反馈

## 6. i18n

所有用户可见字符串 `tr()` 包裹，已同步 `translations/simple_markdown_{zh_CN,en_US}.ts`
并 `lrelease`（INV-I18N-*）。涉及串：检查更新…/启动时自动检查更新/发现新版本/打开下载页/
稍后/您已是最新版本/检查更新失败/解析最新版本信息失败 等。

## 7. 安全与隐私

- 仅请求公开 GitHub API，不发送任何用户数据。
- 不自动下载/静默安装（INV-UPD-NO-AUTODOWNLOAD），避免静默提权风险。
- 打开的下载 URL 来自 GitHub API 返回的 `html_url`（github.com 域），不构造任意外链。
