# 自动保存

## 1. 目的

写 Markdown 时每次都按 `Ctrl+S` 才能落盘会打断心流。本特性引入**编辑停顿后静默写盘**：用户停下输入达到设定延迟后，应用自动把当前所有有磁盘路径的 Tab 写回原文件。

## 2. 输入 / 输出

- **输入**：
  - `Document::textChanged(int offset, int removed, int added)` 信号（来自任意 Tab 的编辑器）
  - 用户菜单选择：开关 `Settings → Auto Save` / 延迟 `Settings → Auto Save Delay` (1.5s/3s/5s)
  - 持久化设置 `autoSave/enabled` (bool)、`autoSave/delayMs` (int)
- **输出**：
  - 对**有磁盘路径**且**已修改**的 Tab：调用 `Document::saveToFile()` 写回原文件
  - 对未命名（filePath 为空）的 Tab：跳过，不弹文件对话框
  - Tab 标题 dirty 标记同步刷新
  - 失败时：状态栏右下显示瞬时红字"自动保存失败：xxx.md"，5s 后自动消失

## 3. 行为契约（不变量）

### [INV-1] 仅保存有磁盘路径的 Tab

`performAutoSave` 遍历所有 Tab，对每个 Tab：
- `Document::filePath().isEmpty()` 为真 → **跳过**，不弹任何对话框
- `Document::isModified()` 为假 → **跳过**，避免无意义的磁盘写入
- 否则 → 调 `Document::saveToFile()` 写回原文件

未命名 Tab **永远不会**触发自动保存对话框。这避免了"输入两秒，弹一个 Save As 对话框打断输入"的灾难性 UX。

### [INV-2] 失败不打断输入

`Document::saveToFile()` 返回 false 时（磁盘只读 / 权限缺失 / 外部占用 / 路径已删除）：
- **禁止**弹出 QMessageBox
- **必须**在状态栏右下区显示瞬时提示："自动保存失败：<filename>"
- 提示文字 5s 后自动清空（不需要用户操作）
- 编辑器和文档**继续可用**，用户可以手动 `Ctrl+S` 再试或修改路径

如果有多个 Tab 同时失败，提示文本拼接所有失败的文件名（逗号分隔）。

### [INV-3] 防抖（连续编辑只触发一次）

`scheduleAutoSave()` 必须使用 `QTimer::start()`（而非 `start() if !isActive()`）来 **重置** 计时：
- 用户每次按键都 restart timer，timer 不到期不触发保存
- 只有连续停顿超过 `m_autoSaveDelayMs` 后才执行 `performAutoSave`
- 保证连续输入 100 个字符**只**触发一次保存，而非 100 次

### [INV-4] 关闭开关后立即停止

用户在菜单关闭 `Auto Save` 时：
- `m_autoSaveEnabled = false`
- **必须** 调 `m_autoSaveTimer.stop()` 取消已 pending 的 timer
- 后续 `scheduleAutoSave` 调用立即返回，不重启 timer
- 用户可以继续编辑，没有任何后台保存动作

### [INV-5] 切换延迟立即生效

用户改 `Auto Save Delay` 时：
- `m_autoSaveDelayMs` 立即更新
- 下一次 `scheduleAutoSave` 用新的延迟值
- 当前已 pending 的 timer 不强制 restart（避免延迟切换时的"延后一次"边界条件）

### [INV-6] 自动保存与手动保存等价

`performAutoSave` 中对单个 Tab 的保存动作必须与 `MainWindow::onSaveFile` 行为一致：
- `unwatchFile(fp)` 暂停外部修改监控
- `Document::saveToFile()` 用当前 filePath 保存
- `updateTabTitle(idx)` 同步 Tab 标题
- `QTimer::singleShot(500, watchFile)` 异步重新监控

如果 onSaveFile 后续行为有任何调整，performAutoSave 也必须同步更新（避免双份真实）。

### [INV-7] 持久化

设置项写入 `QSettings`：
- `autoSave/enabled` (bool, **默认 false**——默认不开启自动保存)
- `autoSave/delayMs` (int, **默认 300000=5min**，仅接受 30000/300000/600000 三档：30s/5min/10min)

加载时如果 `delayMs` 不在合法档位中（含旧版本遗留的 1500/3000/5000 小延时），必须
**回退到 300000(5min)** 而非保留非法值。

## 4. 接口

### 4.1 MainWindow 新增成员

```cpp
class MainWindow : public QMainWindow {
private:
    QTimer m_autoSaveTimer;             // singleShot, 防抖
    bool m_autoSaveEnabled = true;
    int m_autoSaveDelayMs = 1500;       // 1500 / 3000 / 5000
    QAction* m_autoSaveEnabledAct;
    QVector<QAction*> m_autoSaveDelayActions;

    // 状态栏失败提示
    QLabel* m_statusAutoSaveMsg;        // 平时 hide()
    QTimer m_autoSaveMsgClearTimer;     // 5s 后清空 + hide

    void scheduleAutoSave();   // textChanged 信号槽
    void performAutoSave();    // timer 到期
    void showAutoSaveError(const QString& message);
};
```

## 5. 核心算法

```
scheduleAutoSave():
    if not m_autoSaveEnabled: return
    m_autoSaveTimer.start(m_autoSaveDelayMs)   # restart, 防抖

performAutoSave():
    if not m_autoSaveEnabled: return
    failed = []
    for tab in m_tabs:
        doc = tab.editor.document
        if doc.filePath.isEmpty(): continue    # INV-1
        if not doc.isModified(): continue
        unwatchFile(doc.filePath)
        ok = doc.saveToFile()
        if ok:
            updateTabTitle(idx)
            singleShot(500ms, watchFile(doc.filePath))
        else:
            failed.append(basename(doc.filePath))
    if failed:
        showAutoSaveError("自动保存失败：" + ", ".join(failed))
```

## 6. 性能预算

| 操作 | 目标 |
|------|------|
| `scheduleAutoSave` 开销（每次按键） | < 0.05 ms |
| `performAutoSave`（10 个有路径 Tab） | < 50 ms |
| 状态栏失败提示 → 清空（5s 后） | 不阻塞 UI |

## 7. 验收条件

- **[T-AS-1 基本保存]** 打开一个已有 .md 文件，编辑 + 等 1.5s → 文件 mtime 改变，文件内容包含编辑后的字符
- **[T-AS-2 防抖]** 连续按键 100 次 → 不会触发 100 次保存，而是停下后 1.5s 触发**1 次**
- **[T-AS-3 未命名跳过]** 新建 Tab（未保存），输入文字，等 5s → **不弹**任何对话框，文件没被创建
- **[T-AS-4 关闭开关]** 关闭 Auto Save → 编辑后等 5s 没有后台保存动作，文件 mtime 不变
- **[T-AS-5 失败不打断]** 把目标文件设为只读 → 编辑后等 1.5s → 状态栏出现红字提示，编辑器仍可输入，5s 后提示自动消失
- **[T-AS-6 多 Tab]** 同时打开 3 个有路径 Tab，分别编辑 → 1.5s 后 3 个文件都被保存，dirty 标记全部清除
- **[T-AS-7 切换延迟]** 切换到 5s 延迟 → 编辑后必须等满 5s 才保存
- **[T-AS-8 持久化]** 关闭 Auto Save / 切到 3s → 重启应用 → 设置仍然是关闭 + 3s
- **[T-AS-9 持久化非法值]** 手动改 QSettings 把 `autoSave/delayMs` 设为 9999 → 重启应用后回退到 1500
- **[T-AS-10 已保存不重保]** 文件已保存（dirty=false），等待 5s → `performAutoSave` 跳过该 Tab，不写盘

## 8. 已知陷阱

### 8.1 与"保存时间戳"显示的联动

后续如果做"已保存 · X 分钟前"状态栏显示（参考 `plans/2026-04-14-状态栏重新布局.md`），自动保存后必须刷新该时间戳。本 Spec 不处理该交互——交给状态栏 plan。

### 8.2 与外部修改冲突

如果用户用外部编辑器（VS Code）改了同一文件，再回到 SimpleMarkdown 编辑：
- 自动保存会**覆盖**外部修改，不做合并
- `unwatchFile` → `saveToFile` → `watchFile` 顺序保证不会触发"文件被外部改动"对话框
- 这是预期行为：用户回到本应用编辑就视为"以本地为准"

V2 可以加"外部变更检测"在自动保存前 prompt。

### 8.3 跨 Tab 同名冲突

两个 Tab 打开同一文件路径（不应该发生但需考虑），同时编辑两边，自动保存会按 Tab 顺序写盘，后写的覆盖先写的。当前不做防御，依赖单实例 + 单 Tab/path 保证。

### 8.4 极快编辑 + 进程退出

如果用户输入完最后几个字符立即关闭窗口（< 1.5s），timer 还没到期，`closeEvent` 必须负责把 dirty Tab 通过 `maybeSave` 提示用户。本 Spec 不改动 `closeEvent` 行为。

### 8.5 大文档防抖延迟感

10MB 的 .md 文件，`Document::saveToFile` 可能耗时 30-80ms。1.5s 延迟 + 写入耗时还是可以接受的。如果未来加 incremental save，再修订。
