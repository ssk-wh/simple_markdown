# 演示模式（Presentation Mode）

## 1. 目的

定义"演示模式"这一应用级全屏状态：当用户按 **F11** 时，窗口进入全屏，只显示当前 Tab **渲染后的 Markdown 内容（预览区）**，隐藏一切编辑干扰（菜单栏、Tab 栏、编辑器、TOC、状态栏），让用户可以把 Markdown 文档直接当作幻灯片/演讲稿向观众演示。

本 Spec 替代/升级原有的 **Focus Mode（专注模式）**：

- **Focus Mode（旧）**：全屏化**编辑区**，方便无干扰写作
- **Presentation Mode（新）**：全屏化**预览区**，方便向观众演示

两个模式的关键差别在于"全屏后谁被放大"——演示模式里编辑器隐藏，预览独占窗口；而专注模式里预览隐藏，编辑器独占窗口。

本 Spec **不**定义演示模式下预览区自身的翻页实现细节（滚动量、页断点），只约束演示模式的状态机、UI 显隐、快捷键映射和布局恢复。预览区自身的滚动行为由 `模块-preview` 维护。

## 2. 输入 / 输出

- **输入**：
  - 用户操作：菜单项"视图 → 演示模式"点击、F11 快捷键、Esc 快捷键
  - 当前 MainWindow 布局状态（MainSplitter sizes、Tab Splitter sizes、TOC 可见性、菜单栏/状态栏可见性、当前 Tab 索引）
  - 当前 Tab 的预览区内容与滚动位置
- **输出**：
  - 窗口状态切换：normal ↔ fullScreen
  - UI 元素显隐切换：menuBar / statusBar / tabBar / editor / tocPanel / searchBar 隐藏；preview 占满整个可见区域
  - 快捷键上下文切换：正常模式下的编辑快捷键（Ctrl+S/Z/F 等）临时禁用，翻页快捷键（Space/PageUp/PageDown/Home/End/↑/↓）在预览区生效
  - `m_presentationMode` 状态字段
  - `m_focusModeAct`（或新菜单项）的 checked 状态同步

## 3. 行为契约（不变量）

### [INV-1] F11 切换演示模式

F11 快捷键负责在"正常模式"和"演示模式"之间切换：

- 正常模式下按 F11 → 进入演示模式
- 演示模式下按 F11 → 退出演示模式

F11 是**唯一**进入演示模式的快捷方式（除菜单点击外），且进入与退出使用**同一个**快捷键。

### [INV-2] Esc 退出演示模式

演示模式下按 Esc 键必须退出演示模式。Esc 在正常模式下不触发演示模式相关行为（保留给搜索栏关闭等既有用途）。

### [INV-3] 进入演示模式窗口必须全屏

进入时必须调用 `QWidget::showFullScreen()`，不能用 `showMaximized()` —— 两者的关键区别：

- `showMaximized()` 保留任务栏/标题栏 → 观众仍能看到应用 chrome
- `showFullScreen()` 隐藏任务栏/标题栏 → 真正的"演讲幻灯片"观感

演示模式的本质就是把预览内容变成全屏幻灯片，故必须用 `showFullScreen()`。

### [INV-4] 演示模式必须隐藏的 UI 元素

进入演示模式时**必须**隐藏以下元素，不允许例外：

| 元素 | 隐藏方式 |
|------|----------|
| 菜单栏（QMenuBar） | `menuBar()->hide()` |
| 状态栏（QStatusBar） | `statusBar()->hide()` |
| Tab 栏（QTabBar） | `m_tabWidget->tabBar()->hide()` |
| 编辑器（EditorWidget） | 当前 Tab 的 `editor->hide()` |
| TOC 面板（TocPanel） | `m_tocPanel->hide()` |
| 搜索栏（SearchBar） | 若正在显示，`searchBar->hide()` |
| Tab Splitter 手柄 | 自然随 editor 隐藏消失 |

**非当前 Tab** 的编辑器也必须隐藏（防止 QTabWidget 内部切换时泄露出来）。

### [INV-5] 演示模式预览区占满可视区

进入演示模式后，**当前 Tab 的 PreviewWidget** 必须占满整个主窗口可视区域（等于 `centralWidget()` 的 geometry，去掉被隐藏的元素后）：

- 预览区 `show()` 且可见
- 预览区的 viewport 宽度 = 主窗口宽度
- 预览区的 viewport 高度 = 主窗口高度
- 不允许预览区周围出现灰色/白色的空白边框（除非预览区本身的 padding 需要）

### [INV-6] 退出演示模式必须完整恢复布局

退出演示模式时必须恢复进入前的**所有**布局状态：

| 状态 | 来源 | 恢复方式 |
|------|------|----------|
| 主 Splitter 分隔位置 | `m_savedMainSplitterState` | `m_mainSplitter->restoreState(...)` |
| 当前 Tab 的 Splitter 分隔位置 | `m_savedTabSplitterSizes` | `currentTab()->splitter->setSizes(...)` |
| TOC 面板可见性 | `m_savedTocVisible` | `m_tocPanel->setVisible(...)` |
| 菜单栏/状态栏/Tab 栏可见性 | 默认全部可见 | `show()` |
| 所有 Tab 的 editor 可见性 | 默认全部可见 | `editor->show()` |
| 当前 Tab 索引 | 进入时记录 | 不应自动切换，只恢复 |
| 窗口状态 | `showNormal()` | 从 fullScreen 退回 |
| 光标焦点 | 进入前持有焦点的 widget | `editor->setFocus()` 或保留原焦点 |

保存操作发生在 `enterPresentationMode()` 开头；恢复操作发生在 `exitPresentationMode()` 中。

### [INV-7] 演示模式保留的快捷键

演示模式下以下快捷键必须**保持可用**，用于翻页和滚动：

| 快捷键 | 作用 |
|--------|------|
| F11 | 退出演示模式 |
| Esc | 退出演示模式 |
| Space | 向下翻页（约一屏） |
| PageDown | 向下翻页 |
| PageUp | 向上翻页 |
| Home | 跳到文档开头 |
| End | 跳到文档结尾 |
| ↑ / ↓ | 逐行滚动 |
| Ctrl+滚轮 | 保留（字号缩放，演讲者可临时放大） |
| F1 | 帮助/快捷键说明（可选） |

这些快捷键**不能**被主菜单栏占用的 QShortcut 拦截（菜单栏已隐藏，但 QAction 的 shortcut 可能仍然 active）。

### [INV-8] 演示模式禁用的快捷键

演示模式下以下快捷键**必须**被禁用或无视，以防误操作：

| 快捷键 | 原功能 | 禁用原因 |
|--------|--------|----------|
| Ctrl+S | 保存 | 编辑器不可见，保存无意义 |
| Ctrl+Shift+S | 另存为 | 同上 |
| Ctrl+Z / Ctrl+Y | 撤销/重做 | 同上 |
| Ctrl+F / Ctrl+H | 查找/替换 | 搜索栏已隐藏 |
| Ctrl+N | 新建 | 会破坏演示状态 |
| Ctrl+O | 打开 | 会破坏演示状态 |
| Ctrl+W | 关闭 Tab | 会破坏演示状态 |
| Ctrl+Tab / Ctrl+Shift+Tab | 切换 Tab | 见 INV-9 |
| Ctrl+, | 设置 | 编辑器不可见 |

**禁用实现建议**：通过 `QAction::setEnabled(false)` 或在 `eventFilter` 中拦截 key press 事件。

### [INV-9] 演示模式下禁止切换 Tab

演示模式下**不允许**通过快捷键（Ctrl+Tab、Ctrl+PageUp/Down）、鼠标（Tab 栏已隐藏）或拖拽切换到其他 Tab。一次演示模式只演示进入时的那个 Tab。

**原因**：演讲场景下误切换 Tab 会暴露无关文档，造成尴尬。

**可能的例外**：允许 Ctrl+Shift+Right / Ctrl+Shift+Left 切换（如果 Plan 决定放宽，需要明确写入本 INV 的例外条款）。

### [INV-10] 演示模式与 Focus Mode 互斥

演示模式（Presentation Mode）与专注模式（Focus Mode）是两个**互斥**的状态：

- `m_presentationMode == true` 时 `m_focusMode` 必须为 `false`
- `m_focusMode == true` 时 `m_presentationMode` 必须为 `false`
- 从任一模式切换到另一模式时，必须先退出当前模式再进入目标模式（不允许叠加）
- 布局保存字段可以共用（`m_savedMainSplitterState` 等），但保存/恢复的时机必须清晰

如果 Plan 最终决定**合并**两个模式为单一全屏状态（见 Plan 决策点），本条可降级为历史说明。

### [INV-11] 多显示器场景保持屏幕

演示模式进入时**必须**保持在当前窗口所在的显示器。Qt 的 `showFullScreen()` 在某些 Windows 版本/驱动下会错误地把窗口移到主显示器。

**实现要求**：进入演示模式前记录 `QScreen* currentScreen = windowHandle()->screen()`；若 `showFullScreen()` 后检测到 `windowHandle()->screen() != currentScreen`，必须显式 `setScreen(currentScreen)` 并重新 `showFullScreen()`。

### [INV-12] 切换过程必须瞬时

用户感知的模式切换延迟（按 F11 到首帧稳定）必须 **< 150 ms**。禁止：

- 分步显隐 UI 元素（先隐菜单、再隐编辑器…）造成可见抖动
- 切换过程中触发重新布局和 reflow 在同一帧多次
- 窗口从 normal 到 fullScreen 的动画（若 Qt 默认有动画，要关闭）

**实现建议**：在一次事件循环 tick 内完成所有 `hide()` / `show()` / `showFullScreen()` 调用；必要时用 `setUpdatesEnabled(false)` / `true` 包裹整个过程防止中间帧绘制。

### [INV-13] 进入演示模式保留预览滚动位置

用户可能在"阅读到一半"时按 F11 开始演示（例如讲到第 5 页时切换到幻灯片模式）。演示模式进入时**必须**保留预览区当前的滚动位置：

- 进入前 `yOffset = preview->scrollY()`
- 进入后（完成 UI 显隐和 showFullScreen 之后）仍然 `preview->scrollY() == yOffset`

注意由于 viewport 宽度变化，布局可能会重新换行，"相同的 yOffset"不一定对应"相同的视觉位置"。最小契约是"滚动位置不被 reset 到 0"；更严格的"保持相同段落可见"作为增强目标。

### [INV-14] 菜单项文案与 F11 快捷键绑定

菜单"视图"下必须有一个入口项对应演示模式：

- **选项 A（替换）**：原"Focus Mode"菜单项改名为"演示模式"/"Presentation Mode"，保留 F11 快捷键和 `checkable` 属性。
- **选项 B（并列）**：保留"Focus Mode"，新增"演示模式"菜单项。但此时两个菜单项**不能都绑定 F11**，必须拆分快捷键（例如 Focus Mode 改为 Ctrl+Shift+F11 或取消快捷键）。

具体选 A 还是 B 由 Plan 决策（见 `plans/2026-04-13-演示模式.md`）。本 Spec 不预先锁定。

### [INV-15] 状态查询接口

必须提供 `MainWindow::isPresentationMode() const` 返回当前是否处于演示模式。其他模块（例如快捷键弹窗、会话恢复）可通过此接口查询并调整行为。

### [INV-16] 会话恢复不保留演示模式状态

会话恢复（`saveSessionState` / `restoreSessionState`）**必须不**序列化 `m_presentationMode` 字段。应用下次启动时总是以正常模式打开，即使上次退出时窗口处于演示模式。

**原因**：用户在演讲中意外关闭应用，下次打开预期是"回到编辑状态"而非"再次进入全屏幻灯片"。

### [INV-17] 关闭事件自动退出

`closeEvent` 中必须在执行真正的关闭流程之前主动退出演示模式（与现有 Focus Mode 逻辑对齐）。否则窗口可能以全屏状态被销毁，导致未保存布局状态污染下次启动。

## 4. 接口（API 签名）

```cpp
// MainWindow.h 新增/调整

class MainWindow : public QMainWindow {
public:
    // 查询当前是否处于演示模式
    bool isPresentationMode() const { return m_presentationMode; }

private slots:
    // 切换演示模式（F11 / 菜单项 triggered 信号入口）
    void togglePresentationMode();

private:
    void enterPresentationMode();
    void exitPresentationMode();

    // 演示模式状态
    bool m_presentationMode = false;
    QAction* m_presentationModeAct = nullptr;

    // 布局保存（可与 Focus Mode 共用，也可独立）
    QByteArray m_savedMainSplitterState_Presentation;
    QList<int> m_savedTabSplitterSizes_Presentation;
    bool m_savedTocVisible_Presentation = true;
    QScreen* m_savedScreen_Presentation = nullptr;
    qreal m_savedPreviewScrollY_Presentation = 0.0;

    // 演示模式下被禁用的 QAction 列表（进入时批量 setEnabled(false)，退出时恢复）
    QList<QAction*> m_disabledInPresentation;
};
```

键盘快捷键注册入口：

```cpp
// 在 createMenus() 里
m_presentationModeAct = viewMenu->addAction(tr("Presentation Mode"));
m_presentationModeAct->setCheckable(true);
m_presentationModeAct->setShortcut(QKeySequence(Qt::Key_F11));
connect(m_presentationModeAct, &QAction::triggered, this, &MainWindow::togglePresentationMode);
```

事件过滤（用于 Esc 退出、翻页快捷键）：

```cpp
bool MainWindow::eventFilter(QObject* obj, QEvent* event) override;
// 在 m_presentationMode == true 时：
//   - Key_Escape → exitPresentationMode(), return true
//   - Key_Space / Key_PageDown → preview->scrollByPage(+1)
//   - Key_PageUp → preview->scrollByPage(-1)
//   - Key_Home → preview->scrollToTop()
//   - Key_End → preview->scrollToBottom()
//   - Ctrl+(任意编辑快捷键) → 拦截并吞掉
```

## 5. 核心算法

### 进入演示模式

```
enterPresentationMode():
    if m_presentationMode: return                              // INV-1 幂等
    if m_focusMode: exitFocusMode()                            // INV-10 互斥

    // 1. 保存状态
    m_savedMainSplitterState = m_mainSplitter.saveState()
    m_savedTocVisible = m_tocPanel.isVisible()
    tab = currentTab()
    m_savedTabSplitterSizes = tab.splitter.sizes()
    m_savedScreen = windowHandle().screen()
    m_savedPreviewScrollY = tab.preview.scrollY()              // INV-13

    // 2. 冻结绘制，减少闪烁（INV-12）
    setUpdatesEnabled(false)

    // 3. 隐藏所有被遮挡元素（INV-4）
    menuBar().hide()
    statusBar().hide()
    m_tabWidget.tabBar().hide()
    m_tocPanel.hide()
    for t in m_tabs:
        t.editor.hide()
        t.searchBar.hide() if visible
        t.preview.show()  // 保留可见，但下面会让当前 tab 独占

    // 4. 禁用编辑快捷键（INV-8）
    for act in m_editingActions:
        m_disabledInPresentation.append(act)
        act.setEnabled(false)

    // 5. 切换到全屏（INV-3）
    m_presentationMode = true
    m_presentationModeAct.setChecked(true)
    showFullScreen()

    // 6. 校正屏幕（INV-11）
    if windowHandle().screen() != m_savedScreen:
        windowHandle().setScreen(m_savedScreen)
        showFullScreen()

    // 7. 恢复预览滚动位置（INV-13）
    tab.preview.setScrollY(m_savedPreviewScrollY)

    // 8. 焦点转给预览区（接收键盘翻页事件）
    tab.preview.setFocus()

    setUpdatesEnabled(true)
```

### 退出演示模式

```
exitPresentationMode():
    if not m_presentationMode: return                          // INV-1 幂等

    setUpdatesEnabled(false)

    // 1. 退出全屏
    showNormal()

    // 2. 恢复所有 UI 元素（INV-6）
    menuBar().show()
    statusBar().show()
    m_tabWidget.tabBar().show()
    m_tocPanel.setVisible(m_savedTocVisible)
    for t in m_tabs:
        t.editor.show()

    // 3. 恢复 Splitter 尺寸
    m_mainSplitter.restoreState(m_savedMainSplitterState)
    if not m_savedTabSplitterSizes.isEmpty():
        currentTab().splitter.setSizes(m_savedTabSplitterSizes)

    // 4. 恢复编辑快捷键
    for act in m_disabledInPresentation:
        act.setEnabled(true)
    m_disabledInPresentation.clear()

    // 5. 状态字段归零
    m_presentationMode = false
    m_presentationModeAct.setChecked(false)

    // 6. 焦点回到编辑器
    if currentTab(): currentTab().editor.setFocus()

    setUpdatesEnabled(true)
```

### eventFilter 处理键盘

```
eventFilter(obj, event):
    if event.type() == QEvent::KeyPress and m_presentationMode:
        ke = cast<QKeyEvent>(event)
        switch ke.key():
            case Qt::Key_Escape:
            case Qt::Key_F11:
                exitPresentationMode()
                return true
            case Qt::Key_Space:
            case Qt::Key_PageDown:
                currentTab().preview.scrollByPage(+1)
                return true
            case Qt::Key_PageUp:
                currentTab().preview.scrollByPage(-1)
                return true
            case Qt::Key_Home:
                currentTab().preview.scrollToTop()
                return true
            case Qt::Key_End:
                currentTab().preview.scrollToBottom()
                return true
            // INV-9: 禁止切换 Tab
            if ke.modifiers() & Qt::ControlModifier:
                if ke.key() in {Qt::Key_Tab, Qt::Key_PageUp, Qt::Key_PageDown,
                                 Qt::Key_S, Qt::Key_O, Qt::Key_N, Qt::Key_F, Qt::Key_H,
                                 Qt::Key_W, Qt::Key_Z, Qt::Key_Y}:
                    return true  // 吞掉
    return QMainWindow::eventFilter(obj, event)
```

## 6. 性能预算

| 操作 | 目标 |
|------|------|
| F11 按下到首帧全屏稳定 | < 150 ms（INV-12） |
| Esc 按下到首帧恢复稳定 | < 150 ms |
| PageDown 按下到预览区滚动响应 | < 30 ms |
| 进入演示模式的内存峰值增长 | < 5 MB（仅保存状态结构，不缓存额外内容） |
| 退出演示模式后 Splitter 位置恢复精度 | 像素级（与保存值完全一致） |

## 7. 验收条件（测试用例清单）

- **[T-1]** 正常模式下按 F11：窗口进入全屏，菜单栏、Tab 栏、编辑器、TOC 面板、状态栏全部不可见，当前 Tab 的预览区占满整个屏幕。
- **[T-2]** 演示模式下按 F11：窗口从全屏恢复，菜单栏、Tab 栏、编辑器、TOC、状态栏全部恢复显示。
- **[T-3]** 演示模式下按 Esc：等价于 F11，退出演示模式。
- **[T-4]** 进入演示模式前光标位于编辑器某位置 → 退出后光标仍在同一位置（或至少编辑器重新获得焦点）。
- **[T-5]** 进入前预览区滚动到某个中间位置 → 进入演示模式后预览区仍然显示同一内容区域（滚动位置保留，允许因宽度变化有小幅偏差）。
- **[T-6]** 演示模式下按 Ctrl+S 不触发保存（没有保存对话框或 toast），文件 mtime 不变。
- **[T-7]** 演示模式下按 PageDown：预览区向下滚动约一个 viewport 高度。
- **[T-8]** 演示模式下任何时刻 `menuBar()->isVisible() == false`。
- **[T-9]** 退出演示模式后 `m_mainSplitter->sizes()` 等于进入前的 sizes（误差 0 px）。
- **[T-10]** 退出演示模式后 `m_tocPanel->isVisible()` 等于进入前的可见状态。
- **[T-11]** 多 Tab 场景：打开 3 个 tab，切换到 Tab 2，按 F11 → 演示模式只显示 Tab 2 的预览区内容；其他 tab 的 preview / editor 均不可见。
- **[T-12]** 演示模式期间关闭应用（Alt+F4），重新启动应用 → 以正常模式启动，不恢复演示状态。
- **[T-13]** 深色主题下按 F11：演示模式预览区背景色应为深色主题的背景色，不出现闪白、不出现浅色残留。
- **[T-14]** 菜单项 tooltip / 文案：打开视图菜单，F11 对应的菜单项文案必须是"演示模式"/"Presentation Mode"，不再是"专注模式"/"Focus Mode"（若采用替换方案）；或在并列方案下，两个菜单项均可见且快捷键不冲突。
- **[T-15]** 演示模式下按 Ctrl+Tab：不切换 Tab（INV-9）。
- **[T-16]** 演示模式下按 ↑/↓ 方向键：预览区以逐行粒度滚动。
- **[T-17]** 演示模式下按 Home / End：预览区跳到文档开头/结尾。
- **[T-18]** 双显示器场景：将应用拖到副屏，然后按 F11 → 演示模式在副屏全屏，不跨屏（INV-11）。
- **[T-19]** 演示模式与 Focus Mode 互斥：从 Focus Mode 按 F11 切换到演示模式应该先退出 Focus Mode 再进入 Presentation Mode（不叠加两个状态），或者两者合并为一个状态。
- **[T-20]** 性能：F11 按下到屏幕稳定的时间 < 150 ms（用 QElapsedTimer 测量 keyPress 到 paintEvent 结束）。

## 8. 已知陷阱

### 8.1 Focus Mode 与 Presentation Mode 的命名混淆

当前代码使用 `m_focusMode` 字段和 `focusMode` 术语表示"全屏化编辑区"。引入 Presentation Mode 后必须明确：

- 如果**替换**：重命名 `m_focusMode → m_presentationMode`，所有注释和菜单文案同步更新
- 如果**并列**：两个字段 `m_focusMode` 和 `m_presentationMode` 独立存在，INV-10 必须在运行时强制互斥

绝不允许出现"F11 有时进 Focus Mode 有时进 Presentation Mode"的歧义。

### 8.2 `showFullScreen()` vs `showMaximized()` 的区别

`showMaximized()` 只是把窗口放大到屏幕可用区域，仍然保留任务栏、标题栏、窗口边框。Presentation Mode 必须用 `showFullScreen()` 才能做到真正的"幻灯片"观感。

### 8.3 多显示器下 Qt 的屏幕错位问题

Qt 5.x / 6.x 的 `showFullScreen()` 在某些场景（特别是副屏 + 显示缩放混用）会把窗口移到主显示器，而不是保持在当前屏幕。必须显式调用 `windowHandle()->setScreen(...)` 绑定屏幕后再 `showFullScreen()`。

### 8.4 IME / 全局快捷键占用 F11

部分 IME（如某些中文输入法）或第三方工具会全局捕获 F11。若出现"按 F11 没反应"的反馈，先确认：

1. Qt 层是否收到 KeyPress 事件（加 qDebug 打印）
2. 是否被 Windows Magnifier / QQ 截屏 / 远程桌面客户端吞掉

此类问题 Spec 不能完全规避，但要在文档中说明"若 F11 无效请通过菜单点击"。

### 8.5 TabWidget 隐藏 TabBar 后高度计算偶尔错位

Qt 的 `QTabWidget::tabBar()->hide()` 在某些 style 下不会自动把剩余空间还给 content 区域，导致 content 顶部出现一条灰色边。处理方式：

- 改为 `m_tabWidget->setTabBarAutoHide(false)` + 手动 hide
- 或在 hide 后触发 `m_tabWidget->updateGeometry()` + `invalidate()`

Focus Mode 的历史实现里就踩过类似坑，迁移到 Presentation Mode 时要回归。

### 8.6 会话恢复序列化污染

如果 `saveSessionState` 无意中序列化了 `m_presentationMode` 或"window fullscreen flag"，下次启动会意外进入演示模式。INV-16 明确禁止此行为：序列化代码中必须过滤掉 fullscreen / presentation 相关字段。

### 8.7 编辑快捷键禁用的遗漏

INV-8 列出的快捷键只是常见项，实际代码中的 QAction 可能更多（例如"插入日期"、"粘贴为纯文本"等）。最佳实践是维护一份"演示模式白名单" QAction 列表（只保留翻页、主题切换、退出演示等），其他所有 `menuBar()` 下的 QAction 统统 `setEnabled(false)`，退出时批量恢复。

### 8.8 预览区滚动位置的语义模糊

INV-13 要求"保留滚动位置"，但演示模式下 viewport 宽度变大 → 段落重新 wrap → 同一 y 坐标对应的段落可能不同。严格实现需要按"段落索引 + 段落内偏移"保存，而非像素 y 值。

可行折衷：
- 最小契约：保留像素 y 值（简单但不精确）
- 增强契约：保存"当前视口最顶部的段落索引"，演示模式展开后滚动到该段落

Plan 中决定采用哪种。

### 8.9 与 Focus Mode 共存的代码分支复杂度

如果选择并列方案（保留 Focus Mode 并新增 Presentation Mode），`eventFilter`、`closeEvent`、`switchTab` 等地方都需要同时处理两种模式，容易漏判。建议统一走"单一全屏模式枚举"：

```cpp
enum class FullscreenMode { Normal, Focus, Presentation };
```

而非两个独立 bool 字段。

### 8.10 菜单快捷键显示与菜单栏隐藏的冲突

演示模式下菜单栏隐藏，但 `m_presentationModeAct->shortcut()` 仍然 active，F11 事件能被捕获。这一点 Qt 是支持的（QAction 的 shortcut 不依赖父菜单可见性）。但如果开发者把 QAction 的 shortcut 改成了 `Qt::WindowShortcut` context 且把 action 设为 `parent = menuBar()`，可能会出现"菜单栏隐藏后 shortcut 也失效"的诡异现象。必须：

```cpp
m_presentationModeAct->setShortcutContext(Qt::ApplicationShortcut);
```

或把 action 挂载到 MainWindow 本身而非 menuBar。
