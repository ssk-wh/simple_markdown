# 编辑/预览分隔条拖拽吸附刻度线

对应源文件：`src/app/SnapSplitter.h/cpp`、`src/app/MainWindow.cpp`（createTab）

## 1. 背景

每个 Tab 的 editor | preview 用 `QSplitter(Qt::Horizontal)` 分栏，用户希望拖拽分隔条时有常用比例的视觉参考线，并在靠近刻度时自动吸附，方便还原到常见等分/1:3/3:1 布局。

## 2. 行为定义

### 2.1 视觉刻度

拖拽开始（mouse press on handle）到拖拽结束（mouse release）之间，在 splitter 区域覆盖一层透明 overlay，绘制 3 条垂直虚线：

- 位置：splitter 宽度的 25% / 50% / 75%
- 样式：1 像素虚线（`Qt::DashLine`，pattern 建议 `{4, 4}` 逻辑像素）
- 颜色：`theme.accentColor`，alpha = 96（非激活）/ 200（激活/命中吸附）
- 非拖拽状态：overlay 隐藏，不占空间、不拦截鼠标

### 2.2 吸附逻辑

- 阈值：5 逻辑像素（固定，V1 不暴露配置）
- 判断：对拖拽中的 `pos`（handle 左边 pane 的宽度），若 `|pos - target_i| <= 5`，则 `setSizes({target_i, total - target_i})` 强制对齐
- 命中时对应虚线切换到"激活"颜色
- 防递归：`setSizes` 会再次触发 `splitterMoved`，用 `m_snapping` guard 屏蔽一次

### 2.3 只适用 editor|preview splitter

- 主窗口的 content|toc 主 splitter 不加吸附（TOC 有自适应宽度逻辑）
- 演示模式下 editor 被收起（`setSizes({0, 1})`），handle 宽度 0 仍存在但用户无法拖动——overlay 不显示（无 press 事件）

## 3. 不变量（INV）

- **[INV-SNAP-THRESHOLD]** 吸附阈值固定为 5 逻辑像素
- **[INV-SNAP-TARGETS]** 吸附目标固定为 25% / 50% / 75%，基于 `splitter->width()` 实时计算
- **[INV-SNAP-VISIBLE-ON-DRAG]** overlay 仅在 handle press→release 期间可见
- **[INV-SNAP-TRANSPARENT]** overlay 必须 `Qt::WA_TransparentForMouseEvents`，不得拦截 handle/editor/preview 的鼠标事件
- **[INV-SNAP-THEME]** overlay 颜色随主题 `accentColor` 实时更新
- **[INV-SNAP-NO-RECURSION]** 吸附产生的 `setSizes` 必须带 guard，禁止递归触发 `splitterMoved`

## 4. 算法

```
on handle mousePress:
    overlay.setActive(-1)
    overlay.setGeometry(splitter.rect())
    overlay.show(); overlay.raise()

on splitterMoved(pos, 1):
    if snapping: return
    W = splitter.width()
    targets = {0.25*W, 0.5*W, 0.75*W}
    hit = index i such that |pos - targets[i]| <= 5 (最近的)
    overlay.setActive(hit)  // -1 表示没命中
    if hit != -1:
        snapping = true
        setSizes({targets[hit], W - targets[hit] - handleWidth})
        snapping = false

on handle mouseRelease:
    overlay.hide()
    overlay.setActive(-1)

on splitter resize while dragging:
    overlay.setGeometry(splitter.rect())  // 跟随宽度
```

## 5. 主题集成

- `MainWindow::applyTheme` 调用 `snapSplitter->setAccentColor(theme.accentColor)` 同步给 overlay
- 主题切换时若 overlay 可见（拖拽中切主题），下一次 `paintEvent` 自动使用新色

## 6. 高 DPI

- 虚线 dash pattern 用逻辑像素，Qt 会按 devicePixelRatio 自动缩放
- 吸附阈值 5 逻辑像素在 200% DPI 下物理像素约 10 像素，手感符合预期

## 7. 验收条件

- **T1** 静止状态未拖拽：splitter 区域无虚线，editor/preview 渲染不受影响
- **T2** 鼠标按住 handle：overlay 显示 3 条等距虚线，位置分别对应 splitter 宽度 25% / 50% / 75%
- **T3** 拖到 50% 附近（|Δ| ≤ 5px）：尺寸自动吸附到精确 50%，中间虚线高亮
- **T4** 拖到 25% / 75% 附近同样吸附 + 高亮
- **T5** 拖到非吸附区（所有目标距离 > 5px）：正常跟随，无吸附，3 条虚线均非激活态
- **T6** 松开鼠标：overlay 立即消失
- **T7** 高 DPI（175%/200%）下虚线位置按逻辑像素准确对齐到比例点
- **T8** 拖拽中切主题：虚线颜色同步（下次 mouseMove 或 repaint 生效）
- **T9** 演示模式下 editor 被 setSizes({0,1}) 收起：不触发 overlay（handle 宽度不变但用户不拖）
- **T10** Tab 切换：上一个 Tab 的 overlay 随其 splitter 隐藏，不会"串"到新 Tab

## 8. 非目标

- 不支持用户自定义吸附点位置 / 阈值（V1 硬编码）
- 不支持关闭吸附（可作为 V2 菜单项）
- 不在主 splitter（content|toc）应用吸附
