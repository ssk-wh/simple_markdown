## 3. 行为契约

### [INV-1] 启动性能

| 指标 | 目标 | 基准机器 |
|------|------|---------|
| 冷启动到首窗口显示 | < 500 ms | i5-8250U + SSD |
| 首次 Tab 内容就绪 | < 200 ms | 同上 |
| 空载内存占用 | < 30 MB | 同上 |

### [INV-2] 编辑性能

| 指标 | 目标 |
|------|------|
| 单次按键到屏幕更新 | < 3 ms |
| 滚动一屏 | < 10 ms |
| 全文档语法高亮 | < 200 ms |
| 撤销/重做 | < 5 ms |
| 查找（1k 行文件） | < 50 ms |

### [INV-3] 预览性能

| 指标 | 目标 |
|------|------|
| 预览更新（含 30ms 防抖） | < 80 ms |
| 滚动一屏 | < 16 ms（60fps） |
| 本地图片加载 | < 30 ms |
| 代码块高亮 | < 10 ms/块 |

### [INV-4] 解析性能

| 指标 | 目标 |
|------|------|
| 1k 行 Markdown 解析 | < 10 ms |
| 10k 行 Markdown 解析 | < 50 ms |
| 防抖延迟 | 30 ms |

### [INV-5] 文件 IO 性能

| 指标 | 目标 |
|------|------|
| 加载 1 MB 文件 | < 50 ms |
| 加载 10 MB 文件（mmap） | < 100 ms |
| 保存文件 | < 50 ms |

### [INV-6] 主线程不阻塞

任何主线程事件处理单次 < 16 ms（保证 60fps）。超过必须：
- 移到后台线程
- 或分片处理

### [INV-8] 大文件交互式重排预算

拖拽 splitter / 调整窗口宽度等**用户交互过程**触发的连续 `PreviewLayout::buildFromAst`：

| 文档规模 | P50 单帧目标 | P95 单帧目标 |
|---------|-------------|-------------|
| ≤ 1000 行 | < 16 ms（60fps） | < 33 ms（30fps） |
| 1000 ~ 10000 行 | < 33 ms（30fps） | < 50 ms |

**基线演进**（1024 行样例文档）：
- **2026-05-06 初次基线**：拖拽 P50 = 41 ms / 稳态 P50 = 43 ms（劣于 30fps 预算 33 ms）
- **2026-05-06 修复表格越界 bug 顺带优化**（见 INV-15 模块-preview/02）：
  - 拖拽 P50 = 16.7 ms / P95 = 19.1 ms
  - 稳态 P50 = 15.3 ms / P95 = 18.3 ms
- **2026-05-06 后续多项改动后复测**（多 Tab 休眠 / 字号综合度量等不直接影响 buildFromAst 的改动后）：
  - 拖拽 P50 = **11.7 ms** / P95 = **14.7 ms**
  - 稳态 P50 = **11.4 ms** / P95 = **13.1 ms**
  - **所有场景 P50 + P95 都 ≤ 16ms**——60fps 单帧预算完美达成
- 详见 `plans/归档/2026-05-06-大文件渲染性能基线脚本.md` + `plans/归档/2026-05-06-表格单元格内容超出与下方重合.md`

测量工具：`tests/preview/PreviewRenderBenchmark.cpp`，可重复运行作为优化前后对比标尺。

## 8. 已知陷阱

- **Debug 构建不代表 Release 性能**：测试必须在 Release 模式
- **第一次运行包含冷启动开销**：取第 2-N 次运行的中位数
- **QPainter 抗锯齿开销**：某些绘制操作开启 AA 后慢 2-3x，非必要不开
