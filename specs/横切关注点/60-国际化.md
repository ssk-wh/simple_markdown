# 国际化（i18n）

## 1. 目的

定义多语言支持、翻译文件的组织方式、以及字符串如何标记供翻译。

## 2. 输入 / 输出

- **输入**：系统 locale 或用户显式选择
- **输出**：对应语言的 UI 文本

## 3. 行为契约

### [INV-1] 翻译文件位置

- 源翻译文件 `.ts` 位于 `translations/`
- 编译产物 `.qm` 内嵌到资源（`resources.qrc`）或随发行包分发

### [INV-1.1] qm 产物与 qrc 路径一致

- `resources.qrc` 引用的 `.qm` 路径（源码 `translations/*.qm`）必须**始终是最新 lrelease 产物**
- CMake 必须通过 `set_source_files_properties(... OUTPUT_LOCATION ${CMAKE_SOURCE_DIR}/translations)`
  让 `qt5_create_translation` 直接输出到 qrc 期望的位置，**每次 build 自动刷新 qm**
- 禁止让 qm 只生成到构建目录而 qrc 指向源码树旧 qm——这样改 .ts 不改 qm，新翻译永远不生效
- 违反导致的典型症状：源码 `tr()` 和 `.ts` 翻译都齐全，运行时却仍显示英文源串

### [INV-2] 字符串标记

- 所有面向用户的文本必须用 `tr("...")` 或 `QObject::tr("...")` 包装
- 源字符串统一用**英文**（便于 tr 匹配）
- 禁止在中文字符串上做 tr 调用（tr 匹配源串，中文作源串易出错）

### [INV-3] 语言加载

- 启动时先尝试加载用户偏好语言（QSettings）
- 无偏好时回退到系统 locale
- Qt 内置 widget 的翻译（`qt_<locale>.qm`）应与 app 翻译一同加载（先于 app 翻译 install），
  覆盖 Qt 标准 UI（如文本编辑右键菜单等）
- 该 `qt_<locale>.qm` 必须**嵌入 qrc**（`:/translations/qt_zh_CN.qm`），不依赖
  `QLibraryInfo::TranslationsPath`（指向开发机 Qt 安装目录，部署后用户机不存在）

### [INV-QMSGBOX-TR] QMessageBox / QDialog 标准按钮必须自定义 tr() 文案

**不能依赖 Qt 内置翻译来翻译 `QMessageBox` 标准按钮（Yes/No/Save/Cancel 等）。**

原因：Qt5.12 的 `QMessageBox` 标准按钮文案取自 `QPlatformTheme` context
（`defaultStandardButtonText`），但本项目可用的 `qt_zh_CN.qm`（遗留聚合格式）把这些翻译
放在 `QDialogButtonBox` context，二者**不匹配** → 中文界面下标准按钮回退英文（典型症状：
弹窗正文中文、按钮 Yes/No 英文）。匹配的 `qtbase_zh_CN.qm` 在本机 Qt 安装中缺失，不可依赖。
（已实测确认：`translate("QPlatformTheme","&Yes")` 返回 `&Yes`，真实 `QMessageBox` 按钮为 `&Yes`。）

规则：项目创建的 `QMessageBox` / `QDialog` 若需要按钮，**显式用自定义 `tr()` 按钮**：

```cpp
auto* box = new QMessageBox(QMessageBox::Question, tr("..."), tr("..."),
                            QMessageBox::NoButton, this);
QPushButton* okBtn = box->addButton(tr("Reload"), QMessageBox::AcceptRole);
box->addButton(tr("Keep Current"), QMessageBox::RejectRole);
// 响应用 box->clickedButton() == okBtn 判断，不用 result == QMessageBox::Yes
```

2026-06-16「文件外部修改弹窗」即此问题，已改自定义按钮。

### [INV-4] 默认语言

- 当前优先支持：**简体中文**、**英文**
- 其他语言视贡献情况添加

### [INV-5] 菜单重建

- 切换语言后必须重建菜单栏和弹窗（Qt 不会自动刷新已创建的 widget）

N/A（使用 Qt 原生 i18n 基础设施）

## 5. 核心算法

### 启动时加载翻译

```cpp
// MainWindow 构造（必须在任何 tr() 调用和子控件创建之前）
QString locale = QSettings().value("language/locale", "zh_CN").toString();

// 先 install Qt 内置 widget 翻译（标准按钮 Yes/No/Save 等）；qm 嵌入 qrc，
// 不用 QLibraryInfo path（部署后用户机无 Qt 安装目录）。
if (m_qtTranslator->load(":/translations/qt_" + locale + ".qm"))
    qApp->installTranslator(m_qtTranslator);
// 再 install app 自身翻译。
if (m_translator->load(":/translations/simple_markdown_" + locale + ".qm"))
    qApp->installTranslator(m_translator);
```

## 7. 验收条件

- [T-1] 简体中文 locale 启动后菜单是中文
- [T-2] 英文 locale 启动后菜单是英文
- [T-3] 切换语言后所有菜单与弹窗文本更新
- [T-4] 未翻译的字符串显示英文源文，不显示空串
- [T-5] 首次启动欢迎对话框、Help → Show Welcome 在中文 locale 下全部显示中文
- [T-6] CMake build 后 `translations/simple_markdown_zh_CN.qm` 时间戳新于最近一次 .ts 修改，
  `lconvert -i <qm> -o <tmp.ts>` 反解后必须包含所有新增 UI 字符串的中文翻译
- [T-7] 中文 locale 下「文件外部修改」弹窗按钮显示中文（"重新加载"/"保留当前"），
  即项目用自定义 `tr()` 按钮（INV-QMSGBOX-TR），不依赖 Qt 内置标准按钮翻译；
  对应 `Reload` / `Keep Current` 在 zh_CN.ts 有中文翻译且非 unfinished

## 8. 已知陷阱

- **字符串拼接**：`"Hello, " + name + "!"` 无法正确翻译；应使用 `tr("Hello, %1!").arg(name)`
- **复数形式**：英文单复数、中文无变化，使用 `tr("%n file(s)", "", count)` 的复数形式
- **Menu 切换**：已创建的 QMenu / QAction 不会自动刷新翻译，切换语言必须重建或调用 `changeEvent(QEvent::LanguageChange)`
- **弹窗分组标题 / 快捷键描述遗漏 tr()**：典型场景是 `ShortcutsDialog` / `SettingsDialog` 这类弹窗，
  会维护一份 `{category, action, shortcut}` 的静态数据表。开发者很容易只对 `setWindowTitle` 和按钮
  加 `tr()`，而忘了表内分类名（"File" / "Editor"）、操作描述（"Save File" / "Find"）。
  必须**逐条**检查，所有面向用户的字符串字面量（包括数据表中的）都要走 `tr()`。
  对应翻译条目要在 `translations/simple_markdown_zh_CN.ts` 同步补全，不能停留在 `unfinished`。

## 9. 新增 UI 字符串 Checklist（强制）

**任何新增对话框、面板、菜单项、提示、状态栏文案，完成后必须逐条确认：**

1. [ ] 所有用户可见字符串用 `tr()` 包裹（源字符串为英文）
2. [ ] 打开 `translations/simple_markdown_zh_CN.ts`，确认新条目均有中文翻译且无 `type="unfinished"`
3. [ ] 打开 `translations/simple_markdown_en_US.ts`（en 常无需翻译，但 unfinished 需清掉或保持合理）
4. [ ] 执行一次完整 build（CMake 会自动 lrelease 到 `translations/*.qm`，见 INV-1.1）
5. [ ] `git diff translations/*.qm` 应有变化，确认 qm 随 ts 同步更新
6. [ ] 中文 locale 下启动 exe，目视验证新 UI 显示中文
