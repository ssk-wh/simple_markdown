# 坐标系统

## 1. 目的

定义预览渲染树中**父块到子块的绝对坐标传递规则**，杜绝"列表序号与文本基线不对齐""嵌套块渲染位置错误"这一类坐标计算 bug。

## 2. 输入 / 输出

- **输入**：`LayoutBlock` 树（每个块有相对于父块的 `bounds` 矩形）
- **输出**：每个块在 viewport 中的正确绝对坐标

## 3. 行为契约

### [INV-1] 子块绝对坐标的唯一正确公式

```cpp
qreal childAbsX = parentAbsX + child.bounds.x();
qreal childAbsY = parentAbsY + child.bounds.y();
```

### [INV-2] 严禁把父块坐标直接传给子块

```cpp
// ✗ 错误：子块继承父块坐标，丢失相对偏移
for (const auto& child : block.children) {
    paintBlock(p, child, absX, absY, ...);
}

// ✓ 正确：每个子块都显式计算自己的绝对坐标
for (const auto& child : block.children) {
    qreal childAbsX = absX + child.bounds.x();
    qreal childAbsY = absY + child.bounds.y();
    paintBlock(p, child, childAbsX, childAbsY, ...);
}
```

### [INV-3] 所有容器块遵守同一规则

以下容器类型**全部**必须遵守 INV-1：

- `BlockQuote`（引用块）
- `List`（有序/无序列表）
- `Table`（表格及单元格）
- 任何将来新增的嵌套容器

### [INV-4] 序号与内容使用同一个 absY

列表的序号（如 `1.`）和对应列表项的内容必须用同一个绝对 Y 坐标绘制，绝不能一个是 `absY`、另一个是 `absY + item.y()`。

## 4. 接口

```cpp
// 递归绘制的典型签名
void paintBlock(
    QPainter* p,
    const LayoutBlock& block,
    qreal absX,      // 本块的绝对 X
    qreal absY,      // 本块的绝对 Y
    const RenderContext& ctx
);
```

## 5. 核心算法

```
paintBlock(block, absX, absY):
    根据 block.type 绘制本块背景/边框
    for each child in block.children:
        childAbsX = absX + child.bounds.x()
        childAbsY = absY + child.bounds.y()
        paintBlock(child, childAbsX, childAbsY)
```

### 为什么子块 bounds 不是零？

布局阶段对容器内的子块应用了 `moveLeft(offset)` / `moveTop(offset)` 来表达缩进、序号宽度、边框宽度等。所以 `child.bounds.x() / y()` 不一定是 0，渲染时必须加上。

## 6. 性能预算

N/A（纯坐标计算，开销可忽略）

## 7. 验收条件

- **[T-1]** 有序列表的序号与首行文字基线对齐（`1.` 和 `首行文字` 的 y 坐标相同）
- **[T-2]** 引用块内的嵌套列表位置正确（不出现双倍缩进或缺失缩进）
- **[T-3]** 表格单元格内的内联格式（粗体/斜体/代码）位置正确
- **[T-4]** 嵌套列表（列表中的列表）的子列表序号与父项正文对齐
- **[T-5]** `grep -rn "paintBlock.*absX.*absY" src/preview/` 的所有递归调用都显式计算 `childAbsX/childAbsY`

## 8. 已知陷阱

### 8.1 commit 4073b22 的根本原因

早期实现中 `BlockQuote` 和 `List` 的递归绘制只传递了 Y 偏移，没有处理 X 偏移：

```cpp
// 早期错误代码
case BlockType::List:
    for (const auto& item : block.children) {
        paintBlock(p, item, absX, absY + item.bounds.y(), ctx);
        //                  ^^^^  ← X 坐标忘记加 item.bounds.x()
    }
```

由于布局阶段对列表项应用了 `moveLeft(24)` 来给序号留空间，`item.bounds.x() == 24`。但绘制时只传了原始 `absX`，导致序号画在 X=0，内容画在 X=24，视觉上序号脱离列表。

**修复方案**：
- `BlockQuote` case：`childAbsX = absX + child.bounds.x()`
- `List` case：`itemAbsX = absX + child.bounds.x()`
- `Table` case 早已正确实现（是参考范例）

### 8.2 序号与内容 Y 坐标分裂

容易写成：
```cpp
// ✗ 错误
drawNumber(absY);                          // 用列表块的 absY
drawContent(absY + item.bounds.y());       // 用列表项的 absY
```

两者必须统一用 `itemAbsY`。
