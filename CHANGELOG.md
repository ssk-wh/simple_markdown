# Changelog

All notable changes to this project will be documented in this file.

## [1.0.3] - 2026-04-21

### Improved
- 语言切换对话框改为双按钮（立即重启 / 确认），支持切换后一键重启
- 翻译加载从 main.cpp 移至 MainWindow，为未来运行时切换预留架构
- 文件夹面板选中项背景色改为半透明浅色（亮色主题 15% 透明、暗色 30% 透明），不再使用深色链接色

## [1.0.2] - 2026-04-20

### Added
- 预览区右键菜单：在标记区域右键显示"清除当前标记"选项
- TOC 面板右键菜单：有标记的章节右键显示"清除章节标记"选项
- Tab 懒加载：启动恢复会话时仅加载当前活跃 Tab，其余 Tab 切换时才加载，减少启动时间
- 文件夹面板支持同时打开多个文件夹（单 TreeView 多根节点），右键顶层节点可关闭

### Improved
- TOC 面板和左侧面板默认宽度改为窗口百分比自适应（TOC 15% [160,320]，左侧 18% [180,350]）

### Fixed
- 预览区双击选词和 Ctrl+C 复制内容不准确（表格后 charCounter 漂移导致偏移）
- 预览区代码块行间点击可能选到相邻行（segment rect 高度不覆盖行间距）
- 侧边 Tab 拖拽调整高度时文件夹区域不再被过度压缩

## [1.0.1] - 2026-04-20

### Added
- 文件夹侧栏：文件 → 打开文件夹... 在左侧显示目录下的 Markdown 文件树
- 单击文件打开到当前 Tab，右键菜单支持新建文件/文件夹、重命名、删除、在资源管理器中显示
- 上次打开的文件夹启动时自动恢复，主题切换时样式跟随
- "最近打开"菜单同时记录最近打开的文件和文件夹
- 打开文件后文件夹面板自动选中对应文件
- Tab 栏位置可调：视图 → 标签栏位置 → 显示在左侧 / 显示在左侧并隐藏顶部
- 侧边标签栏带"最近使用文档"标题栏，hover 显示关闭按钮和文字右移效果，选中项加粗和左侧指示条
- 文件夹面板固定标题"资源管理器"，文件夹名独立行带关闭按钮
- 左侧面板与顶部 Tab 栏水平对齐（布局重构）
- 搜索框配色跟随主题、按钮选中圆角、F3/Shift+F3 导航、hover tooltip、失焦自动隐藏
- 编辑区右键菜单：剪切/复制/粘贴 + 插入图片/链接/表格/代码块/分隔线
- 视图 → 显示区域：支持编辑器+预览 / 仅编辑器 / 仅预览三种模式
- 切换 Tab 时内容区 150ms 淡入动画

### Changed
- "最近文件"菜单重命名为"最近打开"
- "重新显示欢迎页"菜单简化为"显示欢迎页"

### Fixed
- 演示模式（F11）现在只显示预览区，隐藏所有其他面板
- TOC 面板 item 焦点矩形边框已移除
- 启动恢复会话时文件夹面板自动选中当前文件
- 切换 Tab 时文件夹面板自动同步选中对应文件
- 标记重叠区域现在自动合并而非拒绝添加
- 标记跨越多个章节时 TOC 面板全部高亮
- 预览区选中拖拽到视口上下方时自动滚动
- 关闭文件夹后侧边 Tab 栏不再跳动
- 文件夹面板选中 item 的虚线焦点框已移除
- 文件夹面板全部菜单和右键菜单补全国际化翻译
- "打开文件夹"菜单补全国际化翻译

## [0.2.8] - 2026-04-17

### Added
- 预览区支持渲染本地图片（相对路径 / 绝对路径 / file:// URI）、在线图片（http/https 异步下载）和 data URI 内嵌图片
- 在线图片异步加载不阻塞 UI，加载中显示"Loading image..."占位符，失败显示错误提示
- 图片按预览区宽度等比缩放、水平居中显示
- 单图最大 20MB 限制，超过自动拒绝

## [0.2.7] - 2026-04-17

### Added
- Windows 标题栏跟随主题配色：通过 DWM API（DWMWA_CAPTION_COLOR / DWMWA_TEXT_COLOR）让系统标题栏背景和文字颜色自动匹配当前主题，弹窗标题栏同步适配（需 Win11 22H2+）

## [0.2.6] - 2026-04-17

### Added
- 编辑器语法染色补齐 `~~删除线~~` 与 `***粗斜体***`：原本输入 `~~foo~~` 在编辑区无任何样式提示（虽然预览渲染正常、`Ctrl+D` 也能包裹），现在会直接画出删除线；`***foo***` 现可同时呈现粗体+斜体；两者都尊重 inline code 范围（`` `~~foo~~` `` 内部不触发）

## [0.2.5] - 2026-04-15

### Added
- 目录面板支持按章节层级折叠：父级标题前显示 `+` / `−` 按钮，点击即可展开或收起其子节点
- 折叠状态按文件自动记住，重新打开同一文件时恢复上次的折叠形态
- 目录面板获得焦点后可用方向键导航：`←` 折叠当前节点或跳到父级、`→` 展开当前节点、`Enter` 跳转到对应标题行
- 目录面板宽度按当前文档标题内容自动调整：短标题自动收窄、长标题扩宽以完整显示无省略号；最大不超过屏幕宽度 1/5
- 手动拖动目录分隔条后保持用户自定义宽度，不再被自动重算覆盖
- 目录下方新增「文档信息」卡片，常驻显示字数、字符数（含/不含空格）、行数、文件大小、修改时间（以「刚刚 / 几分钟前 / 几天前」等相对时间展示）；点击标题栏可折叠，折叠态重启应用后保持
- 目录每条目左侧新增圆点标识（一级标题稍大、二级及以下稍小）；当前滚动到的章节圆点变柠檬黄并带柔光光环，位置一目了然
- 编辑区/预览区分隔条拖动时显示 1/4、1/2、3/4 位置的吸附参考虚线，拖到 5 像素内自动吸附到精确比例，还原等分布局更方便；命中吸附时对应虚线加粗高亮
- 视图 → 主题 → 「打开主题目录」首次打开（或目录被清空后）自动生成 `template.toml` 模板和 `HOW_TO.md` 说明，并弹窗指引下一步；目录已有自定义主题时不打扰、不覆盖
- 自动保存：编辑停顿 1.5 秒后静默保存当前所有有磁盘路径的标签页，未命名标签页跳过、不弹文件对话框；可在「设置 → 自动保存」开关，并在「设置 → 自动保存延迟」选 1.5 / 3 / 5 秒；保存失败时状态栏红字瞬时提示，5 秒后自动消失
- 状态栏右下新增「编码 / 换行 / 主题名 / 保存状态」四个 label：保存状态按文件状态自动切换为「未保存（新文件）」「未保存修改」（红字）「已保存 · 刚刚 / N 分钟前 / N 小时前 / N 天前」，每 30 秒自动刷新相对时间
- Windows 崩溃报告收集：应用异常退出时自动在 `%APPDATA%\SimpleMarkdown\crashes\` 写入 `crash.dmp`（minidump）和 `crash_report.txt`（异常代码 / 崩溃模块 / 时间戳）；下次启动检测到新崩溃时弹窗提示，可一键打开崩溃报告目录；不会自动上传，完全本地保存
- 性能监控（诊断用）：新增 `帮助 → 诊断 → 性能日志` 开关，启用后向 stderr 打印解析、预览布局、预览绘制三个关键路径的耗时（微秒），便于定位卡顿；默认关闭，Release 下几乎零开销；也可通过环境变量 `SM_PERF=1` 在启动时开启
- MarkdownParser 单元测试：新增 12 个用例覆盖标题级别 / 列表 / 代码块 / 行内代码 / 链接 / 图片 / 引用 / GFM 表格 / 换行 / 强调 / 空输入 / 行号等基础契约，为后续 cmark-gfm 升级或解析层重构提供回归保护
- Linux 下"跟随系统"主题检测：新增三级探测（GNOME `gsettings color-scheme` → KDE `~/.config/kdeglobals` → Qt Palette 亮度启发式），GNOME 42+ 和 KDE Plasma 现在能直接识别系统的深 / 浅偏好，其他 DE 沿用原 palette 启发式；gsettings 进程 500ms 超时保护，不阻塞启动

### Changed
- 标签栏现在横跨整个窗口宽度，目录面板只占据编辑区与预览区的高度，顶部三栏顶端对齐
- 目录条目悬停只改背景色、不再改字色，视觉更克制
- 当前章节在目录中的高亮样式升级：深色加粗文字 + 柔光圆点
- 状态栏右下去掉「主题名」label：切换主题时整个 UI 颜色已明显变化、菜单里有勾选指示，再显示文字名多余且占位

### Fixed
- 目录下方「文档信息」卡片、折叠/展开按钮的提示文字、相对时间文案（"just now / %1 min ago" 等）共 16 条原先未翻译，中文界面下显示英文；现已全部翻译
- 目录折叠/展开章节时面板宽度会跟着变化（长子节点被隐藏后宽度缩小）；现宽度只在首次加载 / 切换文档时计算一次，折叠/展开过程中保持稳定
- 编辑/预览分隔条拖到最右（或最左）时预览区/编辑区被压缩到 0 宽度、分隔条贴边无法再次抓回的 bug：现在两侧都至少保留 80px 可视空间，分隔条始终可抓；同时支持双击分隔条一键恢复 50/50 对称布局
- 打开主题目录后注入的 `template.toml` 样板会被「重新扫描主题」当成真实主题显示在菜单里：现在 loader 主动跳过这个文件名，样板只作参考不会出现在选项里；模板头注释和 HOW_TO.md 同步提示"必须改名才会被识别"

## [0.2.4] - 2026-04-14

### Added
- 主题插件系统：主题改由 TOML 配置文件驱动，内置 8 款主题可切换
  - Liquid Glass 组：冰霜蓝白、琥珀桃暖、深夜极光
  - 清新 / 极简组：纸面极简（GitHub 风）、薄荷清新（日系）、墨禅单色（宣纸墨书）
  - 经典组：Light、Dark
- 用户主题：把 TOML 文件放到 `%APPDATA%/SimpleMarkdown/themes/`（Linux：`~/.config/SimpleMarkdown/themes/`）即可在菜单出现；视图 → 主题 → 「打开主题目录 / 重新扫描主题」支持快速管理
- 主题配置支持行内代码与代码块分别设色（原先共用一组颜色），旧主题不写新字段会自动退回通用颜色
- 标签栏尾部新增 `+` 按钮，紧贴最后一个标签（Chrome / Edge 风格，随标签数量移动），点击新建标签
- 首次启动弹窗显示项目简介、7 条核心特性与快捷键提示；帮助菜单「重新显示欢迎页」可再次打开
- 标签右键菜单新增「打开所在目录」：Windows 下用资源管理器高亮选中文件，Linux 下打开包含目录
- 视图菜单新增「文档统计信息...」弹窗：字数、字符数、行数、段落、各级标题、代码块、图片、链接、表格、引用块、估算阅读时间
- 一键打包：`pack_on_win.bat` / `pack_on_linux.sh` 自动先完成 release 构建，无需先手动 build
- F11 演示模式：全屏化预览区（原行为是全屏编辑区），菜单项改名「演示模式」
- 预览区链接 Ctrl+点击跳转
  - 本地 `.md / .markdown / .txt` → 在新标签页打开
  - `http / https / mailto / ftp` → 系统默认应用
  - 其他文档类型 → 系统默认应用
  - `.exe / .bat / .sh` 等可执行文件 → 安全拒绝并提示
  - 文件不存在 → 友好提示，不崩溃
  - 鼠标悬停在链接上时光标变为手型
- 预览区渲染文档头部 YAML frontmatter 为圆角背景的二列表格：key 用等宽字体 + 主题色，value 与代码块同色，背景/边框按浅色/深色主题差异化

### Changed
- 搜索高亮改为亮黄色，浅色主题下更醒目、深色主题下由暗棕改为金黄
- 菜单项勾选符号（✓）与文字之间的间距增大，不再紧贴

### Fixed
- 多款浅色主题下弹窗按钮原先是系统灰白，未体现主题色；现按钮 hover / 按下 / 默认状态均使用主题强调色
- 会话恢复的标签页不跟随当前主题：打开应用时沿用的旧标签仍显示上次的主题色；现自动重新应用当前主题
- 菜单栏与标签栏颜色相同时视觉融成一块；现菜单栏下方加一道跟随主题色的分隔线
- 预览区 frontmatter 块无法框选 / 复制；现整块可选、可复制（输出原始 YAML 文本）
- 主题子菜单中「主题 / 打开主题目录 / 重新扫描主题 / 字符 / 演示模式」等条目在中文界面下仍显示英文；现已全部翻译
- 深色主题（如深夜极光）下应用外壳（菜单栏、标签栏、状态栏、滚动条）不跟主题；现全部跟随
- 目录面板原先硬编码深灰色不跟主题；现颜色跟随主题派生
- 预览区代码块字号偏小：与编辑器代码字号不一致；现视觉等价
- 冰霜蓝白、琥珀桃暖、墨禅单色三款浅色主题的代码块改为浅底深字，不再反色突兀
- 切换主题后编辑器中的代码块 / 行内代码残留上一主题的背景条；现正确清除
- 反色代码块主题（如墨禅单色）下 frontmatter 值在浅背景上看不见；现仅在主题未显式声明时才派生颜色
- 标签右键菜单「关闭其他 / 关闭左侧 / 关闭右侧」按上下文禁用无对象的项（只有 1 个标签、或已在最左/最右时置灰）
- 预览区 Ctrl+滚轮缩放对部分元素无效（代码块、列表序号、表格单元格、图片占位符等）；现全部跟随缩放
- 拖拽无后缀的纯文本文件（TODO / LICENSE / README 等）到编辑区现可打开；明确的二进制类型（.exe / .zip / .pdf 等）仍拒绝并弹窗说明
- 首次启动欢迎页在中文界面下显示英文：翻译文件未被正确打进资源；现已修复，后续新 UI 走完整 i18n 流程
- 行内代码（反引号包围片段）的垂直基线漂移：小字号片段显得向上翘；现所有片段共享行主基线，不再抖动
