# Changelog

All notable changes to this project will be documented in this file.

## [1.1.1] - 2026-05-07

### Fixed
- 在预览区打开查找栏后切焦点到编辑区再按 Ctrl+F，预览区查找栏会自动关闭并把焦点移交给编辑区查找栏（反向同样成立）；任何时刻最多只显示一个查找栏，避免双栏共存让用户分不清当前在哪一侧搜索

## [1.1.0] - 2026-05-07

### Added
- 预览区支持 Ctrl+F 查找：输入关键词后高亮所有命中，按 F3 / Shift+F3 跳转上下匹配项，Esc 关闭并清除高亮；编辑器和预览区同时打开时 Ctrl+F 按当前焦点决定唤起哪一侧（之前查找仅编辑器可用，阅读长文档需切回编辑器才能定位）

## [1.0.11] - 2026-05-07

### Fixed
- 拖动文档中央分隔条时 1/4、1/2、3/4 三档吸附刻度的实时高亮恢复，靠近刻度时对应辅助线立即变粗，松手时自动对齐（1.0.10 改为只显示参考虚线后该高亮反馈意外丢失）

## [1.0.10] - 2026-05-07

### Changed
- 拖动文档中央分隔条时改为只显示一条参考虚线指示位置，编辑区与预览区在松开手柄时才一次性调整为新宽度（之前两侧区域会跟随手柄实时变化但内容不重排，造成旧布局被裁切或留白的视觉混乱）

## [1.0.9] - 2026-05-07

### Improved
- 拖动文档中央编辑区与预览区之间的分隔条调整宽度时不再卡顿（千行以上的长文档体感尤其明显）；松开手柄时内容一次性更新到新宽度

## [1.0.8] - 2026-05-06

### Fixed
- 在预览区添加的内容标记（荧光笔）现在会随会话保存，重启应用后再次打开同一文档可见标记仍在原位置（此前关闭后即丢失）
- 文档被其他程序频繁修改时不再连续弹出多个"是否重新加载"对话框，每个文档同一时刻最多只显示一个；用户响应后新一轮修改才会再次提示
- 左侧资源管理器和右侧目录面板拖拽时的最大宽度限制为窗口所在屏幕的四分之一，避免主编辑区被压缩过窄
- 调整预览区宽度时表格单元格内容不再越界与下一行单元格重叠（特别是窄列、中英文混排、含长链接的场景）
- 编辑文档导致预览区标记被清除时，右侧目录上的对应章节标记指示同步消失，不再出现"预览没了，目录还在"的错位
- 含 frontmatter 的文档中添加标记后，目录上高亮的章节现在与标记真实所在的章节一致（此前因内部字符偏移计算遗漏 frontmatter 长度，目录高亮会错位到上方相邻章节）
- 渲染区右键菜单的"取消标记"项：在标记**附近空白处**右键时现在正确置灰（此前因点击位置 snap 到最近字符可能误命中相邻标记，导致空白处右键菜单仍可点）
- 文档头部含多行 YAML 列表（如 `related_specs: - foo - bar`）的 frontmatter，列表项现在在卡片左侧对齐显示，不再被缩进到值列右侧
- 含中英文混排长字段的 frontmatter 卡片，文本换行后完全留在卡片内，不再溢出右边框
- 预览区顶部新增视觉间距，文档首块（frontmatter 卡片、标题、段落等）不再紧贴上边界
- 编辑文档时已添加的标记（荧光笔）现在会保留在原位，不再被每次输入抹除
- 编辑区与预览区字号视觉显著更一致——大写字母高度与行整体高度从原本相差 30%+ 改善到几乎相等，主观字号感受不再"两边像两个程序拼起来的"

### Improved
- 长时间使用打开许多文档时，超过 20 个的最早打开的文档会自动进入"休眠"状态（保留 Tab 头但卸载内容），点击 Tab 时自动重新加载，避免内存随会话长度无限增长（未保存修改的文档不会被休眠）
- Windows 任务管理器和文件属性现在显示友好的应用名称"SimpleMarkdown"，而非裸文件名；版本号与发行版本同步，并在中英文系统下分别显示对应描述
- 千行规模文档调整宽度时预览区重排耗时显著下降，长文档的实时拖拽体验从接近 30fps 提升到接近 60fps

## [1.0.7] - 2026-04-29

### Fixed
- 关闭所有打开的文档后，右侧的目录与文档信息面板自动隐藏，让欢迎页占据完整的中央与右侧空间（此前面板仍挂在窗口右侧，显示空目录卡片）
- 调整文档标签页中编辑区与预览区的宽度比例后，重启应用时该比例恢复（此前总是重置为 50/50 默认值）
- 侧边模式下调整左侧资源管理器与"最近使用文档"面板的高度占比后，重启应用时该占比恢复

## [1.0.6] - 2026-04-27

### Added
- 应用启动时窗口默认最大化（首次启动或无保存几何时生效；用户后续手工调整的尺寸/位置仍会被尊重并跨会话恢复）

### Fixed
- 资源管理器面板和目录面板的宽度在用户调整后能在重启时正确恢复（此前在大屏幕下被错误截断)
- 预览区放大字号或拉窄宽度导致标题（H1/H2 等）换行时，第二行与下方段落重叠
- 预览区相邻列表项之间不再出现接近一行高度的多余空白
- 首次打开/粘贴含粗体的列表项时不再短暂出现多余空白（需缩放一次才消失）
- 视图 → 行间距 切换 1.0× / 2.0× 等档位时预览区行高同步变化（此前只作用于编辑区，预览保持原行高，与编辑区视觉错位）
- 预览区代码块中超出宽度的长行会自动软换行（此前长 URL / 长字符串等会从代码块右边界飞出，且后续段落被压在代码块上）
- 快捷键面板表格选中行不再出现单元格虚线焦点框
- 快捷键面板"类别"列文字改为水平居中

### Improved
- 快捷键 / 更新历史 / 关于 三个对话框的关闭按钮风格统一（圆角、内边距、accent 边框、悬停态），主题切换后仍保持一致
- 欢迎页、快捷键速查、更新日志、关于 四处主操作按钮像素级一致（accent 实心、白字、圆角、横向铺满），关于对话框由弹窗改为常规对话框便于按钮铺满

### Removed
- 撤销编辑器焦点下 F3 / Shift+F3 跳转匹配项的尝试（多次实现均验证未通过，已废弃）

## [0.2.5] - 2026-04-15

### Added
- 目录面板支持按章节层级折叠：父级标题前显示 `+` / `−` 按钮，点击即可展开或收起其子节点
- 折叠状态按文件自动记住，重新打开同一文件时恢复上次的折叠形态
- 目录面板获得焦点后可用方向键导航：`←` 折叠当前节点或跳到父级、`→` 展开当前节点、`Enter` 跳转到对应标题行
- 目录面板宽度按当前文档标题内容自动调整：短标题自动收窄、长标题扩宽以完整显示无省略号；最大不超过屏幕宽度 1/5
- 手动拖动目录分隔条后保持用户自定义宽度，不再被自动重算覆盖
- 目录下方新增「文档信息」卡片，常驻显示字数、字符数（含/不含空格）、行数、文件大小、修改时间（以「刚刚 / 几分钟前 / 几天前」等相对时间展示）；点击标题栏可折叠，折叠态重启应用后保持
- 目录每条目左侧新增圆点标识（一级标题稍大、二级及以下稍小）；当前滚动到的章节圆点变柠檬黄并带柔光光环，位置一目了然
- 编辑区/预览区分隔条拖动时显示 1/4、1/2、3/4 位置的吸附参考虚线，拖到 5 像素内自动吸附到精确比例，还原等分布局更方便；命中吸附时对应虚线加粗高亮
- 视图 → 主题 → 「打开主题目录」首次打开（或目录被清空后）自动生成主题模板和说明文件，并弹窗指引下一步；目录已有自定义主题时不打扰、不覆盖
- 自动保存：编辑停顿 1.5 秒后静默保存当前所有有磁盘路径的标签页，未命名标签页跳过、不弹文件对话框；可在「设置 → 自动保存」开关，并在「设置 → 自动保存延迟」选 1.5 / 3 / 5 秒；保存失败时状态栏红字瞬时提示，5 秒后自动消失
- 状态栏右下新增「编码 / 换行 / 主题名 / 保存状态」四个标签：保存状态按文件状态自动切换为「未保存（新文件）」「未保存修改」（红字）「已保存 · 刚刚 / N 分钟前 / N 小时前 / N 天前」，每 30 秒自动刷新相对时间
- Windows 崩溃报告收集：应用异常退出时自动在用户配置目录的 `crashes\` 子目录写入崩溃转储和报告（包含异常代码 / 崩溃模块 / 时间戳）；下次启动检测到新崩溃时弹窗提示，可一键打开崩溃报告目录；不会自动上传，完全本地保存
- 性能监控（诊断用）：新增 `帮助 → 诊断 → 性能日志` 开关，启用后向终端打印解析、预览布局、预览绘制三个关键路径的耗时，便于定位卡顿；默认关闭，正常使用几乎零开销
- Linux 下"跟随系统"主题检测：新增对 GNOME 42+ 与 KDE Plasma 系统深 / 浅偏好的识别，其他桌面环境沿用原有启发式判断；不阻塞启动

### Changed
- 标签栏现在横跨整个窗口宽度，目录面板只占据编辑区与预览区的高度，顶部三栏顶端对齐
- 目录条目悬停只改背景色、不再改字色，视觉更克制
- 当前章节在目录中的高亮样式升级：深色加粗文字 + 柔光圆点
- 状态栏右下去掉「主题名」标签：切换主题时整个界面颜色已明显变化、菜单里有勾选指示，再显示文字名多余且占位

### Fixed
- 目录下方「文档信息」卡片、折叠/展开按钮的提示文字、相对时间文案（"just now / %1 min ago" 等）共 16 条原先未翻译，中文界面下显示英文；现已全部翻译
- 目录折叠/展开章节时面板宽度跟着变化（长子节点被隐藏后宽度缩小）；现宽度只在首次加载 / 切换文档时计算一次，折叠/展开过程中保持稳定
- 编辑/预览分隔条拖到最右（或最左）时预览区/编辑区被压缩到 0 宽度、分隔条贴边无法再次抓回；现在两侧都至少保留 80 像素可视空间，分隔条始终可抓；同时支持双击分隔条一键恢复 50/50 对称布局
- 打开主题目录后注入的样板模板会被「重新扫描主题」当成真实主题显示在菜单里：现在样板只作参考不会出现在选项里；模板头注释和说明文件同步提示"必须改名才会被识别"

## [0.2.4] - 2026-04-14
